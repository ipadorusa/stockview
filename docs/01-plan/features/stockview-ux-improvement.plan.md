# Plan: StockView UX 개선 — 초보 투자자 경험 강화

## Executive Summary

| 관점 | 내용 |
|------|------|
| **Problem** | 초보 투자자가 금융 용어를 이해하지 못해 이탈하고, 온보딩/검색/관심종목 접근이 비효율적이며, 브랜드 정체성이 부재 |
| **Solution** | 인라인 용어 툴팁, 3단계 온보딩, 모바일 바텀탭 5개 확장, 브랜드 컬러 그린 전환, 폰트 성능 최적화 |
| **Function UX Effect** | 종목 상세 체류시간 +30%, 가입 후 7일 리텐션 +30%, 핵심 플로우 탭 수 3→1, LCP 200~500ms 개선 |
| **Core Value** | "배우면서 투자하는 초보자의 첫 번째 플랫폼" — 네이버 증권/Yahoo Finance에 없는 학습형 UX로 차별화 |

---

## 1. 배경 및 목적

### 1.1 현재 상황
- StockView는 39페이지, 75+ 컴포넌트의 듀얼마켓(한국/미국) 주식 정보 플랫폼
- 타겟: 투자 경력 1년 미만 초보 투자자, 모바일 사용 비중 높음
- 기능은 풍부하나(시세, 차트, 뉴스, 스크리너, AI리포트, ETF, 섹터 등), 초보자 맞춤 UX 부족
- 7명 팀 토론(2026-03-28) 결과를 기반으로 핵심 5개 항목 선정

### 1.2 핵심 원칙
- **실시간 시세는 핵심 가치가 아님** — 차트 분석, 리포트 제공, 학습이 핵심
- **무료 자원만 사용** — 추가 비용 0원
- **기존 자산 재활용** — 새 기술 도입 전 이미 있는 것 최대 활용
- **초보자 차별화** — 경쟁사에 없는 "학습형 UX"

### 1.3 범위
본 Plan은 **2주 스프린트**로 실행 가능한 5개 개선 항목을 다룹니다.

| 포함 | 제외 |
|------|------|
| 용어 툴팁 + 지표 해석 | 실시간 시세 (SSE/WebSocket) |
| 온보딩 플로우 | 가격 알림 (Push Notification) |
| 모바일 바텀탭 5개 | PWA 전환 |
| 브랜드 컬러 + 다크모드 접근성 | IA 전면 재설계 |
| Pretendard 폰트 self-host | AppHeader 서버/클라이언트 분리 |

## 2. 개선 항목 상세

### 2.1 [P0] 용어 툴팁 + 지표 해석 가이드

**문제**: PER, PBR, ROE, RSI 등 전문 지표가 설명 없이 숫자만 나열. 가이드 페이지(5개)가 있지만 종목 상세와 분리되어 맥락 학습 불가.

**솔루션**:
- 각 지표에 `(?)` 인라인 툴팁: 1~2문장 정의 + 업종평균 비교
- 신호등 컬러: 업종 평균 대비 초록(양호)/노랑(보통)/빨강(주의)
- 52주 Range Bar: CSS만으로 현재가 위치 시각화
- "더 알아보기" 링크 → 가이드 페이지 해당 섹션 딥링크

**대상 파일**:
- `src/components/stock/stock-info-grid.tsx` — 지표 그리드에 툴팁 추가
- `src/components/common/tooltip-helper.tsx` — 기존 컴포넌트 확장 또는 신규 `<TermTooltip>` 생성
- `src/lib/glossary.ts` (신규) — 용어 사전 데이터 `{ term, shortDesc, guideHref, evaluate(value, sectorAvg) }`

**의존성**: 없음 (shadcn Tooltip 이미 사용 중)
**공수**: 3~4일
**성공 지표**: 종목 상세 체류시간 +30%, 가이드 페이지 유입(툴팁 경유) 일 200+ 클릭

---

### 2.2 [P0] 신규 사용자 온보딩 플로우

**문제**: 회원가입 후 온보딩 0건. 39페이지 중 어디서 시작할지 모름. 빈 관심종목의 차가운 첫인상.

**솔루션**:
- 회원가입 직후 3단계 바텀시트:
  1. 관심 시장 선택 (한국/미국/둘 다)
  2. 관심 섹터 3개 선택 (기존 섹터 데이터 재활용)
  3. 인기 종목 중 3개 관심종목 추가 (PopularStocksTabs 데이터 재활용)
- `localStorage` 기반 `hasCompletedOnboarding` 플래그 — DB 스키마 변경 없음
- 비로그인 사용자: 홈 상단 "초보 투자자 가이드" CTA 배너

**대상 파일**:
- `src/components/onboarding/onboarding-sheet.tsx` (신규) — 3단계 바텀시트
- `src/app/auth/register/page.tsx` — 가입 완료 후 온보딩 트리거
- `src/app/page.tsx` — 비로그인 CTA 배너 추가

**의존성**: shadcn Sheet 컴포넌트 (이미 있음), 인기종목 API (이미 있음)
**공수**: 2~3일
**성공 지표**: 온보딩 완료율 60%+, 가입 후 관심종목 추가율 50%+, D7 리텐션 +30%

---

### 2.3 [P0] 모바일 바텀탭 5개 확장

**문제**: 현재 4탭(홈/시장/뉴스/MY)에 검색과 관심종목 없음. 핵심 루프(검색→상세→관심종목)의 시작점과 종착점이 1탭 접근 불가.

**솔루션**:
- 바텀탭: `홈 | 검색 | 시장 | 관심 | MY`
- 검색 탭: 기존 `SearchCommand` 컴포넌트를 풀스크린 오버레이로 재활용
- 관심 탭: `/watchlist` 직접 연결 (비로그인 시 로그인 유도)

**대상 파일**:
- `src/components/layout/bottom-tab-bar.tsx` — 5탭으로 확장

**의존성**: 없음
**공수**: 반일 (4시간)
**성공 지표**: 관심종목 페이지 접근 탭 수 3→1, 검색 사용 빈도 +50%

---

### 2.4 [P0] 브랜드 컬러 그린 전환 + 다크모드 접근성

**문제**:
- PRD에 "브랜드 컬러: 그린"이지만 실제 `--primary`는 보라/남색(`oklch(0.45 0.15 260)`)
- 다크모드에서 `stock-up-bg` + `text-stock-up` 조합이 WCAG AA 미달 (대비비 3.2:1)

**솔루션**:
- `--primary` 계열을 그린으로 변경: `oklch(0.45 0.16 145)` (라이트), `oklch(0.65 0.18 145)` (다크)
- 다크모드 `--color-stock-up-bg`를 `#3d1f1f`, `--color-stock-down-bg`를 `#1f2d3f`로 조정 (대비비 4.5:1+)
- 등락 표시에 `▲`/`▼` 기호 일관 적용

**대상 파일**:
- `src/app/globals.css` — CSS 변수 6~8개 수정

**의존성**: 없음
**공수**: 2시간
**성공 지표**: Lighthouse 접근성 점수 90+, 브랜드 인지 확립

---

### 2.5 [P0] Pretendard 폰트 self-host + 스크립트 최적화

**문제**: 외부 CDN(jsdelivr)에서 렌더 블로킹 CSS로 Pretendard 로딩. 이중 워터폴(CSS→woff2). GTM 인라인 스크립트도 렌더 경로 차단.

**솔루션**:
- `next/font/local`로 Pretendard Variable woff2 self-host
- GTM consent 스크립트를 `next/script strategy="afterInteractive"`로 이동
- `font-variant-numeric: tabular-nums` 글로벌 적용

**대상 파일**:
- `src/app/layout.tsx` — 외부 CSS link 제거, next/font/local 적용
- `src/app/globals.css` — tabular-nums 추가
- `public/fonts/` (신규) — Pretendard Variable subset woff2 파일

**의존성**: Pretendard GitHub releases에서 woff2 다운로드
**공수**: 2~3시간
**성공 지표**: LCP 200~500ms 개선, CLS 0.05~0.1 개선
