## 상위 3대 강점

1. **리서치 철저성이 탁월하다.** 14개 리서치 파일에 걸쳐 3,818줄, 실제 저장소(next.js, grafana)에서 `@import` 동작을 검증, Claude Code가 AGENTS.md를 네이티브로 읽지 않음을 확인. 기술적 기반이 탄탄하다.

2. **`@import` 브리지는 검증되었다.** 호환성 리서치가 공식 Anthropic 패턴을 정확히 식별하고 프로덕션 저장소에서 검증했다. 이 특정 메커니즘은 작동한다.

3. **위험 식별이 솔직하다.** 네이티브 지원 불확실성, 컨텍스트 예산 우려, 노후화 위험을 명시한 것은 자기인식을 보여준다. 이것들이 실제 실패 모드다.

## 상위 3대 약점

1. **문제를 찾는 솔루션이다.** 현재 CLAUDE.md는 88줄로 — 간결하고, 정확하며, 이미 잘 작동하고 있다 (최근 커밋을 보면 크론 수정, API 전환, 페이지 삭제가 올바르게 이루어짐). 에이전트 성능이 현재 설정으로 인해 저하되고 있다는 증거가 없다. 1인 개발자 프로젝트에 5단계, 수주 계획, 5개 에이전트 역할은 심각하게 과도하다.

2. **하네스 전략이 측정 가능한 성과 없이 조직적 복잡성만 추가한다.** 5개 에이전트 역할(CTO Lead, Plan, Code, Gap Detector, Review)과 `settings.local.json` 권한 모델 — 1명이 관리하는 281개 파일 Next.js 앱에. 현재 워크플로우(CLAUDE.md + bkit PDCA + context-mode)가 이미 작동하는 코드를 생산하고 있다. 에이전트 라우팅, 메모리 시딩, 커스텀 스킬을 추가하면 어떤 그럴듯한 생산성 향상보다 유지보수 부담이 더 커진다.

3. **AGENTS.md는 Claude Code에 구체적으로 제로 가치를 제공한다.** Claude Code는 CLAUDE.md만 읽고 계획의 자체 리서치가 이를 확인하므로, AGENTS.md를 만드는 것은 Cursor, Windsurf, Copilot 또는 다른 도구도 함께 사용하는 경우에만 유용하다. Claude Code가 유일한 에이전트라면, AGENTS.md는 동기화를 유지해야 하는 중복 사본 — 순수한 오버헤드다.

## 대안 접근법

이 계획 대신, 딱 한 가지만 하겠다:

**CLAUDE.md를 현재대로 유지. Vercel 보일러플레이트(65-88줄)를 제거한다 — 자동 주입된 것이며 범용/프로젝트 비특이적이다.** 이렇게 하면 ~64줄의 순수 신호가 된다. 5분이면 완료. AGENTS.md 없음, 하네스 없음, 단계 없음.

멀티툴 지원이 실제 필요해질 때(예: Claude Code와 함께 Cursor를 사용하기 시작하면), 그때 CLAUDE.md에서 공유 콘텐츠를 추출하고 `@AGENTS.md` import를 사용하여 AGENTS.md를 생성한다. 필요가 현실화되었을 때 30분 작업이다.
