## 1. UX 수행 요약

### 전체 UX 건강도: 6.5 / 10

StockView는 디자인 시스템의 기초가 잘 잡혀 있다. 색상 토큰, 타이포그래피, 컴포넌트 라이브러리(shadcn/ui)를 기반으로 한 체계적 구조가 인상적이다. 그러나 "디자인 시스템을 갖추고 있다"와 "사용자가 일관된 경험을 느낀다"는 별개의 문제이며, 현재는 후자에서 격차가 존재한다.

**강점**
- 주가 색상(빨강/파랑) 한국 관례 준수 및 토큰화 완료
- 다크 모드 색상 체계가 체계적 (light/dark 각각 별도 토큰)
- shadcn/ui 기반 컴포넌트 상태(hover/focus/disabled/active) 완비
- PageContainer를 통한 레이아웃 일관성 확보
- Pretendard 폰트 체인 및 tabular-nums 지원

**핵심 약점**
- 빈 상태(empty state) 디자인이 2가지 패턴으로 분열됨
- 에러/404 페이지에 아이콘이나 일러스트가 없어 감정적 연결 부재
- 한국어 톤(친근체/격식체) 혼재로 브랜드 정체성 불명확
- Desktop/Mobile 정보 아키텍처 라벨 불일치
- 여러 페이지에 loading skeleton 누락

### 2.1 색상 팔레트 일관성: 8.5 / 10

| 항목 | 평가 | 근거 |
|------|------|------|
| 주가 색상(상승/하락/보합) | **우수** | `text-stock-up/down/flat` 토큰 사용, PriceChangeText 컴포넌트 재사용 패턴 |

## 3. 빈 상태 / 에러 상태 감사

### 3.1 빈 상태 디자인 감사

현재 빈 상태는 **2가지 패턴**이 혼재하며, 사용자 경험의 품질 차이가 크다.

#### 패턴 A: EmptyState 컴포넌트 (풍부한 디자인)
- 아이콘(원형 배경) + 제목 + 설명 + CTA 버튼
- 적용 페이지: **관심종목 목록**, **포트폴리오 탭** (총 2곳)

#### 패턴 B: 단순 텍스트 (최소 디자인)
- `text-sm text-muted-foreground text-center py-8` 한 줄 텍스트
- 적용 페이지: **24곳 이상** (배당, 실적, 공시, 스크리너, 섹터, ETF, 뉴스, 게시판 댓글, 홈 뉴스 등)

#### 빈 상태 없는 페이지
- `/compare` — 빈 슬롯은 검색 Input으로 대체 (적절한 패턴)
- `/stock/[ticker]` — stock이 null일 때 `notFound()` 미호출 (FAIL, 빈/깨진 UI 노출 위험)

**디자이너 판정**: 패턴 B는 사용자에게 "시스템이 아직 미완성"이라는 인상을 준다. 빈 상태는 사용자가 실망할 수 있는 순간이며, 오히려 이때 브랜드 감성을 전달할 기회다. 최소한 아이콘 + 안내 문구 + 다음 행동 유도(CTA)가 있어야 한다.

### 3.2 에러 상태 디자인 감사

| 페이지 | 에러 처리 | 디자인 품질 | 평가 |
|--------|----------|------------|------|
| 글로벌 404 (`not-found.tsx`) | 텍스트 기반 (text-6xl "404") | 기능적이나 감정적 연결 없음 | **C+** |
| 글로벌 에러 (`error.tsx`) | "오류가 발생했습니다" + 다시 시도 버튼 | 최소 기능적 | **C** |
| 시장 에러 (`market/error.tsx`) | 맥락별 메시지 + 다시 시도 | 글로벌과 동일 패턴 | **C+** |
| 뉴스 에러 (`news/error.tsx`) | 맥락별 메시지 + 다시 시도 | 글로벌과 동일 패턴 | **C+** |
| 종목 에러 (`stock/error.tsx`) | 맥락별 메시지 + 다시 시도 | 글로벌과 동일 패턴 | **C+** |

**공통 문제점**:
1. 모든 에러 페이지에 **아이콘이나 일러스트가 없음** — 사용자가 "무엇이 잘못되었는지" 시각적으로 즉시 인지하기 어려움
2. 에러 페이지에 **홈으로 돌아가기 링크가 없음** (404만 있음) — 사용자가 "다시 시도"만 반복하다 이탈할 수 있음
3. **톤 불일치**: 404 "찾을 수 없**어요**" (친근) vs error "발생했**습니다**" (격식)

### 3.3 404/500 페이지 디자인 품질

- **404**: text-6xl 숫자 + 텍스트. 기능적으로는 충분하나, 금융 서비스의 신뢰감을 고려하면 아이콘(SearchX, FileQuestion 등)과 관련 페이지 추천이 있으면 좋겠다.
- **500**: 글로벌 error.tsx로 대응. `min-h-[50vh]` 중앙 배치는 적절하나, 에러 코드 표시나 고객 지원 링크가 없다.

## 4. UX Flow 문제 분석

### 4.1 callbackUrl 미작동 (AUTH-012 / F-011) — 심각도: P0

**사용자 시나리오**:
1. 비로그인 상태에서 `/watchlist` 접근
2. 미들웨어가 `/auth/login?callbackUrl=/watchlist`로 리다이렉트
3. 사용자가 로그인 성공
4. **기대**: `/watchlist`로 이동 → **실제**: 항상 `/`(홈)으로 이동

**감정적 영향**: 사용자는 자신이 가려던 곳을 기억하고 있다. 로그인 후 엉뚱한 곳에 도착하면 "내가 뭘 하려고 했지?"라는 혼란과 함께 서비스에 대한 미세한 불신이 쌓인다. 특히 관심종목을 확인하려던 반복 사용자에게는 매번 2클릭이 추가되는 것이다.

**영향 범위**: `/watchlist`, `/settings`, `/mypage`, `/reports/stock` 등 보호 라우트 전체

### 4.2 /portfolio 데드 링크 (NAV-006) — 심각도: P1

**사용자 시나리오**:
1. 모바일 사이드시트 또는 Desktop "더보기" 메뉴에서 "포트폴리오" 클릭
2. **기대**: 포트폴리오 페이지 → **실제**: 404 페이지

**감정적 영향**: 내비게이션에 있는 링크가 404를 반환하는 것은 "집 안에 문은 있는데 벽만 있는" 경험이다. 사용자는 서비스가 미완성이라고 느끼며, 다른 메뉴도 의심하게 된다. 실제 포트폴리오 기능은 `/watchlist` 탭에 존재하므로 **발견 가능성(discoverability)도 저해**된다.

### 4.3 MAX_SLOTS 5→4 불일치 (CMP-010) — 심각도: P2

**사용자 시나리오**:
1. 홈에서 "최대 5종목 비교 분석" 문구 확인
2. 비교 페이지에서 4번째 종목까지만 추가 가능
3. 5번째 종목 추가 시도 → **아무 반응 없이 무시됨** (toast 등 알림 없음)

**감정적 영향**: 이중 문제다. (1) 문구와 실제 제한이 다르므로 "약속 위반" 느낌. (2) 제한 도달 시 피드백이 없으므로 "시스템이 고장났나?" 불안감. 사용자는 버그라고 판단할 가능성이 높다.

### 4.4 한국어 톤 혼재 (BR-007) — 심각도: P1

| 위치 | 표현 | 톤 |
|------|------|-----|
| 404 페이지 | "찾을 수 없**어요**" | 친근체 (해요체) |
| 글로벌 에러 | "발생했**습니다**" | 격식체 (합쇼체) |
| 시장 에러 | "불러올 수 **없습니다**" | 격식체 |
| 관심종목 빈 상태 | "관심종목이 **없어요**" | 친근체 |
| 뉴스 빈 상태 | "뉴스가 **없습니다**" | 격식체 |

**감정적 영향**: 톤의 혼재는 마치 한 사람이 존댓말과 반말을 번갈아 쓰는 것과 같다. 금융 서비스는 일관된 톤을 통해 "믿을 수 있는 파트너"라는 인상을 주어야 한다. 현재 상태는 브랜드 정체성이 아직 확립되지 않았음을 드러낸다.

**권장**: 금융 서비스 특성상 **해요체**(친근하지만 존중하는)로 통일. 예: "찾을 수 없어요", "불러올 수 없어요", "다시 시도해 주세요"

## 5. 네비게이션 UX 감사

### 5.1 Desktop ↔ Mobile 정보 아키텍처 비교

| Desktop (navCategories) | Mobile (navGroups) | 불일치 |
|------------------------|-------------------|--------|
| "분석" | "분석 도구" | 라벨 다름 |
| "분석 요청" (subLink) | "리포트 요청" (link) | 용어 다름 |
| "뉴스" (카테고리) | "뉴스 · 커뮤니티" (그룹) | 라벨 다름 |
| "더보기" (카테고리) | "MY" (그룹) | 라벨/개념 다름 |
| "소개" (subLink) | 없음 | 누락 |

**영향**: 사용자가 Desktop에서 학습한 메뉴 구조를 Mobile에서 다시 찾아야 한다. "분석"과 "분석 도구"가 같은 것인지 확신할 수 없다. 특히 "더보기"와 "MY"는 완전히 다른 개념 — Desktop에서는 "기타 기능 모음", Mobile에서는 "내 계정".

### 5.2 메뉴 라벨 차이의 영향

- **인지 부하 증가**: 사용자는 같은 기능을 두 가지 이름으로 기억해야 함
- **크로스 디바이스 사용자 혼란**: 출퇴근 시 모바일, 사무실에서 Desktop 사용하는 패턴에서 특히 문제
- **SEO 관점**: 일관된 용어 사용이 내부 링크 구조 일관성에도 영향

### 5.3 검색 UX (SearchCommand) 접근성

| 접근 경로 | 방법 | 평가 |
|----------|------|------|
| Desktop 키보드 | Cmd+K / Ctrl+K | **우수** — 파워 유저 친화적 |
| Desktop 마우스 | 헤더 검색바 클릭 | **양호** — SearchBar가 인라인이라 바로 입력 가능 |
| Mobile | 하단 탭바 검색 아이콘 | **양호** — SearchCommand 모달 오픈 |
| Mobile 사이드시트 | 인라인 SearchBar | **주의** — 모달이 아닌 인라인 검색, 행동 패턴 불일치 |

### 5.4 주요 Flow 클릭 수 (Task Completion Efficiency)

| 작업 | Desktop 클릭 수 | Mobile 클릭 수 | 업계 평균 | 평가 |
|------|----------------|---------------|----------|------|
| 종목 검색 → 상세 보기 | 2 (Cmd+K → 결과 클릭) | 2 (검색 탭 → 결과 클릭) | 2~3 | **적정** |
| 관심종목 추가 | 2 (상세에서 별 클릭 → 확인) | 2 | 2~3 | **적정** |
| 관심종목 확인 | 1~2 | 1 (하단 관심 탭) | 1~2 | **적정** |
| 종목 비교 시작 | 3+ (관심종목 → 비교 아이콘 × N → 비교 페이지) | 3+ | 3~4 | **적정** |
| 포트폴리오 확인 | **2 (더보기→포트폴리오 404!)** | **3+ (MY→관심→탭 전환)** | 1~2 | **불량** |

## 6. 시각적 개선 제안

### 6.1 Loading 스켈레톤 표준화 제안

**현황**: 7개 페이지에 loading.tsx 존재, 6개 주요 페이지에 누락 (`/board`, `/reports`, `/dividends`, `/earnings`, `/sectors`, `/compare`)

**표준 가이드라인**:
- 모든 SSR/ISR 페이지에 loading.tsx 필수 배치
- 스켈레톤은 실제 콘텐츠 레이아웃을 모방해야 함 (content-aware skeleton)
- `rounded-xl`(카드), `rounded-lg`(행)으로 radius 통일
- 높이는 실제 콘텐츠와 동일 — "깜빡임(layout shift)" 방지
- 기존 `market/loading.tsx`를 모범 사례로 삼을 것 (그리드 구조 모방)

### 6.2 Empty State 디자인 가이드라인 제안

**원칙**: 빈 상태는 "아무것도 없다"를 알려주는 것이 아니라, "여기에 무엇이 올 수 있는지"를 보여주는 것이다.

**표준 구조** (EmptyState 컴포넌트 확장):
```
[아이콘 - 원형 muted 배경]
[제목 - text-lg font-semibold]
[설명 - text-sm text-muted-foreground, 최대 2줄]
[CTA 버튼 - 선택적]
```

**페이지별 적용**:
- 데이터 미존재: 아이콘 + "아직 ~이(가) 없어요" + 데이터 수집 중 안내
- 필터 결과 없음: 아이콘 + "조건에 맞는 ~이(가) 없어요" + 필터 초기화 CTA
- 미로그인 상태: 아이콘 + "로그인하면 ~을(를) 이용할 수 있어요" + 로그인 CTA

### 6.3 Error State 디자인 가이드라인 제안

**표준 구조**:
```
[아이콘 - AlertTriangle 또는 맥락별 아이콘, h-12 w-12 text-muted-foreground]
[제목 - text-xl font-bold]
[설명 - text-sm text-muted-foreground]
[주요 CTA - "다시 시도" Button variant="outline"]
[보조 CTA - "홈으로 돌아가기" 텍스트 링크]
```

**404 개선안**: SearchX 아이콘 추가, "이런 페이지를 찾고 계셨나요?" 관련 링크 제안
**500 개선안**: ServerCrash 아이콘 추가, 홈 링크 + 문의하기 링크 추가

### 6.4 애니메이션/트랜지션 표준 제안

**현황 평가**: 금융 서비스에 적합한 절제된 애니메이션이 전반적으로 잘 구현되어 있다.

| FloatingBar 진입 | slide-in-from-bottom, 200ms ease-out | **유지** — 적절 |
| Button 상태 전환 | transition-all | **유지** — 적절 |
| Sticky 헤더 | transform 200ms | **유지** — 적절 |

| 페이지 전환 | 없음 | **현 상태 유지** — Next.js App Router 기본 동작 충분 |

## 8. 종합 판정

### v1.0 UX 출시 준비도: **조건부 출시 가능**

디자인 시스템의 기반은 충분히 성숙했다. 색상 토큰, 컴포넌트 라이브러리, 반응형 레이아웃이 체계적으로 구축되어 있으며, 금융 서비스에 적합한 절제된 시각 언어를 유지하고 있다.

### 출시 전 필수 수정 (Blockers)

1. **callbackUrl 리다이렉트** (P0) — 사용자 Flow의 기본 기대를 충족시키지 못하는 치명적 UX 결함
2. **/portfolio 데드 링크 제거** (P1) — 내비게이션의 신뢰도를 즉시 훼손
3. **종목 미존재 시 notFound()** (P1) — 빈/깨진 페이지 노출은 서비스 품질 의심 유발
4. **MAX_SLOTS 문구 수정** (P1) — 작은 수정이지만 "약속과 다른 동작"은 신뢰 저하

### 출시 후 우선 개선 (v1.1)

1. 빈 상태 디자인 통일 (24곳)
2. 에러/404 페이지 시각 개선
3. 한국어 톤 통일
4. Desktop/Mobile 라벨 통일
5. loading.tsx 누락 페이지 보완

### UX 관점 핵심 메시지

> 현재 StockView는 "잘 만들어진 도구"이지만 아직 "좋은 경험"에는 도달하지 못했다. 도구에서 경험으로 전환하려면, 사용자가 "아무것도 없을 때", "잘못되었을 때", "기다려야 할 때" 마주하는 순간들의 품질을 높여야 한다. 이 순간들이 바로 사용자가 서비스의 성격을 판단하는 "진실의 순간(Moment of Truth)"이다.
