## B. 신규 발견: 있음 (경미, 4건)

### B-1. `--fg-*` 네이밍과 통합 디자인 문서 불일치 (심각도: 낮음)

- **현상**: D2 결정으로 `--text-primary` → `--fg-primary`로 변경했으나, 통합 디자인 문서(unified-design-system.md)에는 여전히 `--text-primary`로 기술되어 있을 가능성
- **영향**: 구현 시 문서 참조 혼란
- **권장**: Phase 0 시작 전 통합 디자인 문서의 텍스트 변수명을 `--fg-*`로 업데이트하거나, 구현 계획 v2가 최종 기준(source of truth)임을 명시

### B-2. 뱃지 5종 vs 통합 문서 8종 차이 (심각도: 낮음)

- **현상**: Phase 1-1은 "뱃지 5종 (Market KR/US, Sector, News, ETF)"으로 기술. 통합 디자인 문서는 8종(뉴스를 경제/기업/증시/해외로 세분화)
- **영향**: 뉴스 카테고리 뱃지가 단일 색상으로 통합되어 디자인 의도와 다를 수 있음
- **권장**: Phase 3-5에서 뉴스 카테고리별 색상 분화를 적용하거나, 5종으로 간소화한 근거 명시

### B-3. `.mesh-gradient` 빈 상태 배경 미처리 (심각도: 낮음)

- **현상**: R2-#10에서 지적된 `.mesh-gradient`(빈 상태 배경용)가 계획에 없음. 히어로 섹션은 대시보드로 교체되므로 `.hero-gradient`는 불필요하나, 빈 상태(관심종목 없음, 검색 결과 없음 등)의 `.mesh-gradient`는 여전히 유용
- **영향**: 빈 상태 화면의 시각적 완성도 미흡
- **권장**: Phase 6 폴리시에 `.mesh-gradient` 추가하거나, 명시적으로 제외 결정

### B-4. Phase 4-0 "Prisma 스키마 변경 불필요" 주장 검증 (심각도: 낮음)

- **현상**: Phase 4-0에서 "Prisma 스키마 변경 불필요. 기존 Stock/Sector 모델로 쿼리 가능"이라고 명시
- **검증**: Stock 모델에 `sectorId`(FK), Sector 모델에 `id/name` 존재. `marketCap`은 Stock 모델 또는 DailyPrice에 있어야 함. `changePercent`도 마찬가지
- **결론**: `getSectorHeatmapData`의 `SUM(marketCap)`, `AVG(changePercent)` 집계가 가능하려면 해당 컬럼이 Stock 또는 관련 테이블에 존재해야 함. Prisma 스키마를 확인하여 `marketCap`, `changePercent` 컬럼 존재 여부를 검증할 필요 있음. 존재하면 문제없고, 없으면 스키마 변경이 필요하여 "변경하지 않는 것" 원칙과 충돌
