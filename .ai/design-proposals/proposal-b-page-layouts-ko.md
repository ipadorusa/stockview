# 제안서 B — 페이지 레이아웃 재설계

> **디자이너**: UX 디자이너 B — 페이지 레이아웃 설계자
> **날짜**: 2026-03-29
> **목표**: StockView를 "흰 배경 + 텍스트 리스트"에서 TradingView 스타일 금융 대시보드로 변환. 풍부한 데이터 시각화, 강한 시각적 계층, 최적화된 화면 활용.

---

## 현재 상태 분석

### 발견된 문제점

1. **평면적 레이아웃**: `PageContainer`가 단일 `max-w-screen-xl` 칼럼. 모든 페이지가 동일한 간격(`mb-8`)의 수직 스택. 시각적 무게 차별화 없음.
2. **대시보드 느낌 부재**: 홈페이지가 히어로 → 지수 카드 → 환율 → 퀵링크 → 인기 종목 + 뉴스. 모든 섹션이 동일한 무게, 초점 없음.
3. **미활용 화면**: 데스크톱(1280px)에서 종목 상세 페이지가 우측 30-40% 뷰포트 낭비 — 사이드바 없음, 맥락 데이터 패널 없음.
4. **최소한의 데이터 밀도**: 시장 페이지가 지수 카드 4개 + 단순 상승/하락 리스트. 히트맵, 시장 너비 지표, 섹터 시각화 없음.
5. **모바일 = 좁은 데스크톱**: 하단 탭바 있지만 콘텐츠가 단순히 리플로우된 데스크톱 칼럼. 모바일 네이티브 패턴(캐러셀, 스와이프 카드, 제스처 컨트롤) 없음.
6. **실시간 느낌 부재**: 정적 ISR 페이지에 시장 활동 시각적 표시 없음 — 티커 테이프, 펄스 애니메이션, 깜빡이는 가격 없음.

## 1. 홈페이지 — 대시보드 레이아웃

### 디자인 철학
홈페이지는 블룸버그 터미널이나 TradingView 대시보드를 여는 느낌이어야 함 — 즉각적이고, 정보 밀도 높고, 시장 데이터로 살아있는 느낌. 스크롤 없이 보이는 영역의 모든 픽셀이 시장 상태를 전달해야 함.

### 컴포넌트 상세

#### 1.1 티커 테이프 바
- **위치**: 헤더 바로 아래, 전체 너비, `height: 36px`
- **배경**: `bg-muted/50` (라이트), `bg-card` (다크) — 헤더와 미묘한 대비
- **콘텐츠**: KOSPI, KOSDAQ, S&P 500, NASDAQ, USD/KRW — CSS 애니메이션으로 연속 수평 스크롤 (`@keyframes ticker { from { transform: translateX(0) } to { transform: translateX(-50%) } }`)
- **속도**: 40px/초, 호버 시 일시 정지
- **타이포**: `font-mono text-xs tabular-nums`
- **색상 코딩**: 한국 컨벤션에 따라 상승은 빨강/하락은 파랑, 각 값 앞에 색상 점 표시(`●`)
- **인터랙션**: 항목 클릭 → 해당 상세 페이지로 이동
- **다크 모드**: 동일 레이아웃, 다크 주식 컬러 변형 사용 (`--color-stock-up: #fc8181`, `--color-stock-down: #63b3ed`)
- **모바일**: 표시, 약간 작게 (32px 높이, 작은 텍스트)

#### 1.2 지수 위젯 카드 (재설계)
- **사이즈**: 데스크톱에서 동일 1/4 너비, 태블릿에서 1/2, 모바일에서 수평 캐러셀
- **높이**: 140px 고정
- **구조**:
  ```
  ┌──────────────────────────┐
  │ KOSPI              🇰🇷   │  ← 이름 + 국기/시장 뱃지
  │                          │
  │ 2,645.32                 │  ← 큰 모노 폰트, 28px
  │ ▲ +21.34 (+0.81%)       │  ← 배경 필이 있는 색상 변동값
  │                          │
  │ ┌──────────────────────┐ │  ← 미니 영역 차트 (lightweight-charts)
  │ │  ╱╲    ╱╲  ╱╲       │ │     60px 높이, 최근 30개 데이터
  │ │ ╱  ╲╱╲╱  ╲╱  ╲╱╲╱╲  │ │     그래디언트 채움: stock-up/stock-down
  │ └──────────────────────┘ │
  └──────────────────────────┘
  ```
- **배경**: 방향 기반 미묘한 그래디언트 — 상승 시 `from-stock-up-bg/30 to-transparent`, 하락 시 `from-stock-down-bg/30`
- **테두리**: `border` + 좌측 엣지에 `border-stock-up/20` 또는 `border-stock-down/20` 액센트 (3px solid)
- **호버**: `shadow-md` + 약간의 스케일 `scale-[1.01]`
- **다크 모드**: 더 깊은 그래디언트 틴트, 더 밝은 차트 라인
- **애니메이션**: 로드 시 숫자가 0에서 실제 값까지 600ms 카운트업 (스프링 이징). 변동값은 200ms 딜레이로 페이드인.

#### 1.3 시장 심리 게이지
- **새 컴포넌트**: 공포 ↔ 탐욕을 보여주는 원형/호 게이지
- **계산**: 가용 데이터 기반 — 상승/하락 종목 비율, 지수 변동 방향, 거래량 추세
- **시각**: 반원 호, 파랑(공포)에서 회색(중립)을 거쳐 빨강(탐욕)까지 그래디언트
- **바늘**: CSS `transform: rotate()`로 현재 위치까지 애니메이션
- **레이블**: "공포" / "탐욕" + 숫자 값 (0-100)
- **사이즈**: ~180px 너비, 우측 사이드바 패널에 배치
- **업데이트**: ISR로 갱신 (15분)

#### 1.4 인기 종목 섹션
- **대체**: 현재 `PopularStocksTabs` — 같은 데이터, 더 나은 표현
- **탭**: `KR | US` 밑줄 인디케이터 (애니메이션 슬라이드 `transition: left 200ms, width 200ms`)
- **각 행** (`StockRow` 재설계):
  ```
  ┌─────────────────────────────────────────────┐
  │ 1  삼성전자      72,400원  ▲ +2.1%  ████░░ │
  │    005930        vol 12.3조            │
  └─────────────────────────────────────────────┘
  ```
  - 순위 번호: 굵게, 상위 3개 `text-primary`
  - 거래량 바: 상대 거래량을 보여주는 수평 바 차트 (최대 너비 = 최고 거래량 종목)
  - 거래량 바 색상: 주식 방향에 맞춤 (빨강/파랑, 30% 불투명도)
  - 호버: 행 배경 `accent/50`으로 변경, 인라인 스파크라인 프리뷰 표시
- **최대 항목**: 탭당 10개 (현재와 동일)

#### 1.5 뉴스 피드 카드
- **레이아웃**: 수직 스택, 4개 카드
- **카드 디자인**:
  ```
  ┌──────────────────────────────────────┐
  │ [카테고리 뱃지]           2시간 전   │
  │                                      │
  │ 삼성전자, 신규 반도체 공장 투자...     │
  │                                      │
  │ 관련: 삼성전자(005930) SK하이닉스      │
  │ 출처: 한국경제                        │
  └──────────────────────────────────────┘
  ```
- **카테고리 뱃지**: 색상 필 — `경제` (파랑), `기업` (초록), `증시` (앰버), `해외` (보라)
- **관련 종목**: 클릭 가능한 티커 링크
- **호버**: 카드 `shadow-md`로 떠오르고, 제목 색상 `primary`로 변경

#### 1.6 다크 vs 라이트 모드

| 요소 | 라이트 | 다크 |
|------|--------|------|
| 티커 테이프 bg | `oklch(0.97 0 0)` (뮤트) | `oklch(0.205 0 0)` (카드) |
| 지수 카드 bg | 흰색 + 방향 그래디언트 | `oklch(0.205 0 0)` + 더 깊은 그래디언트 |
| 차트 라인 (상승) | `#e53e3e` | `#fc8181` |
| 차트 채움 (상승) | `#e53e3e` 15% 불투명도 | `#fc8181` 10% 불투명도 |
| 차트 라인 (하락) | `#3182ce` | `#63b3ed` |
| 섹션 구분선 | `oklch(0.922 0 0)` 테두리 | `oklch(1 0 0 / 10%)` 테두리 |
| 종목 행 호버 | `accent/50` | `accent/50` (더 어두운 액센트) |
| 심리 게이지 호 | 솔리드 컬러 | 동일 색조, 약간 뮤트 |

## 2. 시장 개요 — 데이터 밀도 높은 대시보드

### 컴포넌트 상세

#### 2.1 섹터 트리맵 (히트맵)
- **새 컴포넌트**: 섹터 성과를 보여주는 트리맵 스타일 히트맵
- **사이즈**: 콘텐츠 영역 전체 너비, ~300px 높이
- **셀 크기**: 섹터의 시가총액 비중에 비례
- **색상 스케일**:
  - 강한 상승 (>2%): `bg-red-500` (한국 컨벤션)
  - 상승 (0-2%): `bg-red-300`
  - 보합: `bg-gray-400`
  - 하락 (0 ~ -2%): `bg-blue-300`
  - 강한 하락 (<-2%): `bg-blue-500`
- **셀 콘텐츠**: 섹터명 + 변동률
- **호버**: 상세 정보 툴팁 (종목 수, 시가총액, 대장주)
- **클릭**: `/sectors/[name]`으로 이동
- **구현**: 시가총액 비율로 계산된 `grid-template-columns` CSS Grid. 무거운 차트 라이브러리 불필요.
- **다크 모드**: 동일 색상 스케일, 다크 배경 대비를 위해 약간 더 채도 높게
- **모바일**: 트리맵 대신 색상 바 리스트(수평 바 차트 형식)로 단순화

#### 2.2 시장 너비 바
- **새 컴포넌트**: 단일 수평 스택드 바
- **너비**: 콘텐츠 영역 100%
- **높이**: 40px
- **세그먼트**: 상승 (빨강) | 보합 (회색) | 하락 (파랑)
- **레이블**: 양쪽 끝에 숫자, 중앙에 퍼센트
- **바 아래**: 텍스트 라인 — "상한가 X · 하한가 Y · 보합 Z"
- **애니메이션**: 바 세그먼트가 로드 시 0에서 실제 너비까지 애니메이션 (400ms ease-out)

#### 2.3 모멘텀 바 (상승/하락)
- **대체**: 현재 단순 `StockRow` 리스트
- **레이아웃**: 나란히 2칼럼
- **각 항목**:
  ```
  삼성전자 (005930)  ██████████████░░░░░░  +8.23%
  ```
  - 수평 바가 변동률에 비례하여 채움 (최대 바 = 최대 변동 종목)
  - 바 색상: 상승은 빨강 채움, 하락은 파랑 채움
  - 호버: 전체 상세 표시 (가격, 거래량, 시가총액)
  - 클릭: 종목 상세로 이동
- **최대 항목**: 칼럼당 10개 (기존 5개에서 증가)

#### 2.4 정렬 가능 종목 테이블
- **새 컴포넌트**: 정렬, 필터, 인라인 스파크라인이 있는 데이터 밀도 높은 테이블
- **칼럼**: 종목명 | 현재가 | 등락 | 등락률 | 거래량 | 거래대금 | 시가총액 | 7일 차트
- **기능**:
  - 칼럼 헤더 클릭으로 정렬 (오름/내림 토글), 화살표 표시
  - 마지막 칼럼 스파크라인: 7일 미니 차트 (40px × 20px, lightweight-charts)
  - 가독성을 위한 줄무늬 행 배경
  - 스크롤 시 고정 헤더
  - 페이지네이션: 페이지당 20개 + "더 보기" 버튼
- **행 호버**: 하이라이트 + 우측에 "상세 보기" 고스트 버튼 표시
- **타이포**: 모든 숫자 `font-mono tabular-nums`로 정렬
- **모바일**: 첫 칼럼(종목명) 고정, 수평 스크롤

#### 2.5 KR/US 시장 전환
- **탭 디자인**: 시장 콘텐츠 영역 상단 밑줄 탭 (칩 아님)
- **동작**: 탭 전환 시 아래 모든 콘텐츠 업데이트 — 트리맵, 너비 바, 변동 종목, 테이블
- **애니메이션**: 콘텐츠 크로스페이드 (opacity 0→1, 150ms)
- **URL**: URL 쿼리 파라미터로 탭 상태 반영 `?market=kr` / `?market=us`
- **키보드**: 방향키로 탭 간 이동

### 컴포넌트 상세

#### 3.1 종목 헤더 (재설계)
- **레이아웃**: 차트 위 전체 너비
- **좌측 섹션**:
  - 종목명: `text-2xl font-bold`

  - 변동: 화살표 + 절대 변동 + 퍼센트가 포함된 색상 필
  - 타임스탬프: `text-xs text-muted-foreground`
  - OHLV 행: `text-sm font-mono` — 시가, 고가, 저가, 거래량을 컴팩트한 수평 라인으로
- **우측 섹션**:
  - 관심종목 버튼: 채우기 애니메이션이 있는 하트/별 토글
  - 비교 버튼: 비교 바에 추가
  - 공유 버튼: 링크 복사
- **프리/포스트 마켓**: 가능한 경우 메인 가격 아래 잔잔한 라인으로 표시 — "프리마켓 $142.50 (+0.3%)" 더 작은 텍스트

#### 3.2 차트 영역 (히어로 요소)
- **사이즈**: 뷰포트 높이의 60%, 최소 400px, 최대 600px
- **라이브러리**: lightweight-charts (이미 사용 중)
- **차트 유형** (차트 아래 툴바):
  - 캔들스틱 (KR 기본)
  - 라인
  - 영역 (홈페이지 미니 차트 기본)
- **기간**: `1일 | 1주 | 1개월 | 3개월 | 1년 | 3년 | 전체` — 필 스타일 토글 버튼
- **지표 버튼**: 체크박스가 있는 팝오버 — MA(5,20,60,120), 볼린저, RSI, MACD, 거래량
- **크로스헤어**: 호버 시 OHLCV 데이터 툴팁 표시
- **거래량**: 메인 차트 아래 서브 차트, 차트 높이의 20%
- **반응형**:
  - 데스크톱: 60vh, 사이드바와 나란히
  - 태블릿: 전체 너비, 50vh
  - 모바일: 풀 블리드 (엣지까지 음수 마진), 50vh, 제스처 컨트롤 (핀치 줌, 스와이프 팬)

- **너비**: 데스크톱에서 320px 고정
- **위치**: 고정(sticky), 독립적으로 스크롤
- **섹션** (각각 카드):
  1. **핵심 통계**: 레이블-값 쌍 그리드, 2칼럼
     - 값에 `font-mono tabular-nums`
     - 뮤트 레이블, 강한 값
  2. **52주 레인지**: 수평 슬라이더 시각화
     - 파란 점 = 현재 가격 위치
     - 저가/고가 양쪽 끝 레이블
  3. **관련 종목**: 같은 섹터 3-5개 종목
     - 이름 + 변동률의 미니 행
     - 클릭으로 이동
  4. **애널리스트 컨센서스** (향후): 데이터 가능 시 매수/보유/매도 바
- **다크 모드**: 카드 `bg-card` (`oklch(0.205 0 0)`), 테두리 `oklch(1 0 0 / 10%)`
- **태블릿/모바일**: 수평 스크롤 가능 카드 스트립 또는 접을 수 있는 아코디언으로 변환

#### 3.4 콘텐츠 탭
- **탭 바**: 스크롤 시 차트 아래에 고정 (헤더 아래 아님)
- **탭**:
  1. **개요**: 기업 설명, 핵심 펀더멘털 카드, 섹터 정보
  2. **재무**: 매출/순이익 차트, 재무제표 테이블
  3. **뉴스**: 관련 뉴스 카드 (NewsCard 컴포넌트 재사용)
  4. **이벤트**: 배당 + 실적 + 공시 통합 (현재 EventsTabWrapper)
  5. **AI 리포트**: AI 리포트 가능 시 링크, 탭에 개수 뱃지
- **애니메이션**: 탭 콘텐츠 페이드인 (`opacity 0→1`, 150ms)
- **지연 로딩**: 비활성 탭은 첫 방문 시 Suspense로 로드

#### 3.5 모바일 종목 상세
- **순서**: 헤더 (컴팩트) → 차트 (풀 블리드, 50vh) → 퀵 통계 (수평 스크롤 카드) → 탭
- **차트**: 16px 패딩 제거로 풀 블리드
- **퀵 통계**: 탭 위 수평 스크롤 가능 통계 칩 행
- **탭**: 밑줄 탭이 아닌 세그먼트 컨트롤 스타일
- **섹션**: 각 탭 콘텐츠에 셰브론 토글로 접을 수 있는 서브 섹션

## 4. 네비게이션 재설계

### 디자인 철학
네비게이션은 필요할 때까지 보이지 않아야. 시장 상태는 항상 보이되 산만하지 않은 앰비언트. 데스크톱은 파워유저 기능, 모바일은 엄지 친화적 영역.

### 4.1 헤더 재설계 (데스크톱)

```
│                                                             │
│         ┌─ 시장 상태 ────────┐                               │
│         │ 🟢 장중 15:22      │ (또는 🔴 장마감, 🟡 프리마켓) │
│         └──────────────────┘                                │
├─────────────────────────────────────────────────────────────┤
│  시장 · ETF · 섹터 · 배당 · 실적        (2차 네비게이션)     │

```

#### 헤더 사양
- **높이**: 주 네비: `h-14` (56px) — 변경 없음
- **서브 네비**: `h-10` (40px) — 변경 없음, 디자인만 개선
- **시장 상태 뱃지**:
  - 위치: 네비 링크와 검색 바 사이
  - 상태:
    - `🟢 장중 HH:MM` — 초록 점, 펄스 애니메이션 (`animate-pulse`)
    - `🔴 장마감` — 빨간 점, 정적
    - `🟡 프리마켓` / `🟡 애프터마켓` — 노란 점
    - `⚪ 휴장` — 회색 점, "주말" 또는 "공휴일" 레이블
  - 폰트: `text-xs font-medium`
  - 로직: KR 시장 시간 (09:00-15:30 KST 평일), US 시장은 KST 조정
  - 클릭: 한국/미국 시장 상태 모두 표시하도록 확장
- **검색 바**: 256px에서 320px로 확장, 키보드 단축키 힌트 `⌘K` 추가
- **배경**: `bg-background` + `border-b`, 블러 효과 없음 (성능)

#### 서브 네비게이션 재설계
- **표시**: 카테고리 활성 시에만 표시 (현재 동작 — 유지)
- **스타일**: 더 컴팩트, 일반 텍스트 대신 필 스타일 링크
- **활성 표시**: 하단 테두리 `border-b-2 border-primary` + 슬라이드 애니메이션
- **스크롤**: 태블릿에서 링크 오버플로 시 수평 스크롤

### 4.2 모바일 하단 탭 바 (재설계)

```
┌──────────────────────────────────────────────────┐
│                                                  │
│   🏠       🔍       📊       ⭐       👤        │
│   홈       검색      시장     관심     MY         │
│                                                  │
└──────────────────────────────────────────────────┘
```

- **높이**: `h-14` (56px) — 변경 없음
- **배경**: `bg-background/95 backdrop-blur-sm` — 프로스티드 글래스 효과
- **활성 상태**: 아이콘 솔리드 채움 + 텍스트 `text-primary` + 아이콘 위 점 표시
- **안전 영역**: iOS 안전 영역 `pb-safe` (env(safe-area-inset-bottom)) 고려
- **햅틱**: 탭 시 `navigator.vibrate(10)` (지원 시)
- **뱃지**: 관심종목 탭에 알림 시 빨간 점

### 4.3 브레드크럼 시스템

```
주식 > 시장 개요 > 섹터별 종목 > IT/반도체
```

- **위치**: 페이지 콘텐츠 상단, 헤더 아래, 페이지 제목 위
- **스타일**: `text-xs text-muted-foreground`, 셰브론 구분자
- **링크**: 마지막 항목 외 모두 클릭 가능
- **스키마**: SEO용 JSON-LD BreadcrumbList (이미 부분 구현)
- **모바일**: 3단계 초과 시 중간 항목 "..."으로 줄여 마지막 2개만 표시

### 4.4 사이드바 네비게이션 (파워유저 옵션 — 향후)
- **컨셉**: 좌측 끝 접을 수 있는 선택적 사이드바
- **너비**: 확장 240px, 접힘 64px (아이콘만)
- **토글**: 햄버거 버튼 또는 키보드 단축키 `[`
- **콘텐츠**: 확장 가능 섹션이 있는 전체 네비게이션 트리
- **지속성**: 상태 localStorage에 저장
- **우선순위**: 2단계 — 초기 재설계에 포함하지 않음. 현재 상단 네비로 충분.

## 5. 모바일 반응형 전략

### 디자인 철학
모바일은 더 작은 데스크톱이 아님. 각 레이아웃은 터치 네이티브 패턴에 맞게 적응. 엄지 영역 최적화: 주요 액션은 하단에서 44-88px 이내. 콘텐츠 계층: 차트/숫자 먼저, 맥락 두 번째.

#### 지수 카드 캐러셀
- **컴포넌트**: 수평 스크롤 스냅 컨테이너
- **CSS**: `scroll-snap-type: x mandatory`, 각 카드 `scroll-snap-align: center`
- **카드 너비**: `calc(100vw - 64px)` — 다음 카드 엿보기(양쪽 16px 보임)와 함께 전체 너비
- **점**: 캐러셀 아래 페이지네이션 점, 활성 점 = `bg-primary`
- **스와이프**: 자연 스크롤 동작, JS 불필요
- **자동 전환**: 자동 스크롤 없음 (혼란 방지를 위해 사용자 주도만)

#### 인기 종목 리스트
- **축약**: 상위 5개만 표시 (10개 아님), "더보기" 링크
- **터치 타겟**: 각 행 최소 `h-12` (48px) 편안한 탭핑
- **스와이프 액션**: 종목 행 우측 스와이프로 관심종목 추가 (선택, 2단계)

- **풀 블리드**: `margin: 0 -16px; width: calc(100% + 32px)`
- **제스처**:

  - 한 손가락 드래그: 좌우 팬
  - 길게 누르기: 데이터 툴팁과 함께 크로스헤어 표시
  - 더블 탭: 줌 리셋
- **기간 선택기**: 차트 아래 수평 스크롤 가능 필 버튼
- **방향**: 가로 모드 전환 시 차트 ~90vh로 확장

#### 고정 하단 CTA
- **위치**: 하단 탭 바 위에 고정
- **콘텐츠**: 로그인 사용자용 "★ 관심종목 추가" 버튼
- **표시**: 차트를 지나 스크롤한 후 페이드인

- **페이지**: 홈, 시장, 관심종목

#### 터치 타겟
- 모든 인터랙티브 요소: 최소 44px × 44px (Apple HIG)

- **옵션**: "표 보기" 토글로 수평 스크롤 테이블 전환
- **고정 칼럼**: 수평 스크롤 시 첫 칼럼 (종목명) 고정

## 7. 애니메이션 & 마이크로 인터랙션 사양

### 페이지 전환
- **라우트 변경**: 콘텐츠 페이드 아웃 (100ms) → 페이드 인 (200ms)
- **구현**: CSS `@starting-style` 또는 `View Transitions API` (점진적 향상)

### 데이터 업데이트
- **가격 변동 플래시**: 가격 업데이트 시 배경이 잠시 stock-up-bg 또는 stock-down-bg로 깜빡 (300ms 페이드)
- **카운터 애니메이션**: 큰 숫자가 새 값으로 카운트 업/다운 (400ms, ease-out)
- **스파크라인 그리기**: 라인이 초기 로드 시 좌에서 우로 애니메이션 (600ms)

### 인터랙티브 요소
- **버튼 누름**: `active:scale-[0.97]` (이미 부분 구현)
- **카드 호버**: `hover:shadow-md hover:-translate-y-0.5 transition-all duration-200`
- **탭 전환**: 콘텐츠 `transition-opacity duration-150`
- **스켈레톤 펄스**: 표준 `animate-pulse`

### 성능 참고
- 모든 애니메이션은 `transform`과 `opacity`만 사용 (GPU 가속)
- 자주 애니메이션되는 요소에 `will-change: transform`
- `prefers-reduced-motion`: 필수 상태 변경 외 모든 애니메이션 비활성
- 레이아웃용 JS 주도 애니메이션 없음 — CSS만

## 8. 구현 우선순위

### 1단계 (높은 임팩트, 낮은 노력)
1. 티커 테이프 컴포넌트
2. 그래디언트 배경이 있는 지수 카드 재설계
3. 헤더에 시장 상태 뱃지
4. 종목 상세 사이드바 레이아웃 (CSS Grid 변경)
5. 모바일 차트 풀 블리드

### 2단계 (높은 임팩트, 중간 노력)
1. 섹터 트리맵/히트맵
2. 시장 너비 바
3. 모멘텀 바 (상승/하락)
4. 정렬 가능 종목 테이블
5. 종목 헤더 재설계

### 3단계 (중간 임팩트, 높은 노력)
1. 시장 심리 게이지
2. 모바일 지수 카드 캐러셀
3. 당겨서 새로고침
4. 탭 애니메이션과 전환
5. 카운터/가격 애니메이션

### 권장하지 않음
- 사이드바 네비게이션 (현재 상단 네비로 이 정보 구조에 충분)
- 자동 전환 캐러셀 (사용자 선호 충돌)
- WebSocket 실시간 업데이트 (ISR이 데이터 갱신 주기에 적합)
