### #1. [P0] Pretendard 폰트 Self-host + 스크립트 로딩 최적화
**제안자**: FE#1 (성능/아키텍처)
**동의**: SEO (Core Web Vitals 영향)

**문제**: 외부 CDN(jsdelivr)에서 렌더 블로킹 CSS로 Pretendard를 로딩. 이중 워터폴(CSS → woff2) 발생. GTM 인라인 스크립트도 렌더 경로 차단.

**솔루션**:
- `next/font/local`로 Pretendard Variable woff2 self-host
- GTM consent 스크립트를 `next/script strategy="afterInteractive"`로 이동
- `preconnect` 힌트 추가

**기대 효과**: LCP 200~500ms 개선, FCP 100~200ms 개선
**공수**: 2~3시간 (S)
**비용**: 무료

### #2. [P0] 모바일 하단 탭 5개 확장 — 검색 + 관심종목 직접 접근
**제안자**: UX기획자
**동의**: UX모바일, PM

**문제**: 현재 4탭(홈/시장/뉴스/MY)에 검색과 관심종목이 없음. 초보 투자자의 핵심 루프(검색→상세→관심종목)의 시작점과 종착점이 모두 1탭 접근 불가.

**솔루션**: `홈 | 검색 | 시장 | 관심 | MY` 5탭 구성
- 검색 탭: 풀스크린 오버레이 (기존 SearchCommand 재활용)
- 관심 탭: /watchlist 직접 연결

**기대 효과**: 핵심 유저 플로우 탭 수 3→1로 단축
**공수**: 0.5일 (S)
**비용**: 무료

### #3. [P0] 종목 상세 페이지 — 용어 툴팁 + 정보 계층화
**제안자**: PM (학습형 인터랙션), UX기획자 (프로그레시브 디스클로저), UX비주얼 (데이터 시각화)

**문제**: PER, PBR, ROE 등 전문 지표가 설명 없이 숫자만 나열. 초보 투자자가 의미 파악 불가. 탭이 과다하고 정보 밀도가 높아 스크롤 과부하.

**솔루션** (3명의 제안 통합):
- 각 지표에 `(?)` 인라인 툴팁: 정의 + 업종평균 비교 + 신호등 컬러
- 52주 Range Bar 시각화 (CSS만으로 구현)
- 탭 재배치: `차트 | 요약 | 뉴스 | 더보기`
- 비활성 탭은 클라이언트 on-demand fetch (FE#1 제안과 연계)
- 종목 요약 카드를 탭 상단에 추가 (현재가 + 한줄요약 + CTA)

**기대 효과**: 종목 상세 체류시간 +30%, TTFB 300~800ms 개선
**공수**: 3~4일 (M)
**비용**: 무료 (기존 shadcn Tooltip + 정적 텍스트)

### #4. [P0] 신규 사용자 온보딩 플로우
**제안자**: PM, UX기획자
**동의**: UX비주얼 (빈 상태 디자인)

**문제**: 회원가입 후 온보딩 없음. 39페이지 중 어디서 시작할지 모름. 빈 관심종목/포트폴리오의 차가운 첫인상.

**솔루션**:
- 회원가입 후 3단계 바텀시트: 관심 시장 선택 → 관심 섹터 → 인기종목 3개 관심종목 추가
- localStorage 기반 `hasCompletedOnboarding` (DB 변경 없음)
- 비로그인 사용자: 홈 상단 "초보 투자자 가이드" CTA 배너

**기대 효과**: 온보딩 완료율 60%+, 가입 후 7일 재방문율 30%+
**공수**: 2~3일 (M)
**비용**: 무료

### #5. [P0] 브랜드 컬러 정체성 확립 (그린) + 다크모드 접근성 수정
**제안자**: UX비주얼

**문제**:
- CLAUDE.md에 "Brand color: green"이지만 실제 `--primary`는 보라/남색 계열(`oklch(0.45 0.15 260)`)
- 다크모드에서 `stock-up-bg` + `text-stock-up` 조합이 WCAG AA 미달 (대비비 3.2:1)

**솔루션**:
- `globals.css`의 `--primary` 계열을 그린으로 전환: `oklch(0.45 0.16 145)`
- 다크모드 `--color-stock-up-bg`를 `#3d1f1f`로 조정 (대비비 4.5:1+)
- 등락 표시에 `▲`/`▼` 기호 일관 적용 (형태적 구분 추가)

**기대 효과**: 브랜드 인지도 확립, 접근성 AA 준수
**공수**: 1~2시간 (S) — CSS 변수 수정만
**비용**: 무료

### #6. [P1] PWA 전환 + 가격 알림 (Web Push)
**제안자**: PM (PWA + Push), FE#2 (PWA + 오프라인)

**문제**: PWA 미지원으로 홈 화면 설치 불가. 관심종목 가격 변동 알림 없어서 재방문 동기 부재.

**솔루션**:
- Phase 1: `manifest.json` + Service Worker (serwist/next-pwa) + A2HS 프롬프트
  - 정적 자산 `CacheFirst`, API `StaleWhileRevalidate`
- Phase 2: `web-push` npm + `PriceAlert` DB 테이블
  - 크론 시세 수집 시 알림 조건 체크 → push 발송
  - 워치리스트 종목별 벨 아이콘

**기대 효과**: PWA 설치 500+, 7일 재방문율 +40%, 알림 오픈율 40%+
**공수**: Phase 1: 3일, Phase 2: 4일
**비용**: 무료 (web-push VAPID 자체 생성)

### #7. [P1] 소셜 공유 기능 + View Transitions
**제안자**: PM (소셜 공유), UX모바일 (공유 + View Transitions)

**문제**: 공유 기능 0건. 바이럴 루프 완전 차단. 페이지 전환이 "깜빡임"으로 웹 느낌.

**솔루션**:
- `navigator.share()` + 클립보드 fallback (종목, 리포트, 게시판)
- 차트 캡처: `<canvas>.toBlob()` → `navigator.share({ files })`
- View Transitions API로 종목 카드→상세 공유 요소 전환
- 기존 OG 이미지 인프라 활용

**기대 효과**: 일일 공유 100+, 레퍼럴 트래픽 15%+
**공수**: 공유 1일 + View Transitions 2일
**비용**: 무료

### #8. [P1] 네이버/다음 검색엔진 최적화 + 종목 페이지 SEO 강화
**제안자**: SEO

**문제**:
- 네이버 서치어드바이저 미등록 (한국 검색 55~60% 놓침)
- 4,500 종목 페이지 meta description이 동일 템플릿 (thin content)
- 내부 링크 구조 체계 부재

**솔루션**:
- Quick Win (30분): 네이버 서치어드바이저 등록, `max-image-preview: "large"` 추가
- meta description에 PER/시총/배당 동적 삽입
- 종목별 자동 FAQ 구조화 데이터
- "같은 섹터 종목" 내부 링크 섹션 추가
- RSS 피드 엔드포인트 (`/feed.xml`) 생성

**기대 효과**: 월 +13,000~45,000 클릭 (네이버 + Google 합산)
**공수**: Quick Win 30분, 전체 2주
**비용**: 무료

### #9. [P1] SSE 기반 장중 준실시간 시세 + 스크리너 UI 완성
**제안자**: FE#2

**문제**: 시세가 크론 기반으로 장중 크게 지연. 스크리너 API는 있으나 프론트엔드 미구현.

**솔루션**:
- `/api/stocks/[ticker]/live` SSE 엔드포인트 (10초 간격 폴링, 55초 자동 종료)
- `useSSE` 훅 → TanStack Query 캐시 직접 업데이트
- 스크리너 펀더멘털 필터 UI: Range Slider + URL params + 프리셋 버튼
- `document.visibilityState` 감지로 비활성 탭 SSE 해제

**기대 효과**: 장중 사용자 경험 근본 개선, 가치투자 사용자층 확대
**공수**: SSE 4일 + 스크리너 UI 3일
**비용**: 무료

### #10. [P1] AppHeader Server/Client 분리 + generateStaticParams 확대
**제안자**: FE#1

**문제**: AppHeader가 287줄 `"use client"` 모놀리스로 전체 번들 팽창. generateStaticParams가 50개만 사전빌드하여 나머지 4,750개 종목 TTFB 1~3초.

**솔루션**:
- AppHeader → Server Component 쉘 + Client Islands 4개 분리
- SearchCommand `next/dynamic` lazy load
- generateStaticParams take 50→500 확대
- GitHub Actions 캐시 워밍 스크립트

**기대 효과**: JS 번들 30~50KB 감소, TTI 100~300ms 개선, 비인기 종목 TTFB 1~3초→0.5초
**공수**: AppHeader 3~5시간, StaticParams 2시간
**비용**: 무료

## 5. 팀원별 핵심 인사이트

### PM
> "5가지 개선 모두 **이미 존재하는 자산**(OG 이미지, 섹터 데이터, 가이드 콘텐츠, 크론 인프라)을 재활용하는 방향. 초보 투자자 특화 경험(온보딩, 맥락 내 학습, 신호등 지표)에 집중하여 네이버 증권/야후 파이낸스와 차별화."

### FE#1 (성능)
> "Pretendard 폰트와 스크립트 로딩 최적화만으로 LCP 300~700ms 개선 가능. S 복잡도 항목 2개를 같은 PR에서 처리하면 Core Web Vitals에서 즉시 측정 가능한 변화."

### FE#2 (기능)
> "스크리너 펀더멘탈 API가 이미 있는데 UI만 없는 상태가 ROI 최고. SSE는 Vercel 60초 timeout 내에서 창의적으로 구현 가능하며, 장중 '살아있는 시세' 경험의 근본 해결책."

### UX기획자
> "39페이지는 초보자에게 과부하. 핵심은 **검색→상세→관심종목** 루프를 1탭 거리로 단축하는 것. 모바일 하단탭 5개 확장이 가장 ROI 높은 IA 변경."

### UX디자이너#1 (비주얼)
> "브랜드 컬러가 PRD(그린)와 코드(보라)가 불일치하는 것은 즉시 수정 필요. globals.css 변수 4~6개 변경이면 전체 앱에 반영되므로 30분 안에 해결 가능."

### UX디자이너#2 (모바일)
> "Pull-to-Refresh와 스와이프 액션은 '앱 같은 느낌'의 핵심. Web Share API는 구현 난이도 최하이면서 바이럴 효과 최대 — 공유 기능을 가장 먼저 구현할 것을 강력 추천."

### SEO전문가
> "한국 검색 시장의 55~60%가 네이버인데 서치어드바이저 미등록은 트래픽 절반을 포기하는 것. 30분 투자로 네이버 등록 + sitemap 제출만 해도 수개월 내 유의미한 트래픽 유입 시작."

## 6. 핵심 합의사항

1. **Week 1의 Quick Wins는 즉시 실행** — 총 12시간 투자로 성능, 브랜드, UX, SEO 모두 개선
2. **초보 투자자 차별화가 핵심 전략** — 네이버 증권에 없는 "학습형 UX"(툴팁, 온보딩, 가이드 연결)
3. **기존 자산 재활용 원칙** — OG 이미지, 섹터 데이터, 가이드 콘텐츠, 크론 인프라, 스크리너 API 활용
4. **모바일 퍼스트** — 하단 탭 5개, PWA, 공유, Pull-to-Refresh는 모바일 사용자 경험의 근본
5. **네이버 SEO는 Quick Win** — 30분으로 한국 검색 트래픽의 문을 여는 것
