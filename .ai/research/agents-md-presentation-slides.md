## 슬라이드 9: 폴더별 구성 예시

```
┌─────────────────────────────────────────────────┐
│                                                   │
│   project/                                        │
│   ├── AGENTS.md          Root ~120줄 (항상 로딩)   │
│   ├── CLAUDE.md          "@AGENTS.md" 1줄          │
│   │                                               │
│   ├── src/app/                                    │
│   │   ├── AGENTS.md      App Router 규칙 ~60줄     │
│   │   └── api/                                    │
│   │       └── AGENTS.md  API Route 규칙 ~50줄      │
│   │                                               │
│   ├── src/components/                             │
│   │   └── AGENTS.md      컴포넌트 규칙 ~50줄       │
│   │                                               │
│   ├── src/hooks/                                  │
│   │   └── AGENTS.md      RQ + Zustand ~40줄        │
│   │                                               │
│   └── src/lib/                                    │
│       └── AGENTS.md      유틸리티 ~30줄             │
│                                                   │
│                                                   │
│   ┌─────────────────────────────────────┐         │
│   │ Root: 빌드 명령, TS 규칙, 라우팅 테이블 │       │
│   │ app/: App Router, 메타데이터           │       │
│   │ api/: 검증→인증→로직→응답 패턴         │       │
│   │ components/: Server/Client 분리       │        │
│   │ hooks/: RQ=서버상태, Zustand=UI상태    │       │
│   │ lib/: 순수 함수, API 클라이언트         │       │
│   └─────────────────────────────────────┘         │
│                                                   │
│   상시 로딩: Root만 ~120줄                         │
│   최대 동시: ~180줄 (Root + 하위 1개)               │
│                                                   │
└─────────────────────────────────────────────────┘
```

**발표 멘트**:
"Root에는 모든 세션에서 항상 필요한 것만. 빌드 명령, TypeScript 규칙, 서버/클라이언트 구분 원칙. 하위 폴더에는 해당 영역에서만 필요한 규칙. hooks 폴더의 AGENTS.md에는 'React Query는 서버 상태, Zustand는 UI 상태'라는 우리 팀의 핵심 경계가 들어갑니다. 이걸 명시해두면 AI가 매번 Zustand에 API 데이터 넣는 실수를 안 합니다."
