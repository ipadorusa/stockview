## 1. 네비게이션 구조 분석

### 1.1 현황

**데스크톱 AppHeader** (8개 링크):
```
홈 | 시장 | ETF | 스크리너 | AI 리포트 | 뉴스 | 게시판 | 관심종목
```

**모바일 BottomTabBar** (4개 탭):
```
홈 | 시장 | 뉴스 | MY
```

**모바일 Sheet(햄버거)**: AppHeader와 동일 8개 + 로그인/회원가입

**어디에도 없는 페이지** (5개):
- `/dividends` (배당 캘린더)
- `/earnings` (실적 캘린더)
- `/sectors` (섹터별 종목)
- `/compare` (종목 비교)
- `/guide` (투자 가이드)

### 1.2 문제 분석

#### 문제 1: 모바일 핵심 기능 접근성 부재
- BottomTabBar 4개 탭에는 ETF, 스크리너, AI 리포트, 게시판이 **완전히 누락**
- 사용자가 이 기능에 접근하려면: (1) 우측 상단 햄버거 아이콘 인지 → (2) 탭하여 Sheet 열기 → (3) 메뉴 선택
- 일반적인 모바일 앱에서 **햄버거 메뉴 내 항목의 클릭률은 하단 탭 대비 70-80% 낮음**
- 현재 구조에서 스크리너, AI 리포트 같은 차별화 기능이 모바일에서 사실상 사장됨

#### 문제 2: 5개 페이지 발견 불가
- 배당 캘린더, 실적 캘린더는 독립 페이지로 잘 구현되어 있으나 진입 경로가 없음
- 종목 비교(`/compare`)는 매우 유용한 기능이나 URL 직접 입력 외에 접근 불가
- 투자 가이드(`/guide`)는 초보 타겟 서비스의 핵심 콘텐츠인데 노출 안 됨
- `/sectors`는 시장 페이지의 SectorPerformance 내 Dialog에서만 간접 접근 가능

#### 문제 3: 데스크톱 메뉴 8개 — 과잉 + 우선순위 미반영
- 8개 수평 메뉴가 동일한 시각적 가중치로 나열
- "게시판"은 서비스의 핵심 기능이 아니나 네비게이션 1급에 위치
- 반면 "배당", "실적", "가이드" 같은 투자 핵심 콘텐츠는 미노출

### 1.3 개선안

#### 안 A: 메가 메뉴 (Mega Menu) 도입

```
데스크톱 상단바:
[홈] [투자 ▼] [분석 ▼] [뉴스] [더보기 ▼] [관심종목]

투자 드롭다운:           분석 드롭다운:         더보기 드롭다운:
├─ 시장 개요            ├─ 스크리너            ├─ 투자 가이드
├─ ETF                  ├─ AI 리포트           ├─ 게시판
├─ 섹터별 종목          ├─ 종목 비교           ├─ 서비스 소개
├─ 배당 캘린더          └─ (빈칸)              └─ 문의하기
└─ 실적 캘린더
```

**장점**: 모든 39개 페이지에 2클릭 이내 접근 가능. 관련 페이지끼리 그루핑.
**단점**: 드롭다운 구현 복잡도 증가. 호버 시 의도치 않은 열림 가능.

#### 안 B: 2단 네비게이션 (권장)

```
1단: [홈] [시장] [종목분석] [뉴스] [관심종목]  ← 5개 핵심
2단 (시장 선택 시):  시장 개요 | ETF | 섹터 | 배당 캘린더 | 실적 캘린더
2단 (종목분석 선택 시): 스크리너 | AI 리포트 | 종목 비교 | 투자 가이드
```

**장점**: 1단은 5개로 깔끔, 2단에서 세부 기능 노출. 사용자 멘탈 모델과 일치.
**단점**: 2단 서브내비 높이(약 40px) 추가로 콘텐츠 영역 감소.

#### 모바일 BottomTabBar 개선

**현재 (4탭):**
```
홈 | 시장 | 뉴스 | MY
```

**개선안 (5탭):**
```
홈 | 시장 | 분석 | 뉴스 | MY
```

- "분석" 탭: 스크리너/AI 리포트/종목 비교를 묶는 허브 페이지 또는 액션시트
- 시장 탭 내에서 ETF/배당/실적/섹터 접근 가능하도록 시장 페이지 내 퀵 링크 추가

**또는 (5탭 + 내비 허브):**
```
홈 | 투자 | 분석 | 뉴스 | MY
```

- "투자" 탭 → 시장 개요 + ETF + 배당 + 실적 + 섹터로의 진입점
- "분석" 탭 → 스크리너 + AI 리포트 + 종목 비교 + 가이드

## 2. 홈 페이지 정보 구조

### 2.1 현황

현재 홈페이지 구성 (수직 순서):
```
1. h1: "한국/미국 주식 시세"
2. 주요 지수 (KOSPI, KOSDAQ, S&P500, NASDAQ — 2x2 그리드)
3. 환율 (5개 — USD, EUR, JPY, CNY, GBP)
4. 인기 종목 (KR/US 탭, 각 10개) | 최신 뉴스 (4개) — 2열 그리드
5. 광고 배너
```

### 2.2 문제 분석

1. **히어로/CTA 부재**: 첫 방문자에게 "이 서비스가 뭔지" 설명 없이 바로 데이터 나열. 초보 투자자 타겟인데 온보딩 없음.
2. **환율 5개 과다 노출**: 대부분 사용자에게 USD/KRW만 의미 있음. EUR, CNY, GBP는 모바일 공간 낭비.
3. **인기 종목 10개 과다**: 모바일에서 상당한 스크롤 필요. 5개면 충분.
4. **h1 텍스트 평범**: "한국/미국 주식 시세"는 서비스 가치 전달 부족.
5. **빠른 접근 링크 부재**: 스크리너, AI 리포트, 배당 캘린더 등으로의 진입점 없음.

### 2.3 개선안

```
[히어로 영역 — 첫 방문자용, 로그인 후 숨김 가능]
  "초보 투자자를 위한 스마트한 주식 분석"
  [스크리너로 시작하기] [투자 가이드 보기]

[주요 지수 + 환율(USD만) — 한 줄 컴팩트 바]
  KOSPI 2,650.32 ▲0.5% | KOSDAQ 850.12 ▼0.3% | S&P 2,650 | USD/KRW 1,380.5

[퀵 액세스 카드 — 핵심 기능 4개]
  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐
  │ 스크리너  │ │ AI 리포트│ │ 배당캘린더│ │ 종목비교  │
  └──────────┘ └──────────┘ └──────────┘ └──────────┘

[인기 종목 (KR/US 탭, 각 5개)]

[최신 뉴스 (3개)]

[투자 가이드 — CTA 카드]
  "주식 투자, 어디서부터 시작할지 모르겠다면?"
  [가이드 보러가기]
```

**핵심 변경:**
- 히어로 섹션으로 서비스 가치 즉시 전달 (첫 방문자)
- 지수+환율을 컴팩트 바로 압축 (현재 카드 4개 → 인라인 1줄)
- 퀵 액세스 카드로 숨겨진 기능 발견성 해결
- 인기 종목 10→5개로 축소, 뉴스 4→3개로 축소
- 가이드 CTA로 초보자 온보딩 유도

## 3. 종목 상세 페이지 UX

### 3.1 현황

페이지 수직 구조:
```
1. Breadcrumb
2. AdSlot (광고!)  ← 콘텐츠 전에 광고
3. 종목 헤더 (이름 + 티커 + 거래소 + 섹터)
4. 가격 표시 (현재가 + 등락 + 장전/장후)
5. 기업 개요 (접기/펼치기)
6. 탭 (차트 | 시세 | 뉴스 | 공시 | 배당 | 실적)
7. 탭 콘텐츠
```

### 3.2 문제 분석

#### 문제 1: 광고가 콘텐츠를 가로막음
- `stock/[ticker]/page.tsx:175` — AdSlot이 Breadcrumb 바로 아래, StockTabs 위에 배치
- 사용자가 종목 상세에 들어왔을 때 처음 보이는 것이 광고
- 특히 모바일에서 뷰포트 대부분을 광고가 차지할 수 있음

#### 문제 2: 탭까지의 스크롤 거리
- 모바일 기준: Breadcrumb(32px) + 광고(250px?) + 헤더(80px) + 가격(80px) + 기업개요(90px) + 탭바(40px) = **약 570px**
- 375px 뷰포트에서 탭 콘텐츠가 첫 화면에 보이지 않음

#### 문제 3: 탭 6개 + flex-wrap
- `stock-tabs.tsx:198` — `TabsList className="mb-4 flex-wrap"`
- 모바일 소형 화면에서 6개 탭이 2줄로 wrap될 수 있음
- 특히 KR 종목은 "공시" 탭이 추가되어 6개 전부 표시

#### 문제 4: 차트 컨트롤 복잡도
- `chart-controls.tsx` — 3줄의 컨트롤: 기간(7개) + 오버레이(9개) + 패널(9개) = **25개 토글**
- 초보 투자자 타겟에 비해 과도한 기술적 지표 옵션
- 모바일에서 flex-wrap으로 4-5줄이 될 수 있음

### 3.3 개선안

#### 광고 위치 이동
```
Before: Breadcrumb → AdSlot → 헤더 → 가격 → 개요 → 탭
After:  Breadcrumb → 헤더 → 가격 → 탭 → 탭콘텐츠 → AdSlot → 관련종목
```
- 광고를 탭 콘텐츠 아래로 이동
- 콘텐츠 먼저, 광고는 스크롤 후 자연스럽게 노출

#### 모바일 헤더 압축
```
[삼성전자 005930]         [★ 관심종목]
KOSPI | 반도체
72,500원  ▲1,500 (+2.11%)
```
- 섹터 Badge를 거래소 옆으로 인라인화
- 기업 개요를 탭 내 "정보" 탭으로 통합 (현재 별도 섹션 + 정보 탭 중복)

#### 탭 구조 개선
```
현재: [차트] [시세] [뉴스] [공시] [배당] [실적]   ← 6개 동급
개선: [차트] [정보] [뉴스] [이벤트]               ← 4개

- 정보 탭: 시세 + 기업 개요 + 펀더멘탈 통합
- 이벤트 탭: 공시 + 배당 + 실적 통합 (서브탭 또는 아코디언)
```
- 탭 수 6→4 축소로 모바일 1줄 유지 보장
- "이벤트" 내에서 공시/배당/실적을 서브 섹션으로 배치

#### 차트 컨트롤 단순화
```
기본 모드 (초보자):
  기간: [1주] [1개월] [3개월] [1년]

고급 모드 (토글로 전환):
  오버레이: SMA EMA BB ...
  패널: MACD RSI ...
```
- 기본 상태에서 기간 선택만 노출
- "지표 더보기" 또는 "고급 설정" 토글로 전체 옵션 노출
- 초보자는 단순, 고급 사용자는 기존과 동일한 경험

## 4. 모바일 경험

### 4.1 BottomTabBar 분석

**현재 코드** (`bottom-tab-bar.tsx`):
```tsx
const tabs = [
  { href: "/", label: "홈", icon: Home },
  { href: "/market", label: "시장", icon: Globe },
  { href: "/news", label: "뉴스", icon: Newspaper },
  { href: "/mypage", label: "MY", icon: User },
]
```

**활성 상태 판정:**
- MY 탭: `/mypage`, `/watchlist`, `/settings` 시 활성
- 나머지: exact match만

**문제:**
- `/screener`, `/reports`, `/etf`, `/dividends`, `/board` 등 접속 시 하단 탭 어디에도 활성 표시 없음
- 사용자가 어디에 있는지 네비게이션 컨텍스트 상실

### 4.2 Sheet 메뉴 분석

**현재 코드** (`app-header.tsx:118-144`):
- Sheet side="right", width 288px (w-72)
- 내부: SearchBar + 8개 navLinks + 로그인/회원가입

**문제:**
- 8개 링크가 수직 나열만 되어 있음 — 그룹핑 없음
- 스크롤 시 메뉴 길이 체감 ("홈"부터 "관심종목"까지 + 로그인까지 약 500px)
- 배당, 실적, 가이드, 종목비교, 섹터는 Sheet에도 없음

### 4.3 테이블/그리드 모바일 이슈

#### 종목 비교 (`/compare`)
- `grid-cols-[1fr_1fr_1fr]` — 375px에서 각 열 약 110px
- 셀 내 가격 정보(font-mono)가 잘릴 수 있음
- **개선**: 모바일에서 카드형 레이아웃으로 전환 (종목 A 카드 → 종목 B 카드, 지표별 비교 바)

#### 배당 캘린더
- `grid-cols-[1fr_auto_auto_auto]` — 4열이 좁은 화면에 밀집
- **개선**: 모바일에서 카드형 (종목명 + 배당락일/배당금/수익률을 아래로 배치)

#### 차트 컨트롤
- 25개 토글이 `flex-wrap`으로 4-5줄 차지
- 차트 자체(300px) + 컨트롤(100px+) = 모바일 뷰포트 대부분
- **개선**: 4.3절의 기본/고급 모드 분리

### 4.4 모바일 개선안 종합

1. **BottomTabBar 5탭**: 홈 / 투자 / 분석 / 뉴스 / MY
2. **Sheet 메뉴 그룹핑**: 섹션 헤더(투자 정보, 분석 도구, 커뮤니티, 가이드) 추가
3. **Sheet에 누락 페이지 추가**: 배당, 실적, 가이드, 종목비교, 섹터
4. **테이블 → 카드 전환**: 종목 비교, 배당 캘린더에서 모바일 카드형
5. **차트 컨트롤 접힘 기본**: 기간만 노출, 지표는 토글

## 5. 검색 경험

### 5.1 현황

**데스크톱 SearchBar** (`search-bar.tsx`):
- 헤더 우측 w-64(256px) 고정
- 2글자 이상 입력 시 디바운스 300ms → API 호출
- 결과: Command 패턴 드롭다운, 종목명 + 티커 + 거래소 Badge
- Cmd+K 단축키 표시 (`<kbd>` 요소)

**모바일 SearchCommand** (`search-command.tsx`):
- 헤더의 검색 아이콘 클릭 → CommandDialog (전체 모달)
- Cmd+K 단축키 리스너 (데스크톱에서도 동작)
- GTM 트래킹: search 이벤트, select_content 이벤트

**Sheet 내 SearchBar**: 햄버거 메뉴 열면 상단에 SearchBar 한 벌 더 있음

### 5.2 문제 분석

1. **최근 검색/인기 검색 없음**: 검색 모달 열면 빈 상태. 쿼리 입력 전까지 아무것도 안 보임.
2. **검색 결과 정보 부족**: 종목명 + 티커 + 거래소만 표시. 현재가/등락률이 없어 종목 구분 어려움 (동명 종목 시).
3. **검색 Scope 제한**: 종목 검색만 가능. 뉴스, 가이드, 페이지 검색 불가.
4. **Sheet 내 SearchBar 중복**: 모바일에서 (1) 검색 아이콘 → CommandDialog, (2) 햄버거 → Sheet → SearchBar — 두 가지 경로가 동일 기능이나 다른 UI.
5. **빈 검색어 UX**: 2글자 미만 입력 시 아무 피드백 없음. "2글자 이상 입력하세요" 같은 안내 부재.

### 5.3 개선안

#### 검색 모달 진입 시 기본 콘텐츠 표시
```
[검색 모달 열림, 쿼리 비어있을 때]
────────────────────────────
  🔍 종목명 또는 티커 검색...
────────────────────────────
  최근 검색                    ✕
  삼성전자 005930         KOSPI
  AAPL                      US

  인기 검색
  SK하이닉스 000660      KOSPI
  NVDA                      US
  TIGER 미국S&P500        ETF
────────────────────────────
```

- 최근 검색: localStorage에 최근 5건 저장
- 인기 검색: `/api/stocks/popular` 상위 5건 활용 (이미 존재하는 API)

#### 검색 결과에 가격 정보 추가
```
  삼성전자  005930  KOSPI    72,500원  ▲2.11%
  삼성SDI   006400  KOSPI   389,500원  ▼0.51%
```

#### 검색 Scope 확장 (중기)
- 종목 외에 "페이지 바로가기" 그룹 추가:
  - "배당" 입력 → [배당 캘린더 페이지로 이동] 결과 표시
  - "가이드" 입력 → [투자 가이드 페이지로 이동] 결과 표시

## 7. 시각적 일관성

### 7.1 색상 체계

**현재 Light 모드:**
- Primary: `oklch(0.205 0 0)` — **거의 검정** (무채색)
- Background: `oklch(1 0 0)` — 순백
- Muted-foreground: `oklch(0.556 0 0)` — 회색
- Stock-up: `#e53e3e` (빨강), Stock-down: `#3182ce` (파랑)

**현재 Dark 모드:**
- Primary: `oklch(0.922 0 0)` — **거의 흰색**
- Background: `oklch(0.145 0 0)` — 거의 검정
- Stock-up: `#fc8181`, Stock-down: `#63b3ed` — 밝아진 빨강/파랑

**문제:**
1. **브랜드 색상 부재**: Primary가 무채색이라 모든 버튼, 링크, 강조가 흑/백. 서비스 정체성 약함.
2. **CTA 구분 어려움**: 검정 버튼(primary)과 검정 텍스트의 시각적 위계 차이 불명확
3. **chart-1~5 색상**: oklch 기반 파란색 계열로 일관 → 차트 데이터 구분에는 좋으나 주식 색상(빨/파)과 혼동 가능

**개선안:**
- 브랜드 Primary 색상 도입: 예시 — `oklch(0.55 0.15 250)` (딥 블루) 또는 `oklch(0.60 0.18 145)` (그린)
- 주식 빨강/파랑은 유지 (한국 컨벤션), 브랜드 색은 별도 채도 있는 색상
- CTA 버튼에 브랜드 색 적용, 텍스트 링크에는 브랜드 색 밝은 변형

### 7.2 타이포그래피

**현재:**
- Font: Pretendard Variable (한글 최적화, 좋음)
- 제목: `text-2xl font-bold` (24px)
- 본문: `text-sm` (14px)
- 캡션: `text-xs` (12px)

**문제:**
- 중간 단계 부재: 24px → 14px 사이의 `text-lg`(18px), `text-base`(16px) 활용이 적음
- 섹션 제목(`h2`)이 `text-lg font-semibold`로 본문과 크게 차이 안 남

**개선안:**
- 명확한 타입 스케일: h1(28px) → h2(20px) → h3(16px bold) → body(14px) → caption(12px)
- 각 페이지 h1에 `text-2xl md:text-3xl`로 반응형 크기 조정

### 7.3 간격 및 카드 디자인

**잘된 점:**
- `PageContainer`로 일관된 padding (px-4 md:px-6 xl:px-8)
- Card 컴포넌트 재사용 (shadcn/ui 기반)
- `rounded-lg`, `border`, `hover:shadow-md` 일관 적용

**개선점:**
- 섹션 간 간격이 `mb-8`, `mb-6`, `mb-4`로 혼재 — 8px 단위로 통일 권장
- IndexCard와 StockRow의 디자인 언어가 다름 (Card vs 리스트형) — 의도적이라 문제는 아니나, 홈에서 혼재 시 시각적 불일치감

### 7.4 다크모드

**잘된 점:**
- Stock 색상이 다크모드에서 밝게 조정됨 (#e53e3e → #fc8181)
- 차트도 `isDark` 감지하여 그리드/텍스트 색상 조정
- next-themes 기반으로 깜빡임 방지 (`suppressHydrationWarning`)

**개선점:**
- 차트의 다크모드 감지가 `document.documentElement.classList.contains("dark")`로 수동 — 테마 변경 시 차트가 즉시 반영되지 않을 수 있음 (리마운트 필요)
- Card 배경이 dark에서 `oklch(0.205 0 0)`로 배경과 차이 미미 — 카드 경계 인식 어려움
