## 1. 배경 및 목적

### 1.1 현재 상태
- **KR 종목**: 6탭 — 차트 | 시세 | 뉴스 | 공시 | 배당 | 실적
- **US 종목**: 5탭 — 차트 | 시세 | 뉴스 | 배당 | 실적 (공시 없음)
- Phase 1에서 `overflow-x-auto` 가로 스크롤 적용하여 모바일 2줄 wrap 방지

### 1.2 문제
1. **모바일 사용성**: 375px 화면에서 6탭이 가로 스크롤 필요 → 오른쪽 탭(배당/실적) 발견성 저하
2. **정보 분산**: 시세 탭과 기업 개요(탭 상단)가 유사 정보를 중복 표시
3. **KR/US 비대칭**: KR 6탭 vs US 5탭으로 시장별 경험 불일치
4. **탭 간 이동 빈도**: 배당/실적/공시는 개별적으로 콘텐츠양이 적어 별도 탭 불필요

### 1.3 목표
- **4탭 통합**으로 모바일 1줄 유지 보장 (KR/US 동일)
- 정보 계층 재설계로 콘텐츠 발견성 향상
- Server Component slot 패턴 최소 변경

## 2. 통합 방안

### 2.1 최종안: 4탭 구조

```
┌──────────┬──────────┬──────────┬──────────┐
│   차트   │   정보   │   뉴스   │  이벤트  │
│  Chart   │   Info   │   News   │  Events  │
└──────────┴──────────┴──────────┴──────────┘
```

### 2.2 탭별 콘텐츠 매핑

#### 차트 탭 (변경 없음)
- 기존 ChartTabServer 그대로 유지
- 기간 선택, 오버레이/패널 지표

#### 정보 탭 (시세 + 기업 개요 통합)
```
┌─────────────────────────────────┐
│ 현재가 상세                     │
│  시가/고가/저가/종가            │
│  거래량/거래대금                │
│  52주 고/저                     │
├─────────────────────────────────┤
│ 펀더멘탈                        │
│  PER / PBR / 시가총액           │
│  EPS / BPS / 배당수익률         │
├─────────────────────────────────┤
│ 기업 개요                       │
│  (현재 탭 상단의 기업 개요를    │
│   이 탭으로 이동)               │
└─────────────────────────────────┘
```
- **변경점**: 기존 시세(InfoTab) + 페이지 상단 기업 개요 섹션 통합
- 기업 개요가 탭 상단에서 제거되어 **탭까지의 스크롤 거리 단축** (모바일 ~90px 절약)

#### 뉴스 탭 (변경 없음)
- 기존 NewsTabServer 그대로 유지

#### 이벤트 탭 (공시 + 배당 + 실적 통합)
```
┌─────────────────────────────────┐
│ [배당] [실적] [공시]  ← 서브탭  │
│  (KR: 3개 / US: 2개)           │
├─────────────────────────────────┤
│                                 │
│  (선택된 서브탭 콘텐츠)         │
│                                 │
└─────────────────────────────────┘
```
- **서브탭 방식**: Tabs 내 Tabs(중첩) 또는 SegmentedControl 스타일 토글
- KR: 배당 / 실적 / 공시 (3개 서브탭)
- US: 배당 / 실적 (2개 서브탭, 공시 없음)
- 기본 선택: **배당** (가장 빈번하게 조회되는 항목)

### 2.3 변경 전후 매핑표

| 변경 전 (6탭) | 변경 후 (4탭) | 이동 방식 |
|---------------|---------------|-----------|
| 차트 | **차트** | 그대로 유지 |
| 시세 | **정보** | 이름 변경 + 기업 개요 흡수 |
| 뉴스 | **뉴스** | 그대로 유지 |
| 공시 (KR only) | **이벤트 → 공시 서브탭** | 이벤트 탭 내 서브탭 |
| 배당 | **이벤트 → 배당 서브탭** | 이벤트 탭 내 서브탭 |
| 실적 | **이벤트 → 실적 서브탭** | 이벤트 탭 내 서브탭 |
| (페이지 상단) 기업 개요 | **정보 탭 하단** | 섹션 이동 |

## 4. 데스크톱 레이아웃

데스크톱에서도 동일한 4탭 구조 적용. 넓은 화면의 이점:

```
┌──────────────────────────────────────────────────┐
│  차트  │  정보  │  뉴스  │  이벤트               │
│  ════                                            │
│                                                  │
│  ┌────────────────────┐ ┌───────────────────────┐│
│  │                    │ │  거래 정보            ││
│  │    차트 영역       │ │  시가: 72,000         ││
│  │    (600px+)        │ │  고가: 73,200         ││
│  │                    │ │  저가: 71,800         ││
│  │                    │ │  거래량: 12.3M        ││
│  └────────────────────┘ └───────────────────────┘│
│  [기간] [오버레이] [패널]                        │
└──────────────────────────────────────────────────┘
```

- 차트 탭: 기존과 동일 (차트 + 사이드 시세 요약)
- 이벤트 탭: 서브탭이 넓은 화면에서는 3열 그리드로도 표시 가능 (옵션)

## 5. 기술 구현 가이드

### 5.1 Server Component Slot 변경

**현재** (`stock/[ticker]/page.tsx`):
```
props로 전달하는 slot:
- chartSlot: <ChartTabServer>
- infoSlot: <InfoTabServer>
- newsSlot: <NewsTabServer>
- disclosureSlot: <DisclosureTabServer> (KR only)
- dividendSlot: <DividendTabServer>
- earningsSlot: <EarningsTabServer>
```

**변경 후**:
```
props로 전달하는 slot:
- chartSlot: <ChartTabServer>        (유지)
- infoSlot: <InfoTabServer>          (유지, 기업 개요 데이터 추가)
- newsSlot: <NewsTabServer>          (유지)
- eventsSlot: <EventsTabWrapper>     (신규 래퍼)
  └─ dividendSlot: <DividendTabServer>
  └─ earningsSlot: <EarningsTabServer>
  └─ disclosureSlot: <DisclosureTabServer> (KR only)
```

### 5.2 EventsTabWrapper 컴포넌트 (신규)

```
역할: 서브탭 UI를 관리하는 Client Component 래퍼
- 내부에 Tabs (서브탭) 상태 관리
- KR/US 분기: market prop으로 공시 서브탭 표시 여부 결정
- 각 서브탭 콘텐츠는 기존 Server Component를 children으로 받음
```

### 5.3 기업 개요 이동

```
현재: page.tsx 상단에 CompanyOverview 섹션 (탭 밖)
변경: InfoTabServer 내부 하단으로 이동
- CompanyOverview 데이터 fetch를 InfoTabServer로 통합
- page.tsx 상단에서 CompanyOverview 제거
```

### 5.4 변경 대상 파일

| 파일 | 변경 내용 | 공수 |
|------|----------|------|
| `stock/[ticker]/page.tsx` | slot 구조 변경, 기업 개요 제거 | 30분 |
| `components/stock/stock-tabs.tsx` | 6탭→4탭, 이벤트 탭 추가 | 45분 |
| `components/stock/events-tab-wrapper.tsx` | **신규** — 서브탭 래퍼 | 30분 |
| `components/stock/tabs/info-tab-server.tsx` | 기업 개요 섹션 통합 | 20분 |
| **합계** | | **~2시간** |

### 5.5 URL/딥링크 고려

- 현재: 탭 상태가 `useState`로 관리 (URL 미반영)
- 유지: 탭 전환 시 URL 변경 없음 (현행과 동일)
- 이벤트 서브탭도 로컬 상태로 관리 (추가 복잡도 최소화)

## 6. 검토한 대안

### 6.1 안 B: 투자+분석 통합 (5탭→3탭)

```
[차트] [뉴스] [투자정보]
              └─ 시세/배당/실적/공시 전부
```

**기각 사유**: 투자정보 탭이 너무 무거움. 스크롤 길이 과다. 서브탭 4~5개 필요.

### 6.2 안 C: 뉴스를 이벤트에 통합 (3탭)

```
[차트] [정보] [소식]
              └─ 뉴스/공시/배당/실적
```

**기각 사유**: 뉴스는 조회 빈도가 높아 독립 탭 유지가 사용성에 유리. 소식 탭 내 서브탭 4개는 과다.

### 6.3 안 D: 아코디언 (탭 내 접기/펼치기)

**기각 사유**: 서브탭 방식이 콘텐츠 전환에 더 직관적. 아코디언은 여러 섹션 동시 열림 시 스크롤 길어짐.
