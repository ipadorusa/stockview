## 1. 현재 상태 분석

### 1.1 정보 아키텍처 맵

```
StockView (Root Layout: AppHeader + BottomTabBar + Footer)
├── / (홈) — 지수, 환율, 퀵링크, 인기종목, 최신뉴스
├── 투자 정보 그룹
│   ├── /market — 시장 개요 (KR/US 탭, 지수, 상승/하락 TOP 5, 섹터)
│   ├── /etf — ETF 목록 (KR/US 탭, 거래대금순)
│   ├── /etf/[ticker] — ETF 상세
│   ├── /sectors — 섹터별 종목
│   ├── /sectors/[name] — 개별 섹터
│   ├── /dividends — 배당 캘린더 (예정/최근/고배당)
│   └── /earnings — 실적 캘린더
├── 분석 도구 그룹
│   ├── /screener — 스크리너 (기술적 시그널 필터)
│   ├── /screener/[signal] — 시그널별 상세
│   ├── /reports — AI 리포트 목록 + 리포트 요청 탭
│   ├── /reports/[slug] — AI 리포트 상세
│   ├── /reports/request — 리포트 요청 작성
│   ├── /reports/stock/[ticker] — 종목별 리포트
│   ├── /compare — 종목 비교 (최대 4개)
│   └── /guide — 투자 가이드 (5개 서브페이지)
├── 뉴스·커뮤니티 그룹
│   ├── /news — 뉴스 피드
│   ├── /board — 게시판
│   ├── /board/new — 글쓰기
│   └── /board/[id] — 게시글 상세/수정
├── MY 그룹
│   ├── /watchlist — 관심종목 + 포트폴리오 (탭)
│   ├── /mypage — 프로필, 관심종목 미리보기, 퀵링크
│   ├── /settings — 설정
│   └── /portfolio — (watchlist 내 탭으로 통합됨)
├── 인증
│   ├── /auth/login
│   ├── /auth/register
│   └── /auth/forgot-password
├── 종목 상세
│   └── /stock/[ticker] — 차트/종목정보/뉴스/이벤트(배당·실적·공시) 탭
└── 기타
    ├── /about, /contact, /privacy, /terms
    └── /admin/* (관리자)
```

**총 39개 페이지** | 4개 네비게이션 그룹 | 2단 헤더(데스크톱) + 5탭 바텀내비(모바일)

### 1.2 주요 사용자 흐름

| 흐름 | 경로 | 터치 수 |
|------|------|---------|
| **시세 확인** | 홈 → 인기종목 클릭 → /stock/[ticker] | 2 |
| **종목 검색** | 검색 아이콘 → SearchCommand → /stock/[ticker] | 3 |
| **관심종목 추가** | /stock/[ticker] → WatchlistButton 클릭 | 2 (로그인 필요) |
| **시장 탐색** | 홈 → /market → KR/US 탭 전환 → 종목 클릭 | 3~4 |
| **종목 비교** | /compare → 검색 × 2~4 → 결과 확인 | 5~8 |
| **뉴스 확인** | 홈 → 뉴스 "전체보기" → /news | 2 |
| **AI 리포트** | /reports → 리포트 클릭 → /reports/[slug] | 2 |
| **포트폴리오** | /watchlist → 포트폴리오 탭 → AddPortfolioDialog | 3 |

### 1.3 강점 분석

1. **듀얼 마켓 통합**: KR/US를 하나의 UI에서 자연스럽게 전환 — 한국 투자자에게 강력한 가치
2. **초보자 친화적 설계**: 가이드 페이지 5개, 각 페이지 상단 교육 텍스트, 히어로 온보딩 배너
3. **한국식 컬러 컨벤션**: 빨강=상승, 파랑=하락 — 타겟 사용자에게 직관적
4. **서버 컴포넌트 활용**: ISR(15분) + Suspense 스트리밍으로 빠른 초기 로딩
5. **풍부한 분석 도구**: 스크리너, AI 리포트, 종목비교, 배당/실적 캘린더 등 포괄적 기능셋
6. **검색 UX**: SearchCommand(Cmd+K)로 빠른 접근, 바텀탭에서도 바로 접근 가능
7. **관심종목+포트폴리오 통합**: watchlist 페이지 내 탭으로 관련 기능 집약
8. **반응형 네비게이션**: 데스크톱 2단 헤더 + 모바일 바텀탭 + 햄버거 시트

### 1.4 약점/문제점

#### A. 정보 아키텍처 문제

| 문제 | 심각도 | 설명 |
|------|--------|------|
| **IA 깊이 불균형** | 높음 | "투자 정보" 5개 vs "뉴스·커뮤니티" 2개. 그룹 크기가 극단적으로 차이남 |
| **관심종목/포트폴리오 소속 혼란** | 높음 | watchlist에 포트폴리오가 탭으로 들어가 있지만, 네비 "더보기"에 포트폴리오가 별도 링크. 마이페이지에도 관심종목 미리보기 중복 |
| **분석 도구 접근성** | 중간 | 스크리너, 비교, AI리포트가 "분석" 하위로 숨겨져 있어 발견 비용 높음 |
| **종목상세 진입점 부족** | 중간 | /stock/[ticker]로의 진입이 검색/인기종목/시장 상승하락 TOP에 한정. 뉴스→종목 연결 약함 |

#### B. 네비게이션 문제

| 문제 | 심각도 | 설명 |
|------|--------|------|
| **바텀탭 5개 중 "시장"의 정체성** | 높음 | 모바일 탭 "시장"이 /market으로만 감. ETF, 섹터, 배당, 실적은 탐색 불가 |
| **"더보기" 카테고리** | 높음 | 관심종목+포트폴리오+마이페이지+소개를 "더보기"에 묶음 — 핵심 기능(관심종목)이 잡동사니 느낌 |
| **2단 서브내비 인지 부족** | 중간 | 데스크톱 서브내비가 bg-muted/30으로 약하게 표시, 사용자가 존재 자체를 인식 못할 수 있음 |
| **모바일 햄버거 메뉴 과다** | 중간 | Sheet 내 15+개 링크가 스크롤 필요. 그룹 라벨이 uppercase 영문("투자 정보")으로 시각적 구분 약함 |

#### C. 사용자 흐름 마찰점

| 마찰점 | 설명 |
|--------|------|
| **종목비교 진입 비용** | /compare 페이지에 도달한 후 매번 검색부터 시작. 종목 상세에서 "이 종목과 비교" 원클릭 진입점 없음 |
| **관심종목 빈 상태** | 비로그인 사용자에게 "로그인하세요" 단일 메시지. 관심종목의 가치를 보여주는 프리뷰 없음 |
| **홈 → 특정 기능 발견** | 홈에 QuickLinkCard 4개만 있고 ETF, 배당, 실적, 섹터 진입점 없음 |
| **마이페이지 기능 빈약** | 프로필 + 관심종목 미리보기 + 3개 링크뿐. 최근 본 종목, 활동 이력 없음 |
| **뉴스→종목 연결** | 뉴스 카드에서 관련 종목으로의 직접 링크 없음 (뉴스만 외부 링크) |
| **모바일 검색 결과 액션 부재** | 검색 → 종목 이동만 가능. 바로 관심종목 추가/비교 추가 불가 |

#### D. UI/비주얼 문제

| 문제 | 설명 |
|------|------|
| **홈 정보 과밀** | 히어로 + 초보자 CTA + 지수 + 환율 + 퀵링크 + 인기종목 + 뉴스 + 광고 = 한 화면에 8개 섹션 |
| **데이터 시각화 부재** | 홈의 지수/환율이 숫자 카드뿐. 미니 스파크라인이 있으나 인터랙티브하지 않음 |
| **일관성 없는 탭 패턴** | /market → KR/US 탭, /etf → KR/US 탭, /watchlist → 관심종목/포트폴리오 탭 — 같은 UI지만 목적이 다름 |
| **광고 배치** | 각 페이지 하단에 AdSlot. 콘텐츠 사이에 삽입되지 않아 수익 최적화 미흡하나, UX 관점에서는 비교적 양호 |

## 2. 개선 전략

### 2.1 IA 재구성안

#### 현재 → 개선안 비교

```
[현재 4그룹]                    [개선안 3그룹 + 컨텍스트 허브]
투자 정보 (5)                   탐색 (Explore)
  시장, ETF, 섹터, 배당, 실적     ├── /market (통합 대시보드: 지수+섹터+ETF+캘린더)
분석 (5)                          ├── /screener (스크리너)
  스크리너, AI리포트,              └── /news (뉴스+커뮤니티 통합)
  리포트요청, 비교, 가이드       분석 (Analyze)
뉴스·커뮤니티 (2)                 ├── /reports (AI 리포트)
  뉴스, 게시판                    ├── /compare (종목 비교)
더보기 (6)                        └── /guide (투자 가이드)
  관심종목, 포트폴리오,          내 투자 (My)
  마이페이지, 소개 등              ├── /watchlist (관심종목)
                                   ├── /portfolio (포트폴리오 — 독립 분리)
                                   └── /mypage (프로필+활동+설정 통합)

                                 [컨텍스트 허브: /stock/[ticker]]
                                   종목 상세에서 비교·리포트·관심종목 등
                                   모든 액션을 원클릭으로 연결
```

#### 핵심 변경점

1. **"시장" → 통합 탐색 대시보드**: ETF, 섹터, 배당, 실적을 /market 내 탭/섹션으로 통합. 별도 페이지 유지하되 /market이 허브 역할
2. **뉴스·커뮤니티 → 탐색에 통합**: 뉴스와 게시판을 별도 그룹에서 빼고, 뉴스는 "탐색"에, 게시판은 커뮤니티로 뉴스 하위 탭
3. **포트폴리오 독립**: watchlist 탭에서 분리하여 독립 페이지. 관심종목과 포트폴리오는 목적이 다름 (모니터링 vs 수익 추적)
4. **마이페이지 강화**: 설정을 mypage 내 섹션으로 흡수. 최근 본 종목, 활동 이력 추가

### 2.2 핵심 흐름 최적화

#### Flow 1: "종목 발견 → 분석 → 액션" 원스톱 흐름

**현재 문제**: 종목을 발견해도 분석(비교, 리포트)과 액션(관심종목, 포트폴리오)이 분리된 페이지에 있어 컨텍스트 전환 필요

**개선안: 종목 상세 "Command Bar"**
```
/stock/[ticker] 페이지 하단 플로팅 바:
┌─────────────────────────────────────────┐
│ ⭐ 관심  | 📊 비교추가 | 📝 리포트 | ··· │
└─────────────────────────────────────────┘
```
- 종목 상세 페이지에 **Floating Action Bar** 추가
- "비교 추가" 클릭 → 현재 종목이 미리 채워진 /compare로 이동
- "리포트" 클릭 → 해당 종목의 AI 리포트로 직행 (없으면 요청 페이지)
- 모바일에서는 바텀탭 위에 겹쳐 표시

#### Flow 2: "시장 탐색 → 종목 발견" 최적화

**현재 문제**: /market에서 상승/하락 TOP 5만 보임. 섹터, ETF, 배당, 실적 확인하려면 다른 페이지로 이동

**개선안: Market Dashboard 스와이프 섹션**
```
/market 페이지:
[지수 카드]
[── 탐색 필터 바 ──]
  인기 | 상승 | 하락 | 거래량 | 섹터 | ETF | 배당예정 | 실적예정
[── 결과 리스트 ──]
  (선택한 필터에 따라 동적 변경)
```
- 단일 페이지에서 다양한 관점으로 종목 탐색
- 가로 스크롤 필터 칩으로 컨텍스트 전환 최소화
- 각 필터는 기존 개별 페이지(/etf, /dividends 등)의 데이터를 재활용

#### Flow 3: "검색 → 즉시 액션" 강화

**현재 문제**: SearchCommand에서 종목 선택 → 종목 상세로만 이동

**개선안: 검색 결과 인라인 액션**
```
검색 결과 각 항목:
┌──────────────────────────────────┐
│ 삼성전자 005930          ₩68,400 │
│ [이동] [⭐ 관심] [📊 비교]      │
└──────────────────────────────────┘
```
- 검색 결과에서 바로 관심종목 추가, 비교 슬롯에 추가 가능
- 가격 정보를 검색 결과에서 바로 노출 (현재는 이름만 표시)

#### Flow 4: "뉴스 → 종목" 크로스링크

**현재 문제**: 뉴스 카드가 외부 링크만 제공. 뉴스에 언급된 종목으로의 연결 없음

**개선안: 뉴스 카드 종목 태그**
```
뉴스 카드:
┌──────────────────────────────────┐
│ "삼성전자, AI 반도체 수주 확대"  │
│ Google News · 2시간 전           │
│ [삼성전자] [SK하이닉스]  ← 종목태그 │
└──────────────────────────────────┘
```
- 뉴스 제목에서 매칭된 종목을 태그로 표시 (DB에 이미 stock 매칭 데이터 있음)
- 태그 클릭 → /stock/[ticker]로 이동

### 2.3 네비게이션 혁신안

#### 모바일 바텀탭 재설계

```
[현재]                    [개선안]
홈 | 검색 | 시장 | 관심 | MY    홈 | 탐색 | 검색 | 관심 | MY
```

**변경점:**
1. **"시장" → "탐색"**: 아이콘을 Compass로 변경. 탭하면 통합 Market Dashboard로 이동
2. **검색 위치 이동**: 중앙 배치 (토스/카카오증권 패턴). 가장 자주 쓰는 액션을 가장 접근성 높은 위치에
3. **검색 동작 변경**: 오버레이가 아닌 전용 검색 페이지 (/search) — 최근 검색, 인기 검색어, 추천 종목 포함

#### 데스크톱 헤더 2단 서브내비 개선

**현재 문제**: 서브내비가 시각적으로 약하고, 카테고리별 활성 상태 인식 어려움

**개선안:**
1. 서브내비 배경을 bg-muted/30 → bg-accent/60으로 강화
2. 활성 서브링크에 하단 2px 인디케이터 바 추가 (토스증권 스타일)
3. "현재 위치" 텍스트 breadcrumb를 서브내비와 통합하여 중복 제거

#### 컨텍스트 네비게이션 (신규)

**종목 상세 페이지 내 상단 탭 확장:**
```
[현재 탭: 차트 | 종목정보 | 뉴스 | 이벤트]
[개선 탭: 차트 | 정보 | 뉴스 | 배당·실적 | AI분석 | 유사종목]
```
- "이벤트" → "배당·실적"로 명확화
- "AI분석" 탭 추가: 해당 종목의 AI 리포트 인라인 표시
- "유사종목" 탭: peer-stocks 컴포넌트를 정식 탭으로 승격

### 2.4 차별화 전략

경쟁 서비스와의 차별점을 4가지 전략 축으로 수립:

#### 전략 1: "투자 여정 퍼널" — 초보자→중급자 성장 경로

**경쟁사 약점**: 토스증권은 거래 중심, Yahoo Finance는 데이터 나열형, TradingView는 고급자 전용

**StockView 차별점: 교육 통합형 투자 플랫폼**
- 종목 상세의 모든 지표에 `TooltipHelper` (이미 구현됨) 강화 — "PER이란?" 원터치 교육
- 스크리너 결과에 "이 시그널의 의미" 컨텍스트 카드 추가
- 마이페이지에 "나의 투자 레벨" 게이미피케이션 (가이드 읽기, 리포트 요청, 비교 사용 등 활동 기반)
- 가이드 콘텐츠를 독립 페이지에서 빼내어 각 기능 페이지에 인라인 교육 삽입

#### 전략 2: "크로스마켓 인사이트" — KR↔US 연결

**경쟁사 약점**: 대부분 단일 시장 또는 시장별 분리 UI

**StockView 차별점: 한미 교차 분석**
- 홈에 "미국이 오르면 한국은?" 상관관계 미니 차트 위젯
- 종목비교에서 KR↔US 교차 비교 시 환율 자동 환산 표시
- 섹터별 KR/US 대응 종목 매핑 (예: 삼성전자 ↔ AAPL, INTC)
- 뉴스에 "관련 한국/미국 종목" 크로스 태그

#### 전략 3: "마이 대시보드" — 개인화 허브

**경쟁사 약점**: 관심종목은 단순 리스트, 포트폴리오는 별도 앱

**StockView 차별점: 통합 개인 대시보드**
- 관심종목 리스트 → **미니 대시보드** 전환: 각 종목의 1일 차트 스파크라인 + 알림 조건 설정
- 포트폴리오에 **수익률 시계열 차트** 추가 (일별 포트폴리오 가치 변화)
- "최근 본 종목" 히스토리 (localStorage 기반, 로그인 불필요)
- 마이페이지 → **개인화 홈**: 관심종목 뉴스, 관심 섹터 동향, 맞춤 AI 리포트 추천

#### 전략 4: "제로 프릭션 비교" — 즉시 비교 UX

**경쟁사 약점**: 비교 기능이 별도 메뉴에 숨겨져 있거나 없음 (토스), 복잡함 (TradingView)

**StockView 차별점: 어디서든 비교 시작**
- 종목 상세, 검색 결과, 관심종목 리스트 어디서든 "비교 슬롯"에 추가 가능
- 비교 슬롯은 **글로벌 상태**로 유지 (세션 동안 최대 4개)
- 화면 우하단에 비교 FAB(Floating Action Button): "2개 선택됨 — 비교하기"
- /compare 페이지 진입 시 이미 선택된 종목이 채워져 있음
