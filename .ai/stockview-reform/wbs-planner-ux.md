### 1.1 메가메뉴 → 2단 네비게이션 전면 교체

#### 1.1.1 §2 데스크톱 메가메뉴 섹션 → 2단 네비게이션으로 재작성

- **예상 시간**: 45분
- **선행 조건**: 없음 (즉시 착수 가능)
- **작업 내용**:
  - §2 제목 변경: "데스크톱 메가메뉴 설계" → "데스크톱 2단 네비게이션 설계"
  - §2.1 1단 네비게이션 구조 유지 (5개 카테고리: 홈/투자/분석/뉴스/더보기)
  - §2.2 드롭다운 구조 전면 삭제 → 2단 서브 네비게이션 바 구조로 교체
    - 1단 카테고리 클릭/활성 시 2단에 해당 서브 링크 표시
    - 투자: 시장 개요, ETF, 섹터별 종목, 배당 캘린더, 실적 캘린더
    - 분석: 스크리너, AI 리포트, 종목 비교, 투자 가이드
    - 뉴스/홈/더보기: 2단 없음 (직접 이동)
  - §2.3 활성 상태 로직: pathname 기반 2단 자동 표시 규칙 정의
  - §2.4 구현 방식 변경:
    - Radix NavigationMenu 삭제 → 기존 컴포넌트만으로 구현
    - 호버 로직 불필요, URL 기반 조건부 렌더링으로 변경
    - 2단 바 높이: 40px, 데스크톱(lg:)에서만 표시
- **산출물**: §2 전체 재작성 (와이어프레임 포함)
- **완료 기준**: 메가메뉴 관련 내용이 완전히 제거되고, 2단 네비 구조/동작/스타일이 구체적으로 명시됨

## 2. 브랜드 컬러 충돌 대안 검토

### 2.1 현재 색상 체계 분석

#### 2.1.1 기존 CSS 변수 및 stock 색상과의 충돌 분석

- **예상 시간**: 20분
- **선행 조건**: 없음
- **작업 내용**:
  - 현재 색상 체계 정리:
    - `--primary`: oklch(0.205 0 0) (light) / oklch(0.922 0 0) (dark) — 무채색
    - `--color-stock-up`: #e53e3e — 빨강 (hue ~0)
    - `--color-stock-down`: #3182ce — 파랑 (hue ~210)
    - chart-1~5: oklch 파란색 계열 (hue ~260)
  - 충돌 시나리오 분석:
    - Indigo/파랑 계열 브랜드색 → stock-down(파랑)과 시각적 혼동
    - 빨강 계열 브랜드색 → stock-up(빨강)과 혼동
    - 이 두 색역대는 **금지 영역**
  - 안전한 hue 범위 도출: stock 색상(hue 0, 210)과 최소 60도 이상 차이나는 영역
- **산출물**: 색상 충돌 분석 문서 (spec-navigation-redesign.md 내 부록 또는 별도 섹션)
- **완료 기준**: 금지 색역대와 안전 색역대가 명확히 구분

#### 2.1.2 대안 브랜드 컬러 후보 제안

- **예상 시간**: 25분
- **선행 조건**: 2.1.1 완료
- **작업 내용**:
  - 안전 색역대에서 3개 후보 제안 (oklch 값 포함):

  | 후보 | Hue | Light 모드 | Dark 모드 | 장점 | 단점 |
  |------|-----|-----------|----------|------|------|
  | A. 보라 (Purple) | ~300 | oklch(0.50 0.15 300) | oklch(0.75 0.12 300) | stock 빨강/파랑과 완전 구분, 금융 앱에서 차별화 | 다소 독특한 인상 |
  | B. 초록 (Emerald) | ~145 | oklch(0.55 0.15 145) | oklch(0.72 0.12 145) | 금융에서 "성장/긍정" 연상, 자연스러움 | 한국 주식 관례상 상승=빨강이므로 혼동 낮음 |
  | C. 틸 (Teal) | ~180 | oklch(0.50 0.12 180) | oklch(0.75 0.10 180) | 파랑과 초록 사이, 모던한 인상, stock 색상과 충분히 구분 | stock-down 파랑(hue 210)과 간격이 30도로 좁음 → 주의 필요 |

  - 각 후보에 대해 `text-primary`, `bg-primary`, `border-primary` 적용 시 시각적 예상 영향 기술
  - **권장안**: B(초록) 또는 A(보라) — stock 색상과 가장 명확히 구분

- **산출물**: 브랜드 컬러 후보 3개 비교표 + 권장안
- **완료 기준**: PM이 후보 중 하나를 선택할 수 있을 정도로 구체적

#### 2.1.3 선택된 컬러의 전체 적용 범위 체크리스트

- **예상 시간**: 15분
- **선행 조건**: 2.1.2 완료 (PM 선택 전이라도 범용으로 작성 가능)

## 3. 프론트엔드 개발자 협업 포인트 정리

### 3.1 개발 착수 전 UI 스펙 체크리스트

#### 3.1.1 네비게이션 관련 확정 스펙 정리

- **예상 시간**: 20분
- **선행 조건**: 1.1~1.8 (spec 문서 수정 완료)
- **작업 내용**:
  - 개발자가 구현 전 확인해야 할 핵심 결정 사항을 체크리스트로 정리:

  **데스크톱 2단 네비게이션:**
  - [ ] 1단 카테고리 개수: 5개 (홈/투자/분석/뉴스/더보기)
  - [ ] "홈" 링크 유지 여부: 유지 (로고와 중복이지만 보수적 접근)
  - [ ] 2단 표시 조건: 투자/분석 카테고리 활성 시에만 표시
  - [ ] 2단 높이: 40px
  - [ ] 2단 배경: `bg-muted` 또는 `bg-background border-b`
  - [ ] 2단 활성 링크 스타일: `text-primary font-medium` + 하단 2px 인디케이터
  - [ ] NavigationMenu 설치 불필요 확인

  **모바일 BottomTabBar:**
  - [ ] 탭 개수: 5개 (홈/투자/분석/뉴스/MY)
  - [ ] `/stock/[ticker]`: 모든 탭 비활성
  - [ ] safe-area-inset-bottom 적용
  - [ ] 아이콘: Home, TrendingUp, BarChart3, Newspaper, User (lucide-react)

  **모바일 Sheet:**
  - [ ] 그룹: 투자 정보(5개) / 분석 도구(4개) / 커뮤니티(2개)
  - [ ] 섹션 헤더 스타일: `text-xs font-semibold text-muted-foreground uppercase tracking-wider`
  - [ ] Sheet 닫기: `usePathname()` 변경 감지 방식으로 전환
  - [ ] Sheet 내 SearchBar: 유지 (검색 모달과 별도 경로 — Phase 2에서 통합 검토)

- **산출물**: 개발 착수 전 체크리스트 문서 (이 WBS 내 또는 spec 문서 부록)
- **완료 기준**: 개발자가 이 체크리스트만으로 구현 가능

#### 3.1.2 반응형 breakpoint별 동작 정의

- **예상 시간**: 25분
- **선행 조건**: 3.1.1 완료
- **작업 내용**:
  - Tailwind CSS 4 기본 breakpoint 기준으로 각 네비게이션 요소의 동작 정의:

  | 요소 | < 640px (모바일) | 640~1023px (태블릿) | >= 1024px (데스크톱) |
  |------|-----------------|-------------------|-------------------|
  | 1단 네비 (카테고리) | 숨김 | 숨김 | 표시 (`lg:flex`) |
  | 2단 서브 네비 | 숨김 | 숨김 | 조건부 표시 (`lg:flex`, 투자/분석 활성 시) |
  | 햄버거 메뉴 (Sheet) | 표시 | 표시 | 숨김 (`lg:hidden`) |
  | BottomTabBar | 표시 | 표시 | 숨김 (`lg:hidden`) |
  | 검색 아이콘 | 표시 (모달) | 표시 (모달) | 표시 (인라인 or 모달) |
  | 로고 | 축약 (아이콘만) | 전체 | 전체 |
  | 테마 토글 | 표시 | 표시 | 표시 |

  - **breakpoint 상향 검토 결과**:
    - 프론트엔드 개발자가 메가메뉴 기준 `xl:`(1280px) 상향 검토를 제안했으나, 2단 네비로 변경됨에 따라 **기존 `lg:`(1024px) 유지**가 적절
    - 사유: 2단 네비는 5개 카테고리 텍스트 + 검색 + 테마 + 로그인이 1024px에 충분히 수용됨 (드롭다운 패널이 없으므로)

  - **태블릿(640~1023px) 특이 사항**:
    - 모바일과 동일하게 Sheet + BottomTabBar 사용
    - 태블릿에서는 BottomTabBar 탭 간격이 넓어져 터치 타겟이 더 여유로움 — 별도 대응 불필요

- **산출물**: 반응형 동작 정의 테이블
- **완료 기준**: 모든 breakpoint에서 각 UI 요소의 표시/숨김/동작이 명확

#### 3.1.3 미결 사항 및 개발자 재량 항목 정리

- **예상 시간**: 15분
- **선행 조건**: 3.1.1, 3.1.2 완료
- **작업 내용**:
  - UX 기획에서 확정하지 않고 개발자 재량에 맡기는 항목 명시:
    - 2단 서브 네비 전환 애니메이션 (있어도 되고 없어도 됨)
    - Sheet 열림/닫힘 애니메이션 속도 (현재 기본값 유지)
    - `ChartControls` props 리팩토링 여부 (기능 변경 아닌 코드 품질)
    - `app-header.tsx` Server/Client Component 분리 (2단 네비 기준으로 가능하지만 선택적)
  - PM과 논의 필요한 항목:
    - 광고 위치 이동 최종 승인
    - 브랜드 컬러 최종 선택
    - "홈" 링크 유지/제거 최종 결정
    - 히어로 섹션 필요성 (Phase 2 포함 여부)
- **산출물**: 미결/재량 항목 리스트
- **완료 기준**: 개발 착수 시 블로커가 되는 미결 항목이 0개
