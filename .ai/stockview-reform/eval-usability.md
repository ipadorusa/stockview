## 1. 사용성 평가 (30점)

### U-1. 학습 용이성 (Learnability) — 6/8점

**Playwright 실측 결과 (현재 UI):**
- 홈 → 종목 상세: 2클릭 (검색 아이콘 → 검색어 입력 → 결과 클릭) ✅
- 홈 → 시장: 1클릭 (상단 네비) ✅
- 홈 → 뉴스: 1클릭 (상단 네비) ✅
- 홈 → 관심종목: 1클릭 (상단/하단 네비) ✅
- 홈 → 배당 캘린더: **접근 불가** (네비 미노출, URL 직접 입력 필요) ❌
- 홈 → 종목 비교: **접근 불가** (네비 미노출) ❌
- 홈 → 투자 가이드: 서브내비에서 접근 가능 ✅

**개선안 평가:**
- **강점**: improve-strategy의 IA 재구성으로 3그룹(탐색/분석/내 투자) + 컨텍스트 허브 체계화. 현재 39개 중 ~30%가 미노출되는 문제를 근본적으로 해결
- **강점**: 바텀탭 5개(홈/탐색/검색/관심/MY) 재배치로 모바일 핵심 기능 접근성 대폭 개선
- **강점**: 종목 상세의 Floating Action Bar가 관심종목/비교/리포트에 대한 1차 접근성 제공
- **약점**: IA 재구성 시 "탐색" 라벨이 초보 사용자에게 직관적인지 검증 필요. "시장"이 더 명확할 수 있음
- **약점**: 종목 상세의 탭 확장(기존 4→6개: 차트/정보/뉴스/배당·실적/AI분석/유사종목)은 모바일에서 가로 스크롤 유발 가능성

**근거**: S1 시나리오 결과 — 현재 3단계로 관심종목 추가 가능하나, 관심종목 버튼이 헤더 우측에만 있어 발견성 낮음. 개선안의 FAB가 이를 해결

### U-2. 효율성 (Efficiency) — 6/7점

**Playwright 실측 결과 (현재 UI):**
- 검색 자동완성: ✅ 작동 (SearchCommand, 2글자 이상)
- 최근 검색/최근 본 종목: ❌ 없음 (H6 휴리스틱 위반)
- 종목 리스트에서 관심종목 원터치 추가: ❌ 불가 (종목 상세 진입 필요)
- 검색 결과에서 인라인 액션: ❌ 불가 (이동만 가능)
- 키보드 단축키 (Cmd+K): ✅ 작동

**개선안 평가:**
- **강점 (전략 3: 마이 대시보드)**: "최근 본 종목" localStorage 구현 + 홈/마이페이지 표시 — H6 위반 해결
- **강점 (전략 4: 제로 프릭션 비교)**: 글로벌 비교 슬롯 + FAB — 어디서든 비교 추가 가능
- **강점**: 검색 결과 인라인 액션(관심/비교) — 검색→액션 2단계 단축
- **강점**: 종목 상세 차트 프리셋(기본/추세/모멘텀/커스텀) — 선택 피로도 대폭 감소
- **약점**: 글로벌 비교 슬롯 상태가 세션 기반이라 새로고침 시 유실 가능. sessionStorage나 URL 상태 고려 필요

**근거**: S2 시나리오 결과 — 스크리너에서 비교까지 4단계. 개선안의 체크박스+비교 연결 및 글로벌 슬롯이 이를 2단계로 단축

### U-3. 오류 방지 & 복구 (Error Prevention) — 5/7점

**Playwright 실측 결과 (현재 UI):**
- 존재하지 않는 종목 코드: ✅ 에러 메시지 표시
- 비로그인 관심종목 접근: ✅ 로그인 리다이렉트
- 시세 업데이트 시간 표시: ✅ "기준" 시각 표시
- 빈 검색어 피드백: ❌ 없음 (2글자 미만 시 무반응)
- 비교 페이지 빈 상태 가이드: △ "비교해 보세요" 단일 메시지

**개선안 평가:**
- **강점**: improve-sub-pages의 비교 페이지 "인기 비교 프리셋" — 빈 상태를 유용한 콘텐츠로 전환
- **강점**: improve-core-pages의 마켓 페이지 설명 텍스트 → 첫 방문 투어 툴팁 — 오류가 아닌 맥락적 교육
- **약점**: 네트워크 오류, API 실패 시의 폴백 UI 설계가 개선안에 언급되지 않음
- **약점**: 빈 검색어 피드백 개선이 명시적으로 다뤄지지 않음
- **약점**: 포트폴리오 매수가/수량 입력 시 유효성 검증 설계 부재

**근거**: H5(오류 방지) 체크 결과 — 기본적인 에러 처리는 존재하나, 빈 상태/엣지 케이스 가이드가 부족. 개선안의 프리셋이 부분 해결

**근거**: 첫 테스트에서 13개 이슈 중 5개가 네비게이션 미노출 관련. 개선안의 IA 재구성과 Market Dashboard 통합이 핵심 해결책

## 2. 품질 교차 평가 (핵심 페이지)

> eval-designer-2의 교차 평가: 핵심 페이지(홈/주식상세/마켓)의 품질 관점

### Q-1. 비주얼 완성도 — 8/10점

**개선안 강점:**
- **Market Pulse Strip**: 풀 와이드 다크 배경 + 미니 스파크라인 인라인 — 기존 CompactIndexBar 대비 정보 밀도와 시각적 임팩트 모두 향상
- **Bento Grid**: 지수 카드 + CTA를 불규칙 그리드로 통합 — 시각적 리듬감과 계층 구조 명확
- **시장 온도계**: 지수 등락률을 프로그레스 바로 시각화 — 기존 숫자 나열 대비 직관적
- **Sticky Price Header**: 스크롤 시 가격 축약 트랜지션 — 마이크로 인터랙션의 정교함
- **Price Sparkle**: 가격 변동 shimmer 효과 — 데이터 신선도 감각적 전달

**개선안 약점:**
- Bento Grid 내 gradient accent 카드 4개가 과도한 색상 다양성 유발 가능 — 통일된 accent 파레트 필요
- 시장 온도계의 `-3%~+3%` 범위가 변동성 큰 날에 스케일 초과 시 처리 미명시

### Q-4. 반응형 설계 — 6/7점

**개선안 강점:**
- 홈: Market Pulse Strip의 모바일 자동 스크롤 티커, Bento Grid의 2×3 리플로우
- 주식 상세: Split Layout(데스크톱 2-column → 모바일 단일 컬럼) + 사이드 패널 데이터 인라인화
- 마켓: 4열 지수 → 모바일 2×2 그리드, 섹터 트리맵 터치 인터랙션

**개선안 약점:**
- 주식 상세 모바일에서 Floating Action Bar가 바텀탭과 겹치는 z-index 처리 필요 (bottom-16 명시됨)
- 마켓 페이지 트리맵이 모바일 390px에서 충분한 터치 영역 확보되는지 미검증

### 품질 교차 평가 소계 (부분): 14/17점 (Q-1 + Q-4만)

## 3. 태스크 시나리오 결과

### S1: "삼성전자 현재가 확인 + 관심종목 추가" (목표: 3단계)

| 구분 | 현재 UI | 개선안 |
|------|---------|--------|
| 단계 수 | 3단계 (검색→결과 클릭→관심종목 버튼) | 2~3단계 (검색→결과 인라인 관심 추가 OR FAB) |
| 현재가 표시 | ✅ 종목 상세에서 표시 | ✅ + 검색 결과에서도 가격 노출 |
| 관심종목 버튼 | 헤더 우측 (발견성 낮음) | FAB + 검색 인라인 (발견성 높음) |
| **성공 여부** | ⚠️ 단계는 충족, 발견성 미흡 | ✅ 단계 단축 + 발견성 개선 |
| **마찰 포인트** | 관심종목 버튼 위치 인지 필요 | 해소됨 (FAB가 항상 가시) |

### S2: "스크리너 → 골든크로스 → 비교" (목표: 4단계)

| 구분 | 현재 UI | 개선안 |
|------|---------|--------|
| 단계 수 | 4단계 (스크리너→시그널→URL 직접 비교) | 3단계 (스크리너→체크박스→비교) |
| 복합 시그널 | ❌ 단일 선택만 | ✅ 체크박스 복합 AND 조건 |
| 비교 연결 | ❌ 직접 경로 없음 | ✅ 체크→비교 버튼 |
| **성공 여부** | ❌ 비교까지 자연스러운 흐름 불가 | ✅ 원클릭 연결 |
| **마찰 포인트** | 스크리너→비교 단절 | 해소됨 (인라인 비교 추가) |

### S3: "배당 캘린더 US 필터" (목표: 2단계)

| 구분 | 현재 UI | 개선안 |
|------|---------|--------|
| 단계 수 | 2단계 (투자정보 드롭다운→배당) | 1~2단계 (Market Dashboard 필터 칩) |
| 네비 노출 | 드롭다운에 숨겨짐 | 탐색 탭→Market Dashboard 필터 |
| US 필터 | ✅ 존재 | ✅ 유지 |
| **성공 여부** | ⚠️ 도달 가능하나 발견 비용 높음 | ✅ Market Dashboard에서 통합 접근 |
| **마찰 포인트** | 드롭다운 메뉴 인지 필요 | 감소 (필터 칩 기반) |

### S4: "AAPL AI 리포트 → 뉴스" (목표: 탭 전환)

| 구분 | 현재 UI | 개선안 |
|------|---------|--------|
| 탭 구성 | 차트/정보/뉴스/이벤트 (4개) | 차트(상시) + 뉴스/이벤트/수급(3탭) + 사이드: AI분석 링크 |
| AI 탭 | ❌ 없음 (별도 /reports 페이지) | △ 사이드 패널에 "AI 분석 요청" 링크 (탭은 아님) |
| 뉴스 탭 | ✅ 존재 | ✅ 유지 |
| **성공 여부** | ❌ 페이지 이동 필요 | △ 사이드 링크로 접근 개선되나 탭 전환은 아님 |
| **마찰 포인트** | AI 리포트가 종목 상세와 분리 | 부분 해소 (링크 제공, 완전 통합은 미달) |

**평가 코멘트**: improve-strategy에서 "AI분석" 탭 추가를 제안했으나, improve-core-pages의 실제 와이어프레임에서는 사이드 패널 링크로 구현. 탭 내 인라인 AI 리포트 표시까지 갈 수 있다면 시나리오 완전 충족.

### S5: "모바일 ETF 탐색" (목표: 3탭)

| 구분 | 현재 UI | 개선안 |
|------|---------|--------|

| 하단탭 직접 접근 | ❌ ETF 전용 탭 없음 | △ "탐색" 탭에서 ETF 필터 칩 접근 |
| ETF 상세 | ✅ ETF 클릭→상세 | ✅ 유지 |
| **성공 여부** | ⚠️ 접근 가능하나 경로 비직관 | ✅ Market Dashboard 통합으로 개선 |
| **마찰 포인트** | ETF가 어디에 있는지 찾아야 함 | 감소 (탐색 대시보드에서 통합) |

## 4. 닐슨 10가지 휴리스틱 체크리스트

| 휴리스틱 | 현재 UI | 개선안 | 심각도 |
|----------|---------|--------|--------|
| H1 시스템 상태 가시성 | ○ 시세 "기준" 시각 표시 | ○ + Price Sparkle 애니메이션 | - |

## 7. 핵심 강점 TOP 5

1. **제로 프릭션 비교 (전략 4)**: 글로벌 비교 슬롯 + FAB + 검색 인라인 액션 — 현재 가장 큰 마찰점(스크리너→비교 단절)을 근본적으로 해결. 경쟁사 대비 명확한 차별점
2. **Market Dashboard 통합 탐색**: 단일 페이지에서 필터 칩으로 인기/상승/하락/ETF/배당/실적 모두 접근 — 정보 발견성 문제의 핵심 해결책
3. **뉴스-종목 크로스링크**: 뉴스 카드 종목 태그 + 종목 상세 뉴스 탭 — 현재 완전히 단절된 뉴스↔종목 연결 복원
4. **차트 프리셋 시스템**: 25개 토글 → 4개 프리셋(기본/추세/모멘텀/커스텀) — 초보자 진입장벽 대폭 하향, 고급자 기능 유지
5. **비교 페이지 승패 하이라이트**: 지표별 최적값 자동 판별 + 스코어보드 — 기존 금융앱의 flat 테이블 대비 명확한 차별화

## 8. 핵심 약점 TOP 5

1. **네트워크 오류/API 실패 폴백 설계 부재**: 크론 기반 데이터 수집이므로 데이터 지연/실패 시 사용자 안내가 중요하나 개선안에 미포함
2. **터치 타겟 크기 미준수**: 현재 UI에서 23개 요소가 44px 미만. 개선안에서 차트 컨트롤 축소, 필터 칩 도입 시 이 문제 악화 가능
3. **뉴스 상세 페이지 부재**: 뉴스→종목 태그는 추가되나, 뉴스 자체가 여전히 외부 링크. 사용자 이탈 포인트 해소 안 됨
4. **S4 시나리오 미완충족**: AI 리포트가 종목 상세 탭으로 완전 통합되지 않음 (사이드 링크만). 전략에서 제안한 "AI분석" 탭이 와이어프레임에 미반영
5. **글로벌 비교 슬롯 상태 영속성**: 세션 기반이라 새로고침/탭 닫기 시 유실. URL 파라미터 또는 sessionStorage 보완 필요

## 10. 팀 토론 안건 (Task #7용)

### 토론 안건 1: "탐색" vs "시장" 네비게이션 라벨
- 개선안은 바텀탭 "시장"을 "탐색"으로 변경 제안
- 초보 투자자 타겟에서 "탐색"이 직관적인지? "시장"이 더 구체적이지 않은지?
- 참고: 토스증권은 "홈/종목/공시/잔고/더보기", 삼성증권은 "홈/관심/주문/자산/메뉴"

### 토론 안건 2: AI 리포트 통합 깊이
- 전략 문서는 "AI분석" 탭 추가를 제안했으나 와이어프레임은 사이드 링크만 구현
- 종목 상세 탭 수 증가(4→6)의 모바일 UX 트레이드오프
- 대안: AI 요약을 "정보" 탭 내 섹션으로 삽입? 또는 조건부 탭(리포트가 있는 종목만)?

### 토론 안건 3: 뉴스 내부화 여부
- 현재 뉴스는 100% 외부 링크 (Google News RSS, Yahoo Finance RSS)
- 종목 태그 추가는 확정되나, 뉴스 자체의 사용자 이탈은 해소 안 됨
- 내부 뉴스 요약 페이지 도입 여부 (저작권, 구현 복잡도 vs UX 개선)

### 토론 안건 4: 글로벌 비교 슬롯의 UI 위치
- FAB(화면 우하단)이 콘텐츠를 가리지 않는지?
- 모바일에서 FAB + 바텀탭 + Floating Action Bar 삼중 겹침 리스크
- 대안: FAB 대신 상단 알림 바 또는 사이드 드로어?
