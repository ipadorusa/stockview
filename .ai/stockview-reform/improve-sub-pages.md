## 1. 뉴스 페이지 (/news)

### 현재 분석
- **구조**: 카테고리 탭(전체/한국/미국/산업/경제) + 감성 필터(긍정/부정/중립) + 검색 + 페이지네이션
- **NewsCard**: compact 형태 — 썸네일(80px) + 카테고리 Badge + 감성 Badge + 제목 2줄 + 시간
- **문제점**:
  1. 카드가 수직 1열 나열 — 정보 밀도 낮음, 스크롤 과다
  2. 감성 필터가 카테고리 탭과 분리되어 2단계 필터링이 인지적 부담
  3. 뉴스와 종목의 연결이 없음 — 뉴스를 읽고 관련 종목으로 이동하려면 직접 검색해야 함
  4. 검색이 우측 상단에 치우쳐 모바일에서 발견성 떨어짐
  5. 페이지네이션이 "이전/다음"만 — 특정 페이지로 점프 불가

### 개선안

### 독창적 아이디어

#### 1-A. "뉴스-종목 실시간 연결 사이드바"
- 우측 사이드바에 현재 보고 있는 뉴스의 관련 종목을 자동 표시
- DB의 `newsStocks` 관계를 활용하여, 뉴스 카드에 hover/tap 시 관련 종목 미니 카드 노출
- 종목 카드 클릭 시 `/stock/{ticker}`로 이동 — 뉴스 → 종목 전환이 원클릭

#### 1-B. "뉴스 히트맵 타임라인"
- 하루 뉴스를 시간대별로 시각화 (x축: 시간, y축: 카테고리, 색상: 감성)
- 특정 시간대에 부정 뉴스가 몰리면 빨간 블록으로 직관적 파악
- 블록 클릭 → 해당 시간대 뉴스 필터링

#### 1-C. 모바일 최적화
- 필터를 bottom sheet로 전환 (필터 아이콘 탭 → 슬라이드업)
- 검색바를 상단 고정 sticky로 변경
- 페이지네이션을 무한 스크롤 + "더 보기" 버튼 하이브리드로 전환

## 2. 종목 비교 (/compare)

### 현재 분석
- **구조**: 최대 4슬롯 검색 → 비교 테이블(8개 지표) → 가격 오버레이 차트 → 펀더멘탈 비교
- **문제점**:
  1. 빈 상태에서 가이드가 부족 — "비교해 보세요"만으로는 사용 동기 부족
  2. 비교 테이블이 flat하여 어떤 종목이 우위인지 한눈에 파악 어려움
  3. 검색 슬롯이 단순 리스트 — 드래그 재정렬, 프리셋 없음
  4. 최대 4개 제한이 명확하지 않음

### 개선안

```
┌─────────────────────────────────────────────────┐
│  종목 비교                                       │
│  "투자 판단에 필요한 핵심 지표를 나란히 비교"     │
├─────────────────────────────────────────────────┤
│  💡 인기 비교 프리셋:                            │
│  [삼성전자 vs SK하이닉스] [테슬라 vs 엔비디아]   │
│  [KODEX 200 vs TIGER 200]                       │
├─────────────────────────────────────────────────┤
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ [+ 추가]  │

│  │          │ 삼성전자  │ SK하이닉스│             │
│  ├──────────┼──────────┼──────────┤             │
│  │ 현재가    │ 72,000   │ 185,000  │             │
│  │ 등락률    │ ⬛+2.3%  │ ⬛-1.1%  │  ← 바 차트 │
│  │ 시가총액  │ 430조    │ 127조    │  ← 비율 바  │

│  └──────────┴──────────┴──────────┘             │
│                                                  │
│  📈 가격 추이 비교 (정규화 차트)                  │

│  ┌──────────────────────────────────┐           │
│  │   ╱╲     정규화(%)               │           │
│  │  ╱  ╲   ── 삼성전자              │           │
│  │ ╱    ╲  ── SK하이닉스            │           │
│  └──────────────────────────────────┘           │
│                                                  │
│  [📋 비교 결과 공유] [💾 비교 저장]               │
└─────────────────────────────────────────────────┘
```

### 독창적 아이디어

#### 2-A. "승패 하이라이트 시스템"
- 각 지표에서 가장 우수한 값에 ★ 마크 + 배경색 강조
- PER은 낮을수록, ROE/배당률은 높을수록 우수 — 지표별 방향성 자동 판별
- 하단에 "종합 우위: 삼성전자 3 vs SK하이닉스 4" 형태의 스코어보드

#### 2-B. "인기 비교 프리셋"
- 같은 섹터 내 경쟁 종목을 프리셋으로 제공 (API에서 동일 섹터 조회)
- 사용자 비교 기록을 localStorage에 저장하여 "최근 비교" 표시
- URL 쿼리 파라미터로 비교 공유 가능: `/compare?stocks=005930,000660`

#### 2-C. "레이더 차트 오버레이"
- 핵심 5개 지표(PER, PBR, ROE, 배당률, 시총)를 레이더 차트로 시각화
- 2~4개 종목의 레이더가 겹쳐져 형태 비교 가능
- 어떤 종목이 "밸류", "성장", "배당" 성향인지 시각적으로 파악

## 3. 스크리너 (/screener)

### 현재 분석
- **구조**: 마켓 토글(한국/미국) → 시그널 5종 선택 → 결과 테이블(종목명, 현재가, 등락률, 거래량)
- **문제점**:
  1. 시그널을 하나만 선택 가능 — "골든크로스 + RSI 과매도" 복합 조건 불가
  2. 결과 테이블의 정렬 기능 없음 — 등락률순, 거래량순 정렬 불가
  3. SEO 설명 텍스트가 너무 김 — 사용자가 도구까지 스크롤해야 함
  4. 시그널 버튼이 label+desc 구조라 모바일에서 공간 낭비
  5. 결과에서 차트 미리보기 없음 — 시그널이 실제 차트에서 어떻게 나타나는지 확인 불가

### 개선안

```
┌─────────────────────────────────────────────────┐
│  스크리너                                        │
│  기술적 분석 기반 종목 선별                       │
```

### 독창적 아이디어

#### 3-A. "복합 시그널 필터링"
- 체크박스로 복수 시그널 선택 → AND 조건으로 교집합 결과 표시
- "2개 이상 시그널 충족" 종목에 🔥 마크 — 강한 신호 강조
- 매칭된 시그널을 종목별로 Badge 표시

#### 3-B. "인라인 미니 차트"
- 각 결과 행에 최근 20일 미니 스파크라인 차트 표시
- 시그널 발생 지점을 차트에 마커로 표시 (골든크로스 지점 등)
- 행 클릭/확장 시 더 큰 차트 프리뷰 노출 (프로그레시브 디스클로저)

#### 3-C. "정렬 + 추가 필터"
- 컬럼 헤더 클릭으로 정렬 토글 (등락률, 거래량, 시가총액)
- 시가총액 범위 필터 (대형주/중형주/소형주)
- 결과를 CSV/클립보드로 내보내기 기능

## 4. ETF (/etf)

### 현재 분석
- **구조**: 탭(한국/미국) → StockRow 리스트 30개씩 (거래대금 기준 정렬)
- **문제점**:
  1. 단순 리스트 뷰 — ETF의 핵심인 "테마/섹터 분류"가 없음
  2. 정렬 기준이 거래대금 고정 — 등락률, 수익률 기준 정렬 불가
  3. SEO 설명이 페이지 상단을 차지 — 리턴 유저에게 불필요
  4. ETF의 핵심 정보(수수료, 추종지수, AUM)가 StockRow에 없음
  5. 카테고리(레버리지, 인버스, 섹터, 배당 등) 필터 없음

### 개선안

```
┌─────────────────────────────────────────────────┐
│  ETF                                             │
│  [한국 ●] [미국 ○]        검색: [___________]   │
├─────────────────────────────────────────────────┤
│  카테고리:                                       │
│  [전체] [시장지수] [섹터] [레버리지] [인버스]     │
│  [배당] [채권] [원자재] [해외]                    │
```

### 독창적 아이디어

#### 4-A. "ETF 카테고리 그리드 뷰"
- 리스트 뷰 대신 카테고리별 그리드 카드 뷰 옵션 제공
- 각 카드: ETF명 + 미니 스파크라인 + 등락률 + 거래대금
- 카테고리 헤더에 해당 카테고리 평균 수익률 표시

#### 4-B. "ETF 비교 바로가기"
- 각 ETF에 체크박스 → "선택한 ETF 비교하기" 버튼 → `/compare`로 이동
- 같은 테마 ETF 간 수수료/수익률 비교를 원클릭으로

#### 4-C. "ETF 설명 아코디언"
- 처음 방문 시 설명 펼쳐짐, 재방문 시 접혀있음 (localStorage)
- SEO용 콘텐츠는 유지하되 리턴 유저의 스크롤 부담 제거

## 5. 리포트 (/reports)

### 현재 분석
- **구조**: 탭(AI 리포트/분석 요청) → 마켓 필터(전체/KR/US) → 테이블(날짜, 종목, 시그널, 의견, 요약) + 페이지네이션
- **reports/request**: 종목 검색 → 분석 요청 (로그인 필요, 일 3건 제한)
- **문제점**:
  1. 테이블이 밀도 높지만 스캔하기 어려움 — verdict별 시각적 구분 부족
  2. 시그널 필터 없음 — 골든크로스 리포트만 보기 불가
  3. 요약이 md:table-cell로 데스크톱에서만 보임 — 모바일에서 리포트 핵심 정보 손실
  4. 분석 요청과 리포트 목록 사이 연결이 약함
  5. 리포트 카드 뷰 옵션 없음 — 테이블만 제공

### 개선안

```
┌─────────────────────────────────────────────────┐
│  AI 종목 분석 리포트                             │
│  AI가 기술적 시그널을 감지한 종목을 분석합니다    │
├─────────────────────────────────────────────────┤
```

## 6. 섹터 (/sectors)

### 현재 분석
- **구조**: 탭(한국/미국) → 3열 그리드 카드 (섹터명 + 종목수 Badge)
- **문제점**:
  1. 섹터별 시장 현황(평균 등락률, 거래대금 등) 정보 없음 — 단순 카탈로그

#### 6-A. "섹터 퍼포먼스 히트맵"

- "이 섹터의 대장주" 1~2개 표시 (시총 기준 최상위)

#### 6-C. "섹터 비교 모드"
- 2개 섹터 선택 → 사이드바이사이드 비교
- 섹터 내 상위 종목 리스트 + 평균 PER/PBR 비교

## 7. 관심종목 (/watchlist)

## 8. 페이지 간 연결성 강화 전략

### 현재 문제
각 보조 페이지가 독립적인 "섬"처럼 존재하며, 페이지 간 자연스러운 이동 경로가 부족함.
사용자가 "뉴스 → 종목 → 비교 → 관심종목 추가"와 같은 투자 리서치 플로우를 자연스럽게 수행하기 어려움.

### 연결성 강화 전략

#### 8-A. "컨텍스트 액션 바" (모든 보조 페이지 공통)
```
┌─────────────────────────────────────────────────┐
│  🔗 관련 기능:                                   │
│  [📰 뉴스 보기] [📊 비교하기] [🔍 스크리너]      │
│  [📋 리포트] [⭐ 관심종목 추가]                   │
└─────────────────────────────────────────────────┘
```
- 종목 상세 페이지에 현재 종목 기준의 컨텍스트 액션 표시
- "이 종목의 뉴스", "이 종목 비교하기", "이 종목 분석 요청" 등

#### 8-B. 크로스 페이지 딥링크 체계
| 출발 | 도착 | 연결 방식 |
|------|------|-----------|
| 뉴스 카드 | 종목 상세 | 관련 종목 태그 클릭 |
| 스크리너 결과 | 리포트 | "AI 분석 보기" 링크 (리포트 있는 경우) |
| 스크리너 결과 | 비교 | 체크박스 선택 → 비교 |
| ETF 목록 | 섹터 | ETF 카테고리 → 관련 섹터 링크 |
| 섹터 상세 | 스크리너 | "이 섹터 종목 스크리닝" 버튼 |
| 관심종목 | 비교 | 체크 선택 → 비교 이동 |
| 리포트 상세 | 종목 상세 | 종목명 클릭 |
| 리포트 상세 | 뉴스 | "관련 뉴스" 섹션 |

#### 8-C. "투자 리서치 플로우" 가이드
- 첫 방문 사용자에게 온보딩 투어 제공:
  1. "뉴스에서 관심 종목 발견" → 2. "스크리너로 기술적 확인" → 3. "리포트로 AI 분석" → 4. "비교로 경쟁사 대비" → 5. "관심종목에 추가"
- 각 페이지 하단에 "다음 단계 추천" 배너

#### 8-D. URL 기반 상태 공유
- 모든 필터/정렬 상태를 URL 쿼리에 반영
- `/screener?market=KR&signal=golden_cross,macd_cross&sort=changePercent`
- `/compare?stocks=005930,000660,TSLA`
- 사용자가 비교 결과나 스크리너 결과를 URL로 공유 가능

#### 8-E. "최근 본 종목" 글로벌 위젯
- 모든 페이지 하단 또는 사이드바에 최근 본 종목 5개 표시
- localStorage 기반, 로그인 불필요
- 빠른 재방문 경로 제공

## 9. 전략 정합성 반영 (improve-planner 전략 문서 기반)

improve-planner의 `improve-strategy.md` 4대 차별화 전략과의 정합성을 확인하고, 추가 반영 사항을 기술합니다.

### 전략 1: "투자 여정 퍼널" — 초보자 교육 인라인 통합
- **스크리너(3)**: 각 시그널 버튼에 "이 시그널의 의미" 툴팁/컨텍스트 카드 추가 필요 (전략 2.4-1)
- **ETF(4)**: 설명 아코디언(4-C)이 이 전략에 직접 대응 — 초보자에게 펼침, 중급자에게 접힘
- **리포트(5)**: verdict 뱃지에 "매수 관심이란?" 원터치 교육 툴팁 추가

### 전략 2: "크로스마켓 인사이트" — KR↔US 연결
- **종목 비교(2)**: KR↔US 교차 비교 시 환율 자동 환산 행 추가 (예: "원화 환산 시가총액")
- **뉴스(1)**: 뉴스 카드 종목 태그에 "관련 한국/미국 종목" 크로스 태그 (DB newsStocks 활용)
- **섹터(6)**: 섹터 카드에 "대응 US 섹터" 링크 (예: 반도체 KR ↔ Semiconductors US)

### 전략 3: "마이 대시보드" — 개인화 허브
- **관심종목(7)**: 미니 대시보드 전환 — 각 종목에 1일 스파크라인 추가 (7-B 확장)
- **관심종목(7)**: 포트폴리오를 독립 /portfolio로 분리 검토 (전략 2.1 IA 재구성)

### 전략 4: "제로 프릭션 비교" — 글로벌 비교 슬롯
- **비교(2)**: URL 쿼리 파라미터 공유(2-B)가 글로벌 비교 슬롯의 기반
- **관심종목(7)**: 체크→비교(7-A)가 이 전략의 핵심 구현체
- **스크리너(3)**: 결과 행에 "비교 추가" 아이콘 버튼 추가 → 글로벌 슬롯에 적재
- **ETF(4)**: ETF 체크박스 비교(4-B)도 동일 글로벌 슬롯 활용
