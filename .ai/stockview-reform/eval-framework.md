## 1. 평가 목적 및 범위

### 1.1 평가 목적

StockView 개선안이 **기존 금융 정보 서비스 대비 차별화되고, 높은 비주얼 품질을 유지하며, 초보 투자자가 쉽게 사용할 수 있는지**를 객관적으로 검증한다.

평가는 "주관이 아닌 근거 기반"으로 수행하며, 각 항목별 구체적 지표와 체크리스트를 활용한다.

### 1.2 평가 범위

| 구분 | 범위 |
|------|------|
| **전략 문서** | `improve-strategy.md` — 개선 방향성, 디자인 원칙, 정보 구조 |
| **핵심 페이지** | `improve-core-pages.md` — 홈(`/`), 주식 상세(`/stock/[ticker]`), 마켓(`/market`) |
| **보조 페이지** | `improve-sub-pages.md` — 뉴스(`/news`), 비교(`/compare`), 스크리너(`/screener`), ETF(`/etf`), AI 리포트(`/reports`) |
| **현재 구현** | Playwright를 활용한 현재 UI 캡처 및 비교 분석 |

### 1.3 평가 기준 출처

- Nielsen Norman Group 10 Usability Heuristics
- HEART Framework (Google)
- 금융 앱 UX 벤치마크 (토스증권, 삼성증권, Yahoo Finance, TradingView)
- WCAG 2.1 AA 접근성 기준 (색상 대비 등)

## 2. 평가 기준 (총 100점)

### 2.1 독창성 (30점)

개선안이 기존 금융 정보 서비스와 비교하여 **얼마나 참신하고 차별화된 접근**을 제시하는지 평가한다.

| # | 세부 항목 | 배점 | 평가 지표 |
|---|----------|------|----------|
| O-1 | **정보 구조의 혁신성** | 8점 | 기존 금융앱의 탭/리스트 패턴을 넘어서는 새로운 정보 배치, 네비게이션 구조가 있는가? 사용자 멘탈 모델에 부합하면서도 독창적인 정보 계층이 존재하는가? |
| O-2 | **시각 표현의 차별화** | 8점 | 주가 차트, 지표, 비교 등에서 기존 금융앱과 다른 시각적 표현을 제안하는가? 데이터 시각화에 새로운 접근법이 있는가? 한국식 색상 관례(빨강=상승, 파랑=하락)를 유지하면서도 독자적 비주얼 아이덴티티가 있는가? |
| O-3 | **인터랙션 디자인 참신성** | 7점 | 사용자 상호작용에서 기존 앱과 다른 패턴을 제안하는가? (예: 종목 비교 방식, 관심종목 관리, 스크리너 필터링 등) |
| O-4 | **초보 투자자 특화 차별화** | 7점 | 타겟 사용자(초보 투자자)를 위한 독자적 기능이나 접근법이 있는가? (예: 교육 통합, 용어 설명, 가이드 연동 등) 기존 금융앱이 간과하는 영역을 다루는가? |

**독창성 평가 시 참조할 개선 전략 4축:**

| 전략 축 | 핵심 아이디어 | 관련 항목 |
|---------|-------------|----------|
| 투자 여정 퍼널 | 초보→중급 성장 경로, 교육 통합형 플랫폼, 투자 레벨 게이미피케이션 | O-1, O-4 |
| 크로스마켓 인사이트 | KR↔US 상관관계 위젯, 환율 자동 환산, 섹터별 대응 종목 매핑 | O-1, O-2 |
| 마이 대시보드 | 관심종목→미니 대시보드, 포트폴리오 수익률 시계열, 개인화 홈 | O-2, O-3 |
| 제로 프릭션 비교 | 글로벌 비교 슬롯, 어디서든 비교 추가, FAB | O-3 |

**독창성 평가 방법론:**
- 벤치마크 비교: 토스증권, Yahoo Finance, TradingView, Google Finance와 각 항목을 대조
- 0-2점 = 기존과 동일/유사, 3-5점 = 부분 차별화, 6-8점 = 명확한 독창성
- 전략 4축이 페이지 개선안에 구체적으로 반영되었는지 확인 (전략↔실행 정합성)

### 2.2 품질 (40점)

개선안의 **비주얼 완성도, 디자인 시스템 일관성, 레이아웃 품질**을 평가한다.

| # | 세부 항목 | 배점 | 평가 지표 |
|---|----------|------|----------|
| Q-1 | **비주얼 완성도** | 10점 | 타이포그래피 체계, 색상 팔레트 활용, 아이콘/이미지 사용이 정교한가? 시각적 노이즈 없이 정보가 명확히 전달되는가? |
| Q-2 | **레이아웃 & 그리드** | 8점 | 그리드 시스템이 일관되게 적용되었는가? 여백(spacing)이 체계적인가? 콘텐츠 밀도가 적절한가? (금융앱 특성상 정보 밀도와 가독성의 균형) |
| Q-3 | **디자인 시스템 준수** | 8점 | shadcn/ui + Tailwind CSS 4 기반 디자인 토큰이 일관되게 사용되는가? 컴포넌트 재사용 패턴이 명확한가? base-nova 스타일과의 일관성은? |
| Q-4 | **반응형 설계** | 7점 | 데스크톱-태블릿-모바일 전환이 자연스러운가? 모바일 우선(39개 페이지 중 모바일 비율 고려)으로 설계되었는가? BottomTabBar ↔ 데스크톱 헤더 전환이 매끄러운가? |
| Q-5 | **색상 접근성 & 대비** | 7점 | 주식 색상(빨강/파랑)이 색각이상자에게도 구분 가능한가? WCAG AA 대비비 4.5:1 충족 여부. 다크/라이트 테마 전환 시 가독성 유지 여부. |

**품질 평가 방법론:**
- Playwright 캡처를 통한 현재 UI vs 개선안 비주얼 비교
- 디자인 토큰 일관성 검증 (색상, 간격, 폰트 사이즈 체계)
- 접근성: 색상 대비비 계산, 터치 타겟 크기(최소 44x44px) 확인

### 2.3 사용성 (30점)

개선안이 **사용자가 핵심 태스크를 효율적으로 수행**할 수 있도록 설계되었는지 평가한다.

| # | 세부 항목 | 배점 | 평가 지표 |
|---|----------|------|----------|
| U-1 | **학습 용이성 (Learnability)** | 8점 | 첫 방문 사용자가 주요 기능을 직관적으로 이해할 수 있는가? 39개 페이지 중 핵심 5개(홈, 시장, 종목상세, 뉴스, 관심종목)에 3클릭 이내 도달 가능한가? 네비게이션 라벨이 명확한가? |
| U-2 | **효율성 (Efficiency)** | 7점 | 반복 사용자를 위한 단축 경로가 있는가? (즐겨찾기, 최근 본 종목, 검색 자동완성 등) 페이지 간 이동의 마찰이 최소화되었는가? |
| U-3 | **오류 방지 & 복구 (Error Prevention)** | 7점 | 빈 상태(empty state)에 대한 가이드가 있는가? 잘못된 종목 코드 입력 시 피드백이 있는가? 네트워크 오류, 데이터 로딩 실패 시 대응이 설계되었는가? |
| U-4 | **정보 발견성 (Discoverability)** | 8점 | 기존에 발견 불가했던 5개 페이지(배당, 실적, 섹터, 비교, 가이드)의 노출이 개선되었는가? 사용자 여정(탐색→분석→판단→추적) 흐름이 자연스러운가? 크로스링킹이 효과적인가? |

**사용성 평가 시 참조할 핵심 흐름 최적화안:**

| 흐름 | 현재 문제 | 개선안 | 관련 항목 |
|------|----------|--------|----------|
| 종목 발견→분석→액션 | 분석/액션이 분리된 페이지 | Floating Action Bar (관심/비교/리포트) | U-2, U-4 |
| 시장 탐색→종목 발견 | /market에서 상승/하락 TOP 5만 표시 | 필터 칩 기반 통합 탐색 대시보드 | U-1, U-4 |
| 검색→즉시 액션 | 종목 선택→상세페이지로만 이동 | 검색 결과 인라인 액션 (관심/비교) | U-2 |
| 뉴스→종목 연결 | 뉴스가 외부 링크만 제공 | 뉴스 카드 종목 태그 | U-4 |

**사용성 평가 방법론:**
- 닐슨 10가지 휴리스틱 체크리스트 기반 검토
- 태스크 시나리오 기반 워크스루 (아래 3.2 참고)
- HEART 프레임워크 신호(Happiness, Engagement, Adoption, Retention, Task success) 매핑
- 위 4개 핵심 흐름의 마찰점 해소 여부를 실측 기준으로 검증

### 3.1 Playwright 활용 방법

#### 3.1.1 현재 UI 캡처 (eval-designer-1 담당)

```typescript
// 캡처 대상 페이지 (8개 핵심 페이지)
const pages = [
  { url: '/', name: 'home' },
  { url: '/market', name: 'market' },
  { url: '/stock/005930', name: 'stock-detail-kr' },    // 삼성전자
  { url: '/stock/AAPL', name: 'stock-detail-us' },      // Apple
  { url: '/news', name: 'news' },
  { url: '/screener', name: 'screener' },
  { url: '/compare', name: 'compare' },
  { url: '/etf', name: 'etf' },
];

// 뷰포트 3종 (데스크톱, 태블릿, 모바일)
const viewports = [
  { width: 1440, height: 900, name: 'desktop' },
  { width: 768, height: 1024, name: 'tablet' },
  { width: 375, height: 812, name: 'mobile' },
];
```

#### 3.1.2 캡처 시 확인 사항

| 항목 | 확인 방법 |
|------|----------|
| **시각적 일관성** | 동일 페이지의 3뷰포트 스크린샷 나란히 비교 |
| **색상 접근성** | 스크린샷에서 주가 표시 영역의 빨강/파랑 대비비 확인 |
| **정보 밀도** | 첫 화면(Above the fold)에 노출되는 정보 항목 수 카운트 |
| **네비게이션 도달성** | 메뉴 구조별 클릭 횟수 측정 |
| **빈 상태 처리** | 관심종목 0개, 검색 결과 0건 등 엣지 케이스 캡처 |
| **로딩 상태** | Skeleton/Spinner 표시 여부 및 일관성 |

### 3.2 휴리스틱 평가 체크리스트

닐슨 10가지 휴리스틱을 StockView 맥락에 맞게 구체화한 체크리스트이다.

| # | 휴리스틱 | StockView 맥락 적용 | 관련 평가 항목 |
|---|---------|---------------------|---------------|
| H1 | **시스템 상태의 가시성** | 시세 업데이트 시간 표시 여부 (cron 기반이므로 "장마감 기준" 등 명시 필요), 로딩 중 스켈레톤 | Q-1, U-3 |

| H7 | **유연성과 효율성** | 파워 유저 단축 기능 (키보드 검색, 빠른 관심종목 추가) | U-2 |

| H9 | **오류 인식/진단/복구** | API 오류 시 사용자 안내, 데이터 없음 상태 메시지 | U-3 |

## 5. 개선안 핵심 제안 요약 (평가 시 참조)

### 5.1 전략 (`improve-strategy.md`)
- IA 재구성: 4그룹 → 3그룹 + 컨텍스트 허브
- 4대 차별화 축: 투자 여정 퍼널, 크로스마켓 인사이트, 마이 대시보드, 제로 프릭션 비교
- 핵심 흐름 4개: 종목 발견→분석→액션, 시장 탐색→종목 발견, 검색→즉시 액션, 뉴스→종목 연결
- 네비게이션 혁신: 바텀탭 재배치(탐색 중앙), 서브내비 강화, 종목상세 탭 확장
