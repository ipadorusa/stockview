# 핵심 페이지 UI 개선안

> 작성: improve-designer-1 | 2026-03-28
> 대상: 홈(/), 주식 상세(/stock/[ticker]), 마켓(/market)
> 전략 참조: improve-planner의 `improve-strategy.md` (IA 재구성 + 4대 차별화 전략)

### 전략 연계 요약

improve-planner의 4대 차별화 전략을 각 페이지에 다음과 같이 반영:

| 전략 | 홈 | 주식 상세 | 마켓 |
|---|---|---|---|
| 투자 여정 퍼널 (인라인 교육) | 시장 온도계에 초보 가이드 툴팁 | 기술적 요약 카드에 해석 텍스트 | 섹터 트리맵에 "이 섹터란?" 툴팁 |
| 크로스마켓 인사이트 | Market Pulse Strip에 KR↔US 상관 뱃지 | 유사 종목에 해외 대응 종목 포함 | 4열 지수 동시 비교 |
| 마이 대시보드 | "최근 본 종목" 섹션 추가 | — (Sticky Header에 관심종목 버튼) | — |
| 제로 프릭션 비교 | — | Floating Action Bar (비교 슬롯) | 종목 카드에 비교 추가 버튼 |

## 1. 홈 페이지 (/)

### 현재 분석

**레이아웃 구조:**
- HeroSection (첫 방문 비로그인 사용자만 노출) → h1 → CompactIndexBar → 주요 지수(IndexGroups) → 환율 → 퀵링크(4개 그리드) → 2-column(인기종목 + 뉴스) → AdSlot
- `PageContainer`는 max-w-screen-xl, px-4~px-8 수준의 단순한 중앙 정렬

**문제점:**
1. **정보 계층 부재**: h1("한국/미국 주식 시세")이 HeroSection 바로 아래에서 맥락 없이 등장. 지수·환율·인기종목·뉴스가 동일한 시각적 무게로 나열됨
2. **CompactIndexBar의 중복**: 지수 정보가 CompactIndexBar와 IndexGroups에서 이중 표시
3. **퀵링크의 약한 존재감**: 4개 카드가 작은 아이콘+텍스트로 스크리너/AI리포트 같은 핵심 기능의 가치를 전달하지 못함
4. **모바일에서 2-column 레이아웃 붕괴**: lg:grid-cols-2가 모바일에서 단순 스택으로 바뀌며 뉴스가 인기종목 아래로 밀림
5. **비로그인 배너 2중화**: HeroSection + "주식 투자가 처음이신가요?" 배너가 동시 노출 가능

**컴포넌트 의존 관계:**
```
page.tsx
├── HeroSection (client, localStorage 기반 노출)
├── CompactIndexBar (server, 지수+USD 환율)
├── IndexGroups → IndexCard × 4 + IndexSparkline
├── IndexCard × 5 (환율)
├── QuickLinkGrid → QuickLinkCard × 4
├── PopularStocksTabs → StockRow (client, Tabs)
└── LatestNewsSection → NewsCard × 4 (server, Suspense)
```

### 독창적 아이디어

#### 1. "Market Pulse Strip" — 실시간 감성의 지수 티커
- CompactIndexBar를 대체. **풀 와이드 다크 배경**에 지수·환율을 한 줄로 배치
- 모바일에서 CSS `marquee` 대신 `@keyframes translate` 애니메이션으로 자동 수평 스크롤
- 각 지수에 **미니 스파크라인(20px 높이)**을 인라인으로 삽입해 추세 직관성 강화
- 다크모드에서는 `backdrop-blur-xl` + 반투명 그래디언트로 글래스모피즘 효과
- **크로스마켓 상관 뱃지**: KR/US 지수 사이에 상관관계 아이콘 표시 (↗↗ 동반 상승, ↗↘ 역행 등)

#### 2. "Bento Grid" — 지수 카드 + CTA를 결합한 불규칙 그리드
- 기존의 IndexGroups(지수 4개) + QuickLinkGrid(4개)를 하나의 **벤토 그리드**로 통합
- 좌측 큰 카드(2×2): 대표 지수(KOSPI, S&P) + 일봉 미니차트
- 우측 작은 카드(1×1): 스크리너, AI리포트, 종목비교, 배당캘린더 — 각각 **고유 gradient accent 배경**
- 호버 시 카드가 `scale(1.02)` + `box-shadow` 상승하는 마이크로 인터랙션
- 모바일에서는 2×3 그리드로 자연스럽게 리플로우

#### 3. "시장 온도계" — 지수 강도를 프로그레스 바로 시각화
- 환율 섹션을 독립 섹션에서 분리하고, 지수·환율을 **수평 바 차트** 형태로 통합
- 전일 대비 등락률을 `-3%~+3%` 범위의 프로그레스 바로 표현
- 상승 = 빨강 그래디언트, 하락 = 파랑 그래디언트 (한국식 컨벤션 유지)
- 환율은 하단에 작은 뱃지로 축약 (현재 5개 카드 → 인라인 뱃지 3개로 정보 밀도 향상)

#### 4. "최근 본 종목" 섹션 (improve-planner 전략 반영)
- 인기 종목 섹션 위에 **"최근 본 종목"** 수평 스크롤 카드 배치
- `localStorage`에 최근 방문한 종목 ticker 저장 (최대 10개)
- 각 카드: 종목명 + 현재가 + 미니 스파크라인 (48×24px)
- 로그인 사용자: 서버 저장으로 디바이스 간 동기화
- 비로그인 사용자: localStorage만 사용
- **수평 스크롤 + scroll-snap** → 모바일에서 자연스러운 스와이프

#### 5. 인기종목에 "미니 스파크라인" 추가
- 현재 StockRow는 가격+등락률만 표시. 각 행 우측에 **40×20px SVG 스파크라인** 삽입
- 7일 종가 데이터 기반, 상승=빨강, 하락=파랑 선으로 추세 직관 표현
- 기존 금융앱(네이버 증권, 토스증권)에서도 제공하지 않는 **인라인 추세 차트**

### 컴포넌트 변경 사항

| 현재 컴포넌트 | 변경 | 설명 |
|---|---|---|
| `CompactIndexBar` | **대체** → `MarketPulseStrip` | 풀와이드 다크 배경 + 자동 스크롤 티커 |
| `IndexGroups` + `QuickLinkGrid` | **병합** → `BentoGrid` | 지수카드 + CTA를 불규칙 그리드로 통합 |
| `IndexCard` (환율 5개) | **축약** → `MarketThermometer` | 환율을 인라인 뱃지로, 지수를 바 차트로 |
| `HeroSection` | **간소화** | 첫 방문 배너를 `MarketPulseStrip` 하단 슬라이드로 통합 |
| `PopularStocksTabs` → `StockRow` | **확장** | 각 행에 미니 스파크라인 SVG 추가 |
| 비로그인 배너 (page.tsx 인라인) | **제거** | HeroSection과 중복, MarketPulseStrip으로 대체 |

## 2. 주식 상세 페이지 (/stock/[ticker])

### 현재 분석

**레이아웃 구조:**
- Breadcrumb → AdSlot → StockTabs (client component)
- StockTabs 내부: 헤더(종목명+배지+관심종목) → PriceDisplay → 탭(차트/정보/수급/뉴스/이벤트)
- 차트 탭이 기본. StockChart는 600+줄의 모놀리식 컴포넌트 (14개 기술 지표, 10개 ref)

**문제점:**
1. **가격 정보와 차트의 분리**: PriceDisplay(가격)가 탭 바깥, StockChart가 탭 안에 있어 가격을 보면서 차트를 조작하려면 스크롤 필요
2. **탭 구조의 비효율**: "정보" 탭에 StockInfoGrid + IndicatorSummary + CompanyOverview + PeerStocks가 한꺼번에 들어있어 과부하
3. **차트 컨트롤의 복잡성**: ChartControls가 3행(기간/오버레이/패널)으로 구성되어 초보자에게 압도적. "지표 ▼" 토글이 발견하기 어려움
4. **모바일에서 가격 헤더 공간 낭비**: 종목명+배지+가격이 mb-6+mb-8로 80px 이상 차지
5. **탭 간 이동 시 상태 유실**: 차트 기간/지표 설정이 탭 전환 시 유지되지만 스크롤 위치는 초기화됨
6. **광고 위치**: Breadcrumb 바로 아래에 광고가 있어 콘텐츠 진입 전 광고 노출 — UX 저해

**컴포넌트 의존 관계:**
```
page.tsx (server)
├── Breadcrumb
├── AdSlot
└── StockTabs (client)
    ├── PriceDisplay
    ├── WatchlistButton
    ├── Tabs
    │   ├── chart: ChartTabServer → StockChart + IndicatorSummary
    │   ├── info: InfoTabServer → StockInfoGrid + CompanyOverview + PeerStocks
    │   ├── institutional: InstitutionalFlow (KR only)
    │   ├── news: NewsTabServer → NewsCard[]
    │   └── events: EventsTabWrapper → Dividend + Earnings + Disclosure
    └── Links (실시간 시세, AI 리포트, 분석 요청)
```

### 독창적 아이디어

#### 1. "Split Layout" — 차트+핵심정보 동시 노출
- **데스크톱에서 2-column 레이아웃** 도입: 좌측 메인(차트+뉴스), 우측 사이드(시세+지표+유사종목)
- 기존 탭 5개(차트/정보/수급/뉴스/이벤트) → **차트는 항상 노출**, 하단에 뉴스/이벤트/수급 탭만 유지
- "정보" 탭의 데이터를 사이드 패널로 분산 → 탭 개수 축소(4개 → 3개)

#### 2. "Sticky Price Header" — 스크롤 시 가격 항상 가시
- PriceDisplay를 `sticky top-14`(헤더 높이 아래)로 고정
- 스크롤 시 자연스럽게 축약: 큰 가격(text-3xl) → 작은 가격(text-lg)으로 `transition`
- 종목명 + 가격 + 등락률이 한 줄로 압축되어 차트 조작 중에도 가격 확인 가능

#### 3. "기술적 요약 카드" — 지표를 3개 핵심 카드로 응축
- IndicatorSummary의 20+ 지표를 **3개 핵심 카드**(RSI, MACD, 볼린저 위치)로 요약
- 각 카드에 **신호등 색상**(초록/노랑/빨강)으로 매수/중립/매도 직관 표현
- "더 보기" 클릭 시 전체 IndicatorSummary 슬라이드 오픈 (Sheet 활용)
- 초보자는 3개 카드만 보고, 고급 사용자는 펼쳐서 상세 지표 확인

#### 4. "차트 프리셋" — 초보/고급 모드 전환
- 현재 ChartControls의 "지표 ▼" 토글을 **명확한 프리셋 버튼**으로 대체
- "기본" (캔들+거래량만), "추세" (SMA+BB), "모멘텀" (RSI+MACD), "커스텀" (현재 고급 모드)
- 프리셋 선택 시 관련 오버레이/패널이 한 번에 활성화 → 선택 피로도 감소

#### 5. "Floating Action Bar" (improve-planner 전략 반영)
- 주식 상세 페이지 하단에 **고정 액션 바** (모바일 bottom-safe-area 고려)
- 3개 액션: [★ 관심종목] [⇄ 비교에 추가] [📄 AI 분석 요청]
- 현재 WatchlistButton이 헤더 우측에만 있어 발견성 낮음 → FAB에 통합
- "비교에 추가" 클릭 시 글로벌 비교 슬롯에 종목 추가 (최대 5개, 전역 상태)
- 데스크톱: 사이드 패널 하단에 고정, 모바일: `position: fixed bottom-16` (탭바 위)

#### 6. "Price Sparkle" — 가격 변동 시 마이크로 애니메이션
- 실시간 시세 업데이트 시 가격 숫자에 **shimmer 효과** 적용 (0.3s)
- 상승 시 빨강 glow, 하락 시 파랑 glow가 잠시 나타났다 사라지는 효과
- CSS `@keyframes price-flash` + `text-shadow` 로 구현, 성능 부담 최소

## 3. 마켓 페이지 (/market)

### 현재 분석

**레이아웃 구조:**
- Breadcrumb → h1 + ExchangeRateBadge → 설명 텍스트(2 단락) → Tabs(한국/미국) → 각 탭 내: IndexCard×2 → SectorPerformance → 상승/하락 TOP 5

**문제점:**
1. **설명 텍스트의 과잉**: 2단락 분량의 설명이 "시장 개요" 제목 바로 아래에 — 반복 방문자에게 불필요하며 핵심 데이터를 아래로 밀어냄
2. **한국/미국 탭 전환 방식**: 한 시장만 볼 수 있어 KR/US 동시 비교 불가. 글로벌 투자자는 두 시장을 오가며 확인해야 함
3. **섹터 성과의 시각적 약함**: 2×3 그리드에 텍스트+숫자만 — 한눈에 강세/약세 섹터를 구분하기 어려움
4. **상승/하락 TOP 5의 단순 리스트**: StockRow가 동일한 스타일로 나열되어 1위와 5위의 시각적 구분이 없음
5. **환율 정보 부족**: ExchangeRateBadge 하나(USD/KRW)만 헤더에 있고 다른 환율 정보 없음

**컴포넌트 의존 관계:**
```
page.tsx (server)
├── ExchangeRateBadge (USD/KRW)
├── Tabs (한국/미국)
│   ├── IndexCard × 2
│   ├── SectorPerformance (client, useQuery)
│   │   └── Dialog → SectorStockList
│   └── StockRow × 10 (상승5 + 하락5)
└── AdSlot
```

### 독창적 아이디어

#### 1. "섹터 히트맵 (Treemap)" — 트리맵으로 섹터 성과 시각화
- 기존 `SectorPerformance`의 2×3 텍스트 그리드를 **Treemap 시각화**로 대체
- 각 섹터의 **면적 = 시가총액 비중**, **색상 = 등락률**(빨강 그래디언트~파랑 그래디언트)
- 클릭하면 기존 Dialog로 섹터 내 종목 리스트 표시
- SVG 기반으로 구현 (d3 없이 순수 계산), 모바일에서는 터치 인터랙션
- **기존 금융앱에서 볼 수 없는 참신한 시각화** — finviz.com의 히트맵을 한국/모바일에 최적화

#### 2. "4열 Overview Cards" — KR/US 탭 제거, 동시 비교
- 현재 탭(한국/미국)을 제거하고 **4개 지수를 한 줄에 동시 표시**
- 각 카드에 5일 미니 차트(스파크라인 확장 — 면적 차트 형태)
- 모바일에서 2×2 그리드로 자연스럽게 배치
- KR/US 탭은 섹터 히트맵과 TOP 5 영역에서만 유지 (시장별 데이터가 의미 있는 곳)

#### 3. "Ranked Bar" — 상승/하락 종목에 등락률 비례 바 차트
- StockRow를 확장하여 각 종목의 등락률을 **수평 바**로 시각화
- 1위 = 가장 긴 바, 5위 = 가장 짧은 바 → 상대적 강도를 직관적으로 비교
- 상위 3개에 메달 아이콘(금/은/동) 적용 → 시각적 계층 강화
- 바 호버 시 종목 상세 미니 팝업 (가격, 거래량, 섹터)

#### 4. "설명 텍스트" 제거 → 첫 방문 시 투어 툴팁
- 매번 표시되는 2단락 설명을 제거하고, **첫 방문 시에만 각 섹션에 툴팁 가이드** 표시
- localStorage로 상태 관리, "다시 보지 않기" 옵션
- 반복 방문자는 설명 없이 바로 데이터 확인 가능 → 데이터 밀도 향상

| 로딩 | `Skeleton` + `animate-pulse` | Skeleton 유지 + **shimmer 그래디언트** 애니메이션 추가 |

| 스크롤 | 없음 | 주요 리스트에 `scroll-snap-type: x mandatory` (수평 스크롤 영역) |

### 4.5 반응형 전략 (Breakpoints)

```
모바일 (< 768px):
  - 1열 레이아웃 기본
  - 지수 카드: 2×2 그리드
  - 주식 상세: 단일 컬럼, 차트 풀와이드
  - 하단 탭바 네비게이션 활용 (bottom-tab-bar.tsx 이미 존재)

태블릿 (768px ~ 1024px):
  - 2열 레이아웃
  - 사이드 패널: 축소 또는 접기 가능

데스크톱 (> 1024px):
  - 주식 상세: 2열 (메인 + 사이드)
  - 마켓: 4열 지수 카드
  - 홈: 벤토 그리드 3열
```
