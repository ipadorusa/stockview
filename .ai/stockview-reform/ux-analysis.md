### 1.2 네비게이션 구조

**데스크톱 (lg+):**
- 상단 고정 헤더: 로고 + 8개 주요 메뉴 + 검색바 + 테마 토글 + 로그인/사용자 메뉴

- 하단 고정 탭바: 홈, 시장, 뉴스, MY (4개 탭)

## 2. UX 분석: 강점

### 2.1 잘 설계된 부분

1. **일관된 레이아웃 시스템**: `PageContainer`로 통일된 max-width(screen-xl), 패딩, 구조
2. **한국 투자 컨벤션 준수**: 빨강=상승, 파랑=하락 (한국 주식 색상 규칙)
3. **다크모드 대응**: 주식 색상까지 다크모드별 별도 변수 정의 (stock-up: #e53e3e / dark: #fc8181)
4. **프리렌더링 + 클라이언트 하이드레이션**: SSR + React Query HydrationBoundary로 초기 로딩 최적화
5. **Skeleton UI**: 모든 주요 데이터 로딩 시 Skeleton 컴포넌트 활용
6. **Breadcrumb 일관 적용**: 주요 페이지에 SEO 친화적 breadcrumb 배치
7. **검색 UX**: 디바운스 300ms, Command 패턴, Cmd+K 단축키 지원 (데스크톱)
8. **투자 교육 콘텐츠**: 각 페이지(시장, ETF, 스크리너, 배당)에 초보자용 설명 섹션 배치
9. **접근성 기본**: sr-only 라벨, aria-label 검색, 키보드 접근 가능한 폼
10. **관심종목 Optimistic Update**: 관심종목 토글 시 즉각적 UI 반응 (useMutation + onMutate)

### 2.2 정보 구조

- 홈페이지가 대시보드 역할: 지수 + 환율 + 인기 종목 + 최신 뉴스를 한 눈에 제공
- 종목 상세 페이지가 풍부: 차트, 시세, 뉴스, 공시(KR), 배당, 실적 6개 탭
- 가이드 → 기능 페이지 연결이 잘 되어 있음 (예: 기술적 지표 가이드 → 스크리너)

## 3. UX 분석: 개선 필요 영역

### 3.1 네비게이션 문제

#### [HIGH] 모바일 하단 탭바와 헤더 메뉴 불일치
- **현황**: 헤더 메뉴 8개 vs 하단 탭바 4개 (홈, 시장, 뉴스, MY)
- **문제**: ETF, 스크리너, AI 리포트, 게시판은 모바일에서 하단 탭으로 접근 불가. 햄버거 메뉴를 열어야만 접근 가능.
- **영향**: 핵심 기능인 스크리너, AI 리포트의 모바일 접근성 저하. 사용자가 기능 존재를 인지하지 못할 가능성.

#### [HIGH] 숨겨진 페이지들 (네비게이션 미노출)
- **현황**: 다음 페이지들은 헤더/탭바 어디에도 링크 없음:
  - `/compare` (종목 비교) - 직접 URL 입력 또는 다른 경로 필요
  - `/sectors` (섹터별 종목) - 시장 페이지 내 SectorPerformance에서만 접근 가능
  - `/dividends` (배당 캘린더) - 네비게이션 미노출
  - `/earnings` (실적 캘린더) - 네비게이션 미노출
  - `/guide` (투자 가이드) - 네비게이션 미노출
- **영향**: 전체 콘텐츠의 약 30%가 사용자에게 발견되지 못할 위험

#### [MED] 모바일 마이페이지 플로우
- **현황**: 하단 탭 MY → `/mypage`로 이동. 미로그인 시 빈 상태가 아닌 로딩 상태만 보여줌.
- **문제**: 워치리스트 페이지(`/watchlist`)는 미인증 시 로그인 유도 UI가 있으나, 마이페이지는 로딩 스켈레톤만 표시 후 빈 프로필 카드.

### 3.2 정보 구조 문제

#### [HIGH] 기능 분산 및 중복
- **배당 정보**: 종목 상세 → 배당 탭, `/dividends` 배당 캘린더, 가이드의 배당 투자 문서 - 3곳에 분산
- **실적 정보**: 종목 상세 → 실적 탭, `/earnings` 실적 캘린더 - 상호 연결 부족
- **뉴스**: 홈 → 최신 뉴스, 종목 상세 → 뉴스 탭, `/news` 뉴스 페이지 - 각각 독립적

#### [MED] 종목 상세 페이지 정보 과부하
- **현황**: 헤더 영역에 종목명, 티커, 거래소, 섹터, 가격, 변동, 업데이트 시간, 실시간 시세 링크, AI 리포트 링크, 기업 개요가 스크롤 없이 노출
- **문제**: 모바일에서 탭 콘텐츠에 도달하기까지 스크롤이 많이 필요

#### [LOW] 홈페이지 CTA 부재
- **현황**: 대시보드 형태로 정보만 나열. 미로그인 사용자에 대한 서비스 소개나 행동 유도 없음.
- **문제**: 첫 방문 사용자가 서비스 가치를 즉시 파악하기 어려움

### 3.3 반응형 디자인

#### [MED] 하단 탭바와 Footer 겹침
- **현황**: `<div className="pb-14 lg:pb-0">` 으로 하단 패딩 처리, Footer에 `mb-14 md:mb-0`
- **문제**: md 브레이크포인트(768px)와 lg 브레이크포인트(1024px) 사이에서 Footer margin-bottom이 0이지만 하단 탭바는 여전히 표시됨 → Footer 하단 일부가 탭바에 가려질 수 있음

#### [MED] 종목 비교 페이지 모바일 UX
- **현황**: 3-column 그리드 (`grid-cols-[1fr_1fr_1fr]`)가 모바일에서도 동일하게 적용
- **문제**: 375px 화면에서 각 셀이 약 125px로 좁아져 가독성 저하

#### [LOW] 배당 캘린더 테이블 모바일 가독성
- **현황**: `grid-cols-[1fr_auto_auto_auto]` 4열 레이아웃
- **문제**: 소형 화면에서 열이 너무 좁아짐. 수평 스크롤 또는 카드형 레이아웃 필요

### 3.4 사용자 플로우

#### [HIGH] 검색 → 종목 접근 플로우
- **현황**: 검색 결과에서 ETF는 `/etf/[ticker]`, 일반 종목은 `/stock/[ticker]`로 분기
- **문제**: 사용자는 종목이 ETF인지 아닌지 검색 전에 모를 수 있음. 두 URL 구조가 다른 것은 SEO에는 좋으나 사용자 멘탈 모델과 불일치.

#### [MED] 로그인/회원가입 플로우
- **현황**: 모바일에서 로그인/회원가입 버튼이 `hidden lg:flex`로 숨겨져 있음
- **문제**: 모바일 사용자는 햄버거 메뉴를 열어야 로그인/회원가입 가능. 하단 탭의 MY 탭이 `/mypage`로 보내지만, 미인증 시 적절한 유도 없음.

#### [MED] 관심종목 추가 피드백
- **현황**: WatchlistButton이 종목 상세에서만 사용 가능
- **문제**: 종목 리스트(시장, ETF, 스크리너)에서는 관심종목 추가 불가. 반드시 상세 페이지에 진입해야 함.

### 3.5 시각 디자인 및 일관성

#### [MED] 색상 체계
- **현황**: 모노크롬 기반 (oklch). Primary가 거의 검정(oklch 0.205). 강조색 부재.
- **문제**: 브랜드 아이덴티티가 약함. 모든 UI가 흑백 톤으로 시각적 위계 구분이 어려움. CTAㅂ버튼과 일반 텍스트의 시각적 구분이 미흡.

#### [MED] AdSlot 배치
- **현황**: 거의 모든 페이지 하단에 AdSlot 배치, 종목 상세는 상단(Breadcrumb 아래)에도 배치
- **문제**: 종목 상세 페이지에서 AdSlot이 Breadcrumb과 StockTabs 사이에 위치하여 핵심 콘텐츠 접근을 방해. 광고가 콘텐츠보다 먼저 보이는 구간 존재.

#### [LOW] 타이포그래피
- **현황**: Pretendard Variable 사용. 제목은 text-2xl font-bold, 본문은 text-sm.
- **관찰**: 중간 크기(text-base, text-lg)의 활용이 적어 시각적 위계가 2단(제목/본문)으로 단순함.

### 3.6 접근성

#### [MED] 키보드 네비게이션
- **현황**: Sheet(사이드바)와 DropdownMenu는 Radix 기반으로 키보드 접근 양호
- **문제**: SearchBar의 dropdown 결과 목록이 커스텀 `div`로 구현되어 있어, 키보드로 검색 결과 선택이 제한적일 수 있음

#### [LOW] 색상 대비
- **현황**: muted-foreground가 oklch(0.556), background가 oklch(1) → 약 4.1:1 대비비
- **문제**: WCAG AA 기준(4.5:1)에 약간 미달할 수 있음. 특히 text-xs + muted-foreground 조합에서.

## 4. 페이지별 상세 분석

### 4.1 홈페이지 (`/`)

**구성**: 주요 지수 → 환율 → 인기 종목(탭: 한국/미국) → 최신 뉴스 → 광고
**장점**:
- 대시보드형 레이아웃으로 핵심 정보 한눈에 파악
- Suspense로 뉴스 섹션 스트리밍
- 인기 종목이 거래대금 기준으로 정렬되어 실용적

**개선점**:
- 첫 방문자를 위한 hero 섹션이나 서비스 소개 부재
- 환율 섹션이 5개(USD, EUR, JPY, CNY, GBP) 나열되어 모바일에서 공간 차지 큼
- "전체 보기" 링크가 시각적으로 약함 (text-sm text-primary)

### 4.2 시장 페이지 (`/market`)

**구성**: 지수 → 탭(한국/미국) → 섹터별 성과 → 상승/하락 TOP 5
**장점**: 한/미 시장 탭 분리로 깔끔한 구조
**개선점**:
- 상승/하락 TOP 5가 고정. 사용자가 더 보고 싶으면 방법 없음
- 섹터 성과 → 섹터 상세 페이지로의 연결이 명확하지 않음

### 4.3 종목 상세 페이지 (`/stock/[ticker]`)

**구성**: Breadcrumb → 광고 → 종목 헤더 → 가격 → 기업 개요 → 6탭(차트/시세/뉴스/공시/배당/실적)
**장점**:
- 풍부한 정보량 (6개 탭)
- 차트 데이터 prefetch로 빠른 초기 로딩
- 관심종목 Optimistic Update

**개선점**:
- 광고가 Breadcrumb 바로 아래에 위치 → 콘텐츠 접근 지연
- 모바일에서 탭까지 스크롤이 길음 (헤더+가격+개요)
- 탭이 6개로 많아 모바일에서 `flex-wrap` 적용 시 2줄이 될 수 있음

### 4.4 관심종목 페이지 (`/watchlist`)

**장점**: 미인증 시 깔끔한 빈 상태 + 로그인 유도
**개선점**:
- 삭제 버튼이 데스크톱에서 hover 시에만 나타남 → 터치 디바이스에서 항상 표시되어야 함 (현재 `lg:opacity-0 lg:group-hover:opacity-100`으로 처리됨 - 양호)
- 정렬/필터 옵션 없음 (수동 정렬 불가)
- 그룹핑 기능 없음 (한국/미국 구분 없이 일렬 나열)

### 4.5 뉴스 페이지 (`/news`)

**장점**: SSR 초기 데이터 + 클라이언트 측 필터링
**개선점**: 뉴스 → 관련 종목 연결이 약함 (종목 태깅은 있으나 UI에서의 활용 미확인)

### 4.6 종목 비교 (`/compare`)

**장점**: 심플한 2종목 비교 테이블
**개선점**:
- 티커 직접 입력 방식 → 자동완성/검색 기능 없음
- 모바일에서 3열 그리드 가독성 문제
- 네비게이션에서 접근 경로 없음

## 5. 사용자 플로우 분석

### 5.1 핵심 플로우: 종목 탐색 → 분석 → 관심종목 추가

```
[홈/시장/검색] → [종목 상세] → [탭 탐색] → [관심종목 추가]
                                           → [AI 리포트 확인]
                                           → [실시간 시세 (외부)]
```

**평가**: 기본 플로우는 양호하나, 종목 리스트에서 관심종목 직접 추가가 불가하여 단계가 1개 더 필요.

### 5.2 핵심 플로우: 신규 사용자 온보딩

```
[첫 방문 (홈)] → ??? → [회원가입] → [관심종목 추가]
```

**평가**: 온보딩 플로우 부재. 첫 방문자에게 서비스 가치를 전달하는 단계 없음. 홈이 바로 대시보드.

### 5.3 핵심 플로우: 스크리닝 → 투자 판단

```
[스크리너] → [신호별 종목 리스트] → [종목 상세] → [차트 확인] → [AI 리포트]
```

**평가**: 플로우 자체는 논리적이나, 스크리너에서 종목 상세로 이동 시 컨텍스트(어떤 신호로 왔는지) 유실.

## 6. 기술적 UX 관찰

### 6.1 성능 관련
- ISR(Incremental Static Regeneration) 15분 주기로 데이터 갱신 — 적절
- 뉴스 페이지는 5분 주기 — 뉴스 성격에 맞음
- 차트 데이터 prefetch + React Query staleTime 5분 — 양호
- `Promise.all`로 병렬 데이터 fetch — 양호

### 6.2 SEO
- JSON-LD 구조화 데이터 (Organization, WebSite, WebPage, FinancialProduct) — 양호
- OG/Twitter 메타 태그 — 양호
- sitemap.ts + robots.ts — 양호
- Breadcrumb 컴포넌트 — 양호

### 6.3 분석/모니터링

## 7. 개선 우선순위 요약

### P0 (즉시 개선)
1. 모바일 네비게이션 재설계 — 핵심 기능(스크리너, AI 리포트, 배당/실적) 접근성 확보
2. 숨겨진 페이지 발견성 개선 — 배당, 실적, 가이드, 종목 비교 접근 경로 확보
3. 종목 상세 페이지 모바일 최적화 — 광고 위치 조정, 헤더 압축

### P1 (단기 개선)
4. 브랜드 컬러 도입 — 모노크롬 탈피, 시각적 위계 강화
5. 신규 사용자 온보딩 플로우 — 홈페이지 히어로 섹션 또는 가이드 모달
6. 종목 리스트에서 관심종목 원터치 추가 기능
7. 종목 비교 페이지 검색 자동완성 추가
8. 관심종목 한국/미국 그룹핑 + 정렬 옵션

### P2 (중기 개선)
9. Footer-탭바 브레이크포인트 정합성 수정 (md → lg)
10. 종목 비교 모바일 레이아웃 개선
11. 배당 캘린더 모바일 카드형 레이아웃
12. 접근성 감사 — 색상 대비 개선, ARIA 보강
13. 정보 아키텍처 재구성 — 분산된 배당/실적 정보 허브화

## 9. 결론

StockView는 기능적으로 풍부하고 기술적으로 잘 구축된 서비스이다. SSR/ISR 전략, React Query 활용, SEO, 접근성 기본 등이 양호하게 구현되어 있다.

그러나 **정보 아키텍처(IA)와 네비게이션** 측면에서 개선이 시급하다. 37개 페이지 중 약 30%가 네비게이션에서 발견 불가능하며, 모바일 하단 탭바가 4개 탭으로 제한되어 핵심 기능의 접근성이 떨어진다.

시각 디자인은 깔끔하지만 **브랜드 아이덴티티가 약하고**, 모노크롬 팔레트로 인해 시각적 위계 구분이 어렵다. 첫 방문자를 위한 온보딩 경험이 부재하며, 종목 비교 같은 유용한 기능이 접근 경로 없이 묻혀 있다.

개편의 핵심 방향은:
1. **네비게이션/IA 재설계** — 모든 기능의 접근 경로 보장
2. **모바일 경험 강화** — 하단 탭 재구성, 종목 상세 최적화
3. **브랜드/시각 아이덴티티** — 컬러 시스템 도입, 시각 위계 강화
4. **사용자 플로우 최적화** — 온보딩, 관심종목 원터치 추가, 컨텍스트 유지
