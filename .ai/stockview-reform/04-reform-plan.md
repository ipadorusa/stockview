# StockView 개편 기획안 (최종)

> 작성일: 2026-03-25
> 기반 자료: 서비스 전체 분석, UX/UI 분석, 기능/데이터 분석

---

## 1. 개편 배경 및 목표

### 1.1 현재 상태 요약

StockView는 KR/US 듀얼마켓 주식 정보 플랫폼으로, 39개 페이지, 46개 API, 16개 DB 모델, 13개 cron job 규모의 서비스다. SSR/ISR 전략, React Query, SEO, 기술적 스크리너, AI 리포트 등 기술적으로 잘 구축되어 있으나, **데이터 품질**, **정보 접근성**, **사용자 경험** 측면에서 구조적 한계가 존재한다.

### 1.2 핵심 문제 3가지

| # | 문제 | 영향 |
|---|------|------|
| 1 | **데이터 신뢰도 부족** — 시세 1일 1회, DailyPrice 21일 삭제, Fundamentals 48주 갱신 주기 | 사용자가 장중 방문 시 구형 데이터, 장기 차트 불가, 부정확한 펀더멘탈 |
| 2 | **정보 접근성 결함** — 전체 페이지의 ~30%가 네비게이션 미노출, 모바일 하단탭 4개로 핵심 기능 접근 불가 | 배당, 실적, 가이드, 종목비교 등 유용한 기능이 사용자에게 발견되지 못함 |
| 3 | **사용자 여정 미완성** — 온보딩 부재, 포트폴리오 미지원, 알림 없음, 리스트에서 관심종목 추가 불가 | 신규 사용자 이탈, 재방문 동기 부족, 핵심 사용자 여정에 마찰 |

### 1.3 개편 목표

1. **데이터 신뢰도 확보**: 장중 시세, 장기 차트, 빠른 펀더멘탈 갱신
2. **전체 기능의 발견성 보장**: 네비게이션 재설계, 모바일 경험 강화
3. **사용자 여정 완성**: 탐색 → 분석 → 투자 판단 → 추적 사이클

## 2. 개편 영역 및 우선순위

### Phase 1: 즉시 개선 (1~2주) — "신뢰도와 접근성"

데이터 품질과 접근성의 기본을 바로잡는 단계. 기존 코드의 설정값 변경이나 소규모 수정으로 해결 가능한 항목 위주.

#### 2.1.1 데이터 품질 즉시 수정

| 항목 | 현재 | 개선 | 난이도 |
|------|------|------|--------|
| **DailyPrice 보존 기간** | 21일 삭제 | 365일 이상 보존 (cleanup 크론 수정) | Low |
| **Fundamentals 배치 크기** | 100개/주1회 | 500개/주2~3회 | Low |
| **Compare 페이지 API 버그** | `/api/stock/` (404) | `/api/stocks/` 로 수정 | Low |
| **PBR 저장 일원화** | StockQuote + StockFundamental 중복 | StockFundamental로 통합 | Medium |
| **52주 고저 일괄 수집** | 개별 요청 (지연 발생) | 크론에서 사전 수집 | Medium |

#### 2.1.2 네비게이션 재설계

**데스크톱 헤더 개편:**

현재 8개 플랫 링크 → **그룹화된 메가메뉴** 구조로 전환

```
로고  |  [시장▾]  [투자도구▾]  [뉴스]  [커뮤니티▾]  |  검색  테마  로그인
         ├ 시장 개요      ├ 스크리너          ├ 게시판
         ├ 섹터           ├ AI 리포트         ├ 투자 가이드
         ├ ETF            ├ 종목 비교
         └ 환율           ├ 배당 캘린더
                          └ 실적 캘린더
```

**모바일 하단 탭바 개편:**

현재 4탭 → **5탭**으로 확장

```
[홈]  [시장]  [투자도구]  [뉴스]  [MY]
                 ↓
        (터치 시 드로어/시트)
        - 스크리너
        - AI 리포트
        - 종목 비교
        - 배당 캘린더
        - 실적 캘린더
```

#### 2.1.3 광고 위치 조정

- 종목 상세 페이지: Breadcrumb-StockTabs 사이 AdSlot → StockTabs 아래(정보 탭 내부)로 이동
- 모바일에서 첫 화면에 광고가 보이지 않도록 조정

### Phase 2: 핵심 기능 강화 (1~2개월) — "사용자 여정 완성"

사용자가 StockView에서 투자 판단까지 완결할 수 있도록 핵심 기능을 보강하는 단계.

#### 2.2.1 장중 시세 도입

| 항목 | 상세 |
|------|------|
| **KR 장중 폴링** | 09:00~15:30 KST, 5분 간격 Naver polling API |
| **US 장중 폴링** | 09:30~16:00 ET (22:30~05:00 KST), 5분 간격 Yahoo v8 API |
| **구현 방식** | 별도 크론 또는 Vercel Cron Functions |
| **UI 표시** | "5분 전 기준" 타임스탬프, 장중/장마감 상태 배지 |

#### 2.2.2 포트폴리오 기능 (Watchlist 확장)

현재 Watchlist(단순 종목 리스트)를 포트폴리오로 확장:

**DB 스키마 변경:**
```
Watchlist 확장 또는 Portfolio 신규 모델:
- userId, stockId (기존)
- buyPrice (매수 단가)
- quantity (수량)
- buyDate (매수일)
- memo (메모)
- groupName (그룹: "한국 배당주", "미국 성장주" 등)
```

**UI:**
- 관심종목 페이지에 "간편 포트폴리오" 모드 추가
- 수익률(현재가 vs 매수가) 실시간 계산
- KR/US 그룹별 소계
- 종목 리스트(시장, ETF, 스크리너)에서 관심종목 원터치 추가 버튼

#### 2.2.3 스크리너 강화

현재 5개 기술적 신호 → **펀더멘탈 필터 추가**:

| 필터 | 조건 예시 |
|------|-----------|
| PER | < 10, 10~20, > 20 |
| PBR | < 1, 1~3, > 3 |
| 배당수익률 | > 3%, > 5% |
| 시가총액 | 대형주/중형주/소형주 |
| ROE | > 10%, > 15% |

**UI:** 기술적 신호 탭 + 펀더멘탈 필터 탭 병렬 구조

#### 2.2.4 종목 비교 개선

| 현재 | 개선 |
|------|------|
| ticker 직접 입력 | 검색 자동완성 (기존 SearchCommand 재활용) |
| 2개 종목만 | 최대 4개 종목 비교 |
| 네비게이션 미노출 | "투자도구" 메뉴에 포함 |

#### 2.2.5 StockFundamental 이력화

| 현재 | 개선 |
|------|------|
| 1:1 최신 스냅샷만 | FundamentalHistory 모델 추가 (stockId + quarter unique) |
| 추이 분석 불가 | 분기별 EPS/매출/영업이익 차트 |
| 주1회 덮어쓰기 | 분기별 이력 누적 보존 |

#### 2.2.6 신규 사용자 온보딩

- 홈페이지 상단에 조건부 히어로 섹션 (첫 방문 or 미로그인 시)
  - "초보 투자자를 위한 주식 분석 서비스"
  - 핵심 기능 3개 하이라이트 (스크리너, AI 리포트, 배당 캘린더)
  - "시작하기" CTA → 회원가입 또는 인기종목
- 투자 가이드를 네비게이션에 노출

#### 2.2.7 브랜드 컬러 도입

- 현재: 모노크롬 (oklch 0.205 primary ≈ 거의 검정)
- 개선: 브랜드 Primary 컬러 도입 (예: 금융 서비스에 적합한 블루/그린 계열)
- CTA 버튼, 액센트, 활성 상태 등에 적용하여 시각적 위계 강화

### Phase 3: 차별화 기능 (3개월+) — "경쟁 우위"

경쟁 서비스 대비 차별화 포인트를 구축하는 단계.

#### 2.3.1 알림 시스템

| 알림 유형 | 트리거 | 채널 |
|-----------|--------|------|
| 목표가 도달 | 시세 갱신 시 사용자 설정 가격 체크 | 웹 푸시, 이메일 |
| 기술적 신호 | 스크리너 신호 감지 시 관심종목 매칭 | 웹 푸시 |
| 실적 발표 D-1 | 실적 캘린더 기반 | 웹 푸시 |
| 배당락일 D-3 | 배당 캘린더 기반 | 웹 푸시 |

**DB:** UserAlert 모델 (userId, type, condition, isActive, lastTriggeredAt)

#### 2.3.2 외국인/기관 수급 데이터 (KR)

- KRX 일별 투자자별 매매동향 수집
- 종목 상세 페이지에 "수급" 탭 추가
- 외국인 순매수/순매도 TOP 종목

#### 2.3.3 MarketIndex 이력

- MarketIndexHistory 모델 추가 (symbol + date)
- 지수 차트 기능 (홈, 시장 페이지)
- 지수 클릭 시 상세 차트 모달 또는 페이지

#### 2.3.4 Sector 모델 정규화

- 현재: Stock.sector (문자열) → Sector 테이블로 분리
- 섹터별 지수 계산 가능
- 섹터 성과 히스토리 추적

#### 2.3.5 뉴스 감성 분석

- News.sentiment 필드 활용 (현재 미사용)
- LLM 기반 감성 분석 파이프라인 (크론)
- 종목별/시장별 감성 지수 대시보드

#### 2.3.6 AI 리포트 자동화/고도화

- 현재: Ollama(로컬) 수동 트리거
- 개선: 크론 기반 자동 생성 (스크리너 신호 감지 → 해당 종목 자동 리포트)
- 클라우드 LLM API(Claude 등)로 품질 향상 검토

#### 2.3.7 US 종목 확대

- 현재: S&P 500 (~500개)
- 확장: NASDAQ 100 + Russell 2000 주요 종목 → ~1,000개
- Yahoo Finance API로 추가 수집 (기존 인프라 활용)

## 4. 크론잡 최적화

### 현재 구조 (13개 독립 크론)
```
collect-master (토 22:00)
collect-kr-quotes (평일 16:00) → compute-indicators
collect-kr-etf-quotes (별도)
collect-us-quotes (평일 21:15 UTC) → compute-indicators
collect-news (2시간 간격)
collect-fundamentals (토 23:00)
collect-exchange-rate
collect-events
collect-disclosures
collect-dart-dividends
sync-corp-codes
sync-kr-sectors
cleanup
```

### 개선안

1. **KR 파이프라인 통합**: collect-kr-quotes + collect-kr-etf-quotes + exchange-rate → 단일 워크플로우
2. **compute-indicators market 분리**: `?market=KR` / `?market=US` 파라미터로 중복 실행 방지
3. **장중 폴링 추가 시**: GitHub Actions 대신 Vercel Cron 또는 외부 스케줄러 검토 (5분 간격은 GA 무료 tier에서 불가)
4. **Fundamentals 갱신 가속**: 배치 100→500, 주1회→주3회, 우선순위 큐 (관심종목/인기종목 우선)

## 7. 기술적 제약 및 고려사항

1. **Vercel 타임아웃 (60초)**: 대량 배치 처리에 제약. 장중 폴링 시 경량 요청으로 설계 필요.
2. **Supabase 무료 tier**: DB 용량 500MB. DailyPrice 365일 보존 시 용량 모니터링 필요.
3. **GitHub Actions 무료 tier**: 월 2,000분. 장중 5분 폴링은 GA 대신 Vercel Cron 검토.
4. **Naver 스크래핑 의존성**: KR 데이터의 핵심 소스. HTML 변경 모니터링 체계 필수.
5. **NextAuth 5 beta**: 안정 버전 출시 시 마이그레이션 필요 가능성.
